# BondClaw V1 发布前任务卡片

本文档把收尾阶段的关键工作拆成更细的任务卡片，方便直接分配、追踪和验收。  
每张卡片都尽量写清输入、输出、完成标准和优先级。

## 使用规则

- P0 卡片优先执行，没做完不要进入正式发布。
- P1 卡片决定交付质量，建议与 P0 并行准备。
- P2 卡片用于回归和文档收口。

## P0 卡片

### 卡片 1: Windows 安装包

- 目标：产出可安装的 Windows 版本。
- 输入：当前主分支、release manifest、品牌配置、更新配置。
- 输出：安装包、卸载路径说明、首次启动可用性结果。
- 完成标准：
  - 干净 Windows 环境可安装、启动、卸载。
  - 桌面快捷方式可用。

### 卡片 2: 自动升级确认

- 目标：确认升级链路稳定。
- 输入：release manifest、更新仓库、版本号、渠道信息。
- 输出：升级验证结果、版本与渠道截图或记录。
- 完成标准：
  - 刷新后能稳定拿到远程清单。
  - 升级不影响品牌和首页状态。

### 卡片 3: 发布产物与签名

- 目标：统一正式发布产物。
- 输入：构建产物、命名规则、发布仓库、支持页、官网。
- 输出：发布包、清单、签名或校验说明。
- 完成标准：
  - 命名一致。
  - 外部可识别。
  - 可被校验或签名确认。

## P1 卡片

### 卡片 4: Windows 原生验证

- 目标：确认 Windows 端核心路径全部走原生执行。
- 输入：Windows 机器、BondClaw 安装包。
- 输出：安装、首次启动、命令执行、外链、路径验证记录。
- 完成标准：
  - 保持 Windows 原生默认路径。
  - 主要动作都能稳定执行。

### 卡片 5: 联系方式投递链路

- 目标：定版邮箱、存储、归档三路投递。
- 输入：提交表单、回执格式、队列格式、重试策略。
- 输出：投递状态定义、失败重试规则、幂等规则。
- 完成标准：
  - 每条提交都能清楚看到三路状态。
  - 断网重试可恢复。

### 卡片 6: 后台最小查询页

- 目标：能查看提交、状态和重试结果。
- 输入：本地队列、投递回执、最小查询需求。
- 输出：轻量查询页或查询视图说明。
- 完成标准：
  - 不依赖重型 CRM。
  - 至少可人工排查和回看。

### 卡片 7: 模板中心审校

- 目标：复核 7 个角色的 20 个 starter。
- 输入：模板中心目录、manifest、输出格式、QA 清单。
- 输出：审校记录、问题列表、修正项。
- 完成标准：
  - 模板中心可以直接作为工作入口。

### 卡片 8: 信息中心审校

- 目标：复核 20 条案例及其跳转关系。
- 输入：案例库、主题总览、来源卡、Prompt 映射。
- 输出：审校记录、标签修正、跳转修复项。
- 完成标准：
  - 来源 -> 主题 -> 案例 -> Prompt 的链路完整。

### 卡片 9: 设置页整理

- 目标：统一品牌、升级、在线入口和说明卡口径。
- 输入：settings 快照、品牌配置、升级配置。
- 输出：视觉/文案收口记录。
- 完成标准：
  - 设置页能承担品牌、升级和在线入口信息展示。

## P2 卡片

### 卡片 10: Smoke test

- 目标：建立最小可重复检查流程。
- 输入：启动页、模板中心、信息中心、联系方式、设置页。
- 输出：测试脚本或人工检查清单。
- 完成标准：
  - 每次发版前都能快速跑一遍。

### 卡片 11: 文档补齐

- 目标：补齐快速开始、常见故障、发布说明、维护说明。
- 输入：当前仓库文档、收尾计划、检查表、验证计划。
- 输出：更新后的文档集。
- 完成标准：
  - 新人可按文档独立理解项目。

## 推荐分工

- 工程侧优先承担：卡片 1、2、3、4、5、6。
- 产品/内容侧优先承担：卡片 7、8、9。
- 发版保障侧优先承担：卡片 10、11。

## 备注

- 如果需要继续细拆，可以把每张卡片再拆成“设计 / 实施 / 验证”三个子卡片。
- 当前优先级仍然是 P0，再到 P1，最后 P2。
