# AI辅助固收研究系列：第7篇

# 弘则信用债指数：基于收益率分位数的市场分层方法

---

## 摘要

传统的信用债分析框架以外部信用评级（AAA/AA+/AA等）为核心分层依据，但评级调整往往滞后于市场变化，同一评级内的债券风险特征也可能差异巨大。弘则信用债指数项目创新性地提出"收益率分位数分层法"，直接以市场定价结果（收益率）为依据，构建了40个细分指数矩阵，为研究信用债市场结构和信用利差提供了新的分析工具。本文详细阐述了该方法的设计理念、技术实现和应用场景，并总结了AI辅助研究中的关键心得。

---

## 一、研究背景：为什么需要市场视角的信用分层

### 1.1 传统信用评级的局限性

信用债市场是中国债券市场的重要组成部分。2026年开年，市场呈现"利率弱、信用强"的分化格局，信用债收益率普遍下行，信用利差处于历史低位。在这一背景下，信用债研究的重要性愈发凸显。

传统的信用债分析框架依赖外部信用评级作为主要分层依据。这种方法存在几个问题：

**第一，评级调整滞后。** 信用评级机构的评级调整往往滞后于市场变化。当一家发行人的基本面恶化时，市场往往先于评级机构做出反应，收益率率先上行。等到评级下调时，市场价格已经充分反映了风险。

**第二，评级颗粒度不足。** 同一个AA+评级下，可能包含了风险特征差异巨大的债券。有的AA+债券流动性极好，有的则几乎没有成交；有的AA+债券对应的是央企子公司，有的则是地方国企。评级标签无法区分这些差异。

**第三，评级覆盖不完整。** 部分私募债、定向工具等品种可能没有公开评级，传统方法难以将其纳入分析框架。

### 1.2 市场定价：最真实的"评级"

收益率是市场参与者用真金白银投票形成的价格。它综合反映了市场对债券的所有风险判断：

- **信用风险溢价**：市场对违约概率的评估
- **流动性溢价**：市场对变现难度的补偿要求
- **期限风险**：市场对久期风险的定价
- **其他结构性因素**：税收、质押资格等

换言之，收益率本身就是市场对债券的"综合评级"。与其依赖滞后的外部评级，不如直接用市场定价结果作为分层依据。

### 1.3 研究问题的提出

基于上述思考，弘则信用债指数项目提出以下研究问题：

> 能否以收益率作为分层依据，构建一套反映市场实时风险分层的信用债指数体系？

这套指数体系需要满足几个要求：
1. 能够实时反映市场风险偏好变化
2. 分层标准客观、透明、可复现
3. 计算方法与主流指数（如中债指数）一致，便于对比分析
4. 能够支持多维度分析（债券类型、价格类型等）

## 二、核心方法：收益率分位数分层法

### 2.1 方法论概述

本项目的核心方法论是"收益率分位数分层法"。其基本逻辑如下：

**每日将样本债券按收益率升序排列，均匀划分为10个分位数组。**

具体而言，第1组（0-10%分位数）包含收益率最低的10%债券，代表市场认为风险最低的债券；第10组（90-100%分位数）包含收益率最高的10%债券，代表市场认为风险最高的债券。

这种方法的核心逻辑是：收益率直接反映市场对债券风险的综合定价。通过观察不同分位数组的指数表现，可以直观揭示市场的风险分层特征。

### 2.2 分组逻辑详解

**第一步：样本筛选**

每日的样本债券需要满足以下条件：
- 债券类型：信用债或金融债
- 有有效的中债估值收益率
- 有明确的剩余期限和余额数据

**第二步：按收益率排序**

将当日所有样本债券按收益率从小到大排序。收益率越低，市场认为风险越小；收益率越高，市场认为风险越大。

**第三步：均匀分组**

将排序后的债券均匀划分为10组，每组包含约10%的债券。具体的边界处理采用标准分位数计算方法，确保每组的债券数量大致相等。

**第四步：计算组内指数**

对每个分位数组，计算该组的余额加权价格指数。加权公式为：

```
Index = Sum(Price_i * Balance_i) / Sum(Balance_i)
```

这与中债指数的市值加权（派许加权）规则一致，确保指数能够反映实际资金配置。

### 2.3 与信用评级分组的对比

| 维度 | 信用评级分组 | 收益率分位数分组 |
|------|-------------|-----------------|
| 分层依据 | 外部评级机构判断 | 市场参与者集体定价 |
| 更新频率 | 评级调整时（滞后） | 每日动态更新 |
| 颗粒度 | 固定几档（AAA/AA+/AA等） | 10档，每档约10%样本 |
| 客观性 | 含主观判断 | 完全数据驱动 |
| 适用范围 | 有评级的债券 | 所有有估值的债券 |

### 2.4 40指数矩阵设计

为了支持多维度分析，本项目构建了40个细分指数：

```
40 = 2（债券类型） x 10（分位数组） x 2（价格类型）
```

**债券类型维度：**
- 信用债（非金融企业发行的债券）
- 金融债（金融机构发行的债券）

**分位数组维度：**
- D1-D10，分别对应0-10%、10-20%、...、90-100%收益率分位数

**价格类型维度：**
- 净价（Clean Price）：不含应计利息的价格
- 全价（Dirty Price）：包含应计利息的价格

这个矩阵设计提供了丰富的分析视角：

- **纵向比较**：同一分位数组在不同时间的表现，反映该风险层的收益特征
- **横向比较**：同一时间不同分位数组的表现差异，反映信用利差结构
- **类型比较**：信用债与金融债的分层特征对比
- **价格比较**：净价指数与全价指数的差额，反映票息收入的贡献

## 三、技术实现：工程化设计确保可复现性

### 3.1 双数据库架构

本项目采用双数据库架构：

- **PostgreSQL**：存储收益率曲线数据，主要来自中债估值
- **MySQL**：存储行情数据，包括价格、余额等

这种设计有几个考量：

**第一，数据源解耦。** 收益率曲线和行情数据可能来自不同的数据供应商或采集系统，分离存储便于独立管理和扩展。

**第二，性能优化。** 收益率曲线数据量大但访问模式固定，PostgreSQL的列式存储特性有利于这类查询。行情数据更新频繁，MySQL的事务处理能力更适合。

**第三，历史兼容。** 很多机构已有成熟的数据库架构，双数据库设计便于与现有系统集成。

### 3.2 分层缓存策略

债券指数计算涉及大量历史数据和复杂计算。为了提升效率，本项目设计了多层缓存策略：

**第一层：基础数据缓存**
- 债券分类信息（类型、行业等）
- 变化频率低，可以长期缓存
- 格式：Parquet

**第二层：时间段数据缓存**
- 收益率数据、价格数据
- 按日期范围缓存，支持增量更新
- 格式：Parquet

**第三层：中间结果缓存**
- 分组数据（每日每只债券所属的分位数组）
- 按日缓存，便于回查和验证
- 格式：Parquet

**第四层：最终结果缓存**
- 指数数据（40个指数的每日值）
- 按日缓存，支持快速查询
- 格式：Parquet

选择Parquet格式的原因：
- **高压缩比**：对于结构化数据，压缩效果显著
- **列式读取**：只读取需要的列，减少IO
- **跨平台兼容**：Python、R、Spark等工具都支持

### 3.3 模块化代码设计

良好的模块化设计是研究可复现性的基础。本项目的主要模块包括：

**config.py - 集中配置管理**
- 数据库连接参数
- 指数计算参数（分位数数量、价格类型等）
- 缓存路径配置

**DatabaseManager类 - 数据库连接封装**
- 统一的连接池管理
- 自动重连机制
- 查询结果标准化

**核心计算函数**
- `calculate_daily_groups()`：计算每日分组
- `calculate_indices()`：计算指数值
- `update_daily_indices()`：每日增量更新
- `backfill_historical_indices()`：历史数据回填

这种设计的优势在于：
- 参数集中管理，避免硬编码
- 函数职责单一，便于测试和维护
- 与AI协作时，清晰的代码结构有助于AI理解和修改

## 四、应用场景与实际价值

### 4.1 信用利差研究

通过比较高分位数组（高收益率）和低分位数组（低收益率）的收益差异，可以分析信用风险溢价的变化。

**典型分析思路：**

假设我们关注D1组（最低收益率，"最安全"债券）和D10组（最高收益率，"最风险"债券）的价差：

```
Credit Spread = D10 Index Return - D1 Index Return
```

这个价差反映了市场对"风险债券"相对于"安全债券"要求的额外补偿。当价差走阔时，说明市场风险厌恶情绪上升；当价差收窄时，说明市场风险偏好提升。

**与传统的信用利差指标对比：**

传统的信用利差通常用"某评级债券收益率 - 同期限国债收益率"计算。这种方法的问题在于：
- 评级内的债券收益率分布很广，平均值可能失真
- 需要构建期限匹配的国债曲线，技术复杂

收益率分位数方法直接比较指数收益，避免了这些问题。

### 4.2 市场结构监控

跟踪各分位数组指数的相对表现，可以监测市场风险偏好的变化。

**监测维度：**

**分位数组间的相对表现**
- 如果D1-D3组表现强势，说明市场追求安全资产
- 如果D8-D10组表现强势，说明市场追逐收益

**信用债与金融债的对比**
- 金融债通常风险低于信用债，但如果金融债高分组表现异常，可能预示金融机构风险

**指数波动率的变化**
- 高分位数组的波动率上升，可能预示信用风险事件增加

### 4.3 投资策略开发

指数体系为策略开发提供了基础数据支持：

**轮动策略**
- 根据市场环境，在不同分位数组之间轮动配置
- 例如：风险偏好上升时超配高分组，风险偏好下降时超配低分组

**相对价值策略**
- 做多某一分位数组，做空另一分位数组
- 例如：做多D5组、做空D10组，博取风险利差收敛

**指数复制**
- 利用指数成分数据，构建跟踪组合
- 适用于指数型产品或被动投资策略

### 4.4 每日市场报告

指数体系可以自动化生成信用债市场日报，包括：

- 各分位数组的日度收益表现
- 信用债与金融债的相对表现
- 与前一日的变动和原因分析
- 关键图表和可视化展示

## 五、AI辅助研究心得要点

### 5.1 市场定价优于主观分类

**背景：** 传统信用债分析依赖外部信用评级，但评级调整滞后于市场变化。同一评级内的债券风险特征可能差异巨大。

**实践：** 本项目创新性地使用收益率分位数分组。收益率是市场参与者用真金白银投票形成的价格，综合反映了信用风险、流动性溢价、期限结构等多重因素。每日动态分组确保指数始终反映最新市场结构。

**心得：** 市场定价（收益率）是一种实时、客观的风险分类标准。在AI辅助研究中，应优先使用市场内生变量而非外部标签作为分类依据。这种方法论可推广至其他资产类别的研究中。例如，股票研究中可以用市盈率分位数而非行业分类；利率债研究可以用久期分位数而非剩余期限区间。

### 5.2 工程化思维保障研究质量

**背景：** 金融研究常面临可复现性问题：分析脚本散落各处、数据版本不一致、参数硬编码在代码中。这些问题不仅影响研究的可信度，也阻碍了后续的迭代和扩展。

**实践：** 本项目采用模块化工程化设计：

1. **config.py集中管理所有参数和配置**，包括数据库连接、指数计算参数、缓存路径等。任何参数调整只需修改一处。

2. **DatabaseManager类封装数据库连接**，统一连接池管理，自动处理重连和异常。

3. **分层缓存机制**，债券分类、收益率数据、价格数据、分组数据、指数数据各自缓存，避免重复计算。

4. **每日更新和历史回填函数独立封装**，支持增量更新和全量重构两种模式。

**心得：** 工程化思维不仅是技术要求，更是研究质量的保障。良好的模块化设计使研究过程可追溯、可复现、可扩展。在AI辅助研究中，清晰的代码结构便于与AI协作迭代——AI可以更快理解代码逻辑，减少错误修改的风险。建议在项目初期就建立清晰的目录结构和代码规范，而不是等到后期再重构。

### 5.3 分层缓存是效率优化的关键

**背景：** 债券指数计算涉及大量历史数据和复杂计算，每次全量计算耗时较长。且数据库查询和数据处理是主要瓶颈。

**实践：** 设计多层缓存策略：

1. **基础数据缓存（债券分类）** - 变化频率低，可以长期缓存
2. **时间段数据缓存（收益率、价格数据）** - 按日期范围缓存，支持增量更新
3. **中间结果缓存（分组数据）** - 按日缓存，便于回查和验证
4. **最终结果缓存（指数数据）** - 按日缓存，支持快速查询

使用Parquet格式保证读写效率和压缩比。对于典型的信用债指数计算，分层缓存可以将单日更新时间从分钟级降低到秒级。

**心得：** 对于复杂计算任务，分层缓存是提升效率的关键策略。在AI辅助研究中，合理的缓存设计可以减少重复计算，加速迭代验证过程。这一原则可推广至其他需要大量数据处理的研究项目。设计缓存时需要考虑：数据的更新频率、缓存失效条件、存储空间与计算时间的权衡。

### 5.4 数据验证是研究可信度的基石

**背景：** 金融数据来源多样，数据质量问题不可避免。中债估值偶尔会有缺失或异常值，数据库同步可能有延迟。

**实践：** 本项目在多个环节嵌入数据验证：

1. **样本筛选阶段**：过滤掉没有有效估值的债券
2. **分组计算阶段**：检查每组的债券数量是否合理
3. **指数计算阶段**：对比净价指数和全价指数的差额是否合理
4. **结果输出阶段**：与前一日的指数值进行对比，检测异常波动

**心得：** 数据验证不应该只是事后的检查，而应该融入整个数据处理流程。在AI辅助研究中，AI生成的代码往往缺少充分的边界条件处理，需要人工补充数据验证逻辑。建议在项目初期就列出可能的数据问题，并设计相应的处理策略。

## 六、局限性与改进方向

### 6.1 当前局限

**第一，分组的经济含义需要解读。** 收益率分位数分组是纯统计分组，不直接对应具体的信用风险水平。D10组的债券收益率高，可能是因为信用风险高，也可能是因为流动性差，或者因为期限长。解读时需要结合其他指标。

**第二，成分不固定导致跟踪困难。** 每日重新分组意味着指数成分每天变化，这与传统指数的成分稳定性不同。如果要用这些指数做业绩基准或产品跟踪，需要特别注意这一特性。

**第三，品种覆盖有限。** 当前仅覆盖信用债和金融债，未包含城投债、可转债等特殊品种。这些品种在我国债券市场占比较大，缺少它们的分析不够完整。

**第四，余额加权可能导致大券主导。** 余额加权方法下，大余额债券对指数的影响更大，小券的影响被稀释。如果大券的风险特征与同组小券差异较大，指数可能无法准确反映该风险层的整体情况。

**第五，依赖特定数据库架构。** 项目使用的双数据库架构和特定的数据表结构，在其他机构复用需要适配数据源。

### 6.2 改进方向

**引入更多分组维度**
- 在收益率分位数的基础上，叠加行业、区域、期限等维度
- 构建更细粒度的指数矩阵

**增加城投债和可转债**
- 城投债是信用债的重要子类，有独特的风险特征
- 可转债兼具股性和债性，需要单独的分析框架

**开发等权指数**
- 作为余额加权指数的补充，避免大券主导问题
- 两种加权方法对比分析，提供更全面的视角

**构建归因分析框架**
- 将指数收益分解为票息收益、资本利得、利差变化等因子
- 提供更深入的市场分析视角

## 七、延伸思考

### 7.1 收益率分位数方法的普适性

收益率分位数分层法的核心思想——用市场定价结果作为风险分层依据——具有普适性，可以推广到其他资产类别：

**利率债研究**
- 按收益率分位数分组，分析期限溢价的时变特征
- 结合久期维度，构建利率风险分层指数

**股票研究**
- 按市盈率分位数分组，分析价值因子的表现
- 按市值分位数分组，分析规模因子的表现

**外汇研究**
- 按远期升贴水分位数分组，分析汇率预期差异

这种方法论的核心价值在于：使用市场内生变量而非外部标签，确保分类标准的客观性和实时性。

### 7.2 AI辅助研究的工作流优化

本项目的开发过程体现了AI辅助研究的典型工作流：

**第一阶段：需求澄清**
- 与AI对话，明确研究问题和约束条件
- AI帮助梳理思路，提出可能的方案

**第二阶段：方案设计**
- AI生成初步的代码框架
- 人工审核和调整，确保符合业务逻辑

**第三阶段：迭代优化**
- 运行代码，发现问题
- 与AI讨论问题原因，获得修改建议
- 反复迭代，直到达到预期效果

**第四阶段：文档整理**
- AI帮助生成代码注释和文档
- 人工审核和补充，确保准确性

**关键经验：**
- 与AI协作时，清晰的代码结构至关重要
- 工程化设计（配置管理、模块化、缓存）便于AI理解和修改
- 数据验证逻辑需要人工仔细审查
- AI生成的内容需要结合专业知识进行判断

### 7.3 固收研究的数字化转型

弘则信用债指数项目是固收研究数字化转型的一个缩影。传统的固收研究高度依赖人工，分析师需要手动处理数据、编制表格、撰写报告。数字化转型带来几个变化：

**数据处理的自动化**
- 日常数据更新、清洗、计算由程序自动完成
- 分析师从繁琐的数据工作中解放出来

**研究视角的丰富化**
- 高频数据（日度甚至日内）成为可能
- 多维度交叉分析成为常态
- 历史回测和情景分析更加便捷

**研究产出的标准化**
- 指数、因子、信号的编制有统一的规范
- 研究成果便于复用和迭代

**AI辅助的常态化**
- AI成为研究的助手，参与代码编写、数据分析、文档撰写
- 人机协作成为新的工作模式

这一转型才刚刚开始，未来还有很大的发展空间。

## 八、结论

弘则信用债指数项目创新性地提出"收益率分位数分层法"，以市场定价结果为依据构建了40个细分指数。这种方法相比传统信用评级分层更加客观、实时，能够更好地反映市场风险偏好的变化。

项目的工程化设计——双数据库架构、分层缓存策略、模块化代码结构——确保了研究的可复现性和可扩展性。这些经验可以推广到其他研究项目中。

在AI辅助研究的实践中，我们总结了几个关键心得：优先使用市场内生变量而非外部标签；工程化思维是研究质量的保障；分层缓存是效率优化的关键；数据验证需要融入整个流程。

信用债市场正在经历深刻变化，理财真净值化时代对研究工具提出了更高要求。收益率分位数指数体系为理解市场结构、监测风险变化、开发投资策略提供了新的视角。期待这一方法能够不断完善，为固收研究贡献更大的价值。
