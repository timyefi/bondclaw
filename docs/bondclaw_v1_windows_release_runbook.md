# BondClaw V1 Windows 发布 Runbook

本文档是 Windows 发布的现场执行手册。  
它比 P0 步骤更接近实际操作，强调“怎么做、做完看什么、异常怎么回退”。

## 发布目标

- 生成可分发的 Windows 安装包。
- 确认自动升级链路可以稳定刷新。
- 确认发布产物、品牌、版本和清单一致。
- 保证出问题时能快速回退到上一个可用版本。

## 发布前检查

在开始打包前，至少确认以下事项：

- `bondclaw_v1_smoke_test.py` 通过。
- `bondclaw_v1_contact_delivery_spec.md` 已定版。
- `bondclaw_v1_windows_native_validation_plan.md` 已准备好。
- `bondclaw_v1_windows_native_installation_runbook.md` 已准备好。
- BondClaw 首页可正常访问。
- 品牌配置和升级配置都已经统一。

## 1. 构建

### 1.1 构建前确认

- 当前版本号已确定。
- release channel 已确定。
- 更新仓库和 manifest URL 已确定。

### 1.2 执行构建

- 生成 Windows 安装包。
- 生成发布清单。
- 如需要，生成签名文件或校验文件。

### 1.3 构建后检查

- 安装包文件存在。
- 文件名和版本号一致。
- 清单和安装包指向同一版本。

## 2. 分发

### 2.1 发布前确认

- 支持页可访问。
- 官网可访问。
- 文档页可访问。
- 更新说明可访问。

### 2.2 上传与发布

- 将安装包上传到发布仓库或分发位置。
- 将 manifest 更新到对应清单位置。
- 确认清单地址可以被客户端访问。

### 2.3 分发后检查

- 访问发布页时能看到新版本。
- 清单中的版本号和渠道正确。
- 更新仓库和 manifest URL 没有写错。

## 3. 签名与校验

### 3.1 签名目标

- 如果发行渠道要求签名，优先在构建后完成。
- 签名材料应和安装包版本严格对应。

### 3.2 校验目标

- 安装包校验值可复核。
- 清单文件可复核。
- 发布版本与签名材料一一对应。

## 4. 回滚

### 4.1 回滚触发条件

- 安装失败。
- 升级失败。
- 版本号或 manifest 出错。
- 运行态或页面状态异常。

### 4.2 回滚动作

- 退回到上一个稳定安装包。
- 恢复上一个稳定清单。
- 保持品牌与内容配置不变。

### 4.3 回滚检查

- 回滚后能正常启动。
- 回滚后可访问首页。
- 回滚后可继续走 smoke test。

## 5. 发布验收

### 5.1 安装验收

- 干净 Windows 机器可安装。
- 桌面快捷方式可用。
- 首次启动能进入 `/bondclaw`。

### 5.2 升级验收

- 客户端可刷新到新 manifest。
- 升级前后版本号正确。
- 升级后模板中心、信息中心、联系方式、设置页仍可用。

### 5.3 交付验收

- 发布包、清单、官网、支持页和文档页一致。
- 版本号、渠道、发布日期记录完整。
- 可回溯到对应构建和签名材料。

## 6. 发布记录建议

每次发布建议至少记录以下字段：

- 版本号
- 渠道
- 更新仓库
- manifest URL
- 安装包文件名
- 签名或校验摘要
- 发布时间
- 回滚版本

## 7. 建议执行顺序

1. 先做构建前确认。
2. 再做安装包构建。
3. 然后上传并更新清单。
4. 再做签名与校验。
5. 最后做安装、升级和回滚验收。

## 8. 通过标准

以下条件全部满足时，认为 Windows 发布可放行：

- 安装包存在且版本正确。
- 清单可被客户端刷新。
- 升级不破坏首页状态。
- 回滚路径可用。
- 发布记录完整可查。
