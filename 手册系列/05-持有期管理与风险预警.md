# 第 5 册：持有期管理与风险预警

## 1. 持有期管理为什么重要

信用研究最容易被忽略的一点，是买入后的持续跟踪。很多判断在买入时是成立的，但随着主体经营变化、融资环境变化、政策变化和市场定价变化，原先成立的判断可能失效。

所以持有期管理是信用研究的闭环部分。没有它，前面的评级和决策就只是一次时点结论。

## 2. 持有期要跟踪什么

### 2.1 主体信息变化

包括经营、财务、融资、资产处置、担保安排、重大事项和公告信息。

### 2.2 市场信息变化

包括收益率变化、成交变化、流动性变化、信用利差变化和同类主体比较。

### 2.3 风险信号变化

包括评级调整、舆情变化、再融资压力、条款触发以及持续恶化的经营信号。

## 3. 风险预警不是“看见问题再说”

风险预警的目标不是事后解释，而是尽可能提前发现异常。它的工作方式通常分成三步：

1. 建立常态基线。
2. 观察偏离程度。
3. 判断偏离是短期波动还是结构性变化。

只有第三步做对了，预警才真正有意义。

## 4. 为什么持有期管理是循环过程

持有期管理不是“每周看看就行”，而是一个持续循环：

信息更新 -> 风险重估 -> 持仓判断 -> 执行动作 -> 再更新。

这个循环的存在，决定了信用研究必须有一套能重复运行、能保留历史记录、能回看证据的工作流。

## 5. 什么时候应该触发再决策

常见触发条件包括：

- 财务表现明显弱于预期。
- 融资环境发生变化。
- 条款触发或接近触发。
- 市场定价与原始判断明显偏离。
- 新信息足以改变原有判断。

触发再决策不是“更谨慎”，而是信用研究本身的职责。

## 6. 这一册给技术人员的启发

从系统实现角度看，持有期管理意味着研究系统不能只支持一次性输出，还必须支持持续更新、差异比较和历史追踪。也就是说，工作流设计要考虑“时间维度”，而不是只考虑“单次生成”。
