# BondClaw V1 发布前收口清单

本文档只列 V1 进入正式交付前还需要完成的事项，并按优先级排序。当前前置能力已经基本成型，剩余工作重点是发布、验证和最终收口。

## 当前判断

- 桌面壳、模板中心、信息中心、联系方式、设置页已经可用。
- 品牌、升级、运行态和快照数据链路已经打通。
- 内容层已经达到可试用规模，但仍需要最终审校。
- 现在最缺的是正式发布链路和 Windows 端最终验证。

## P0: 发布与安装

1. Windows 安装包
   - 生成可安装的 Windows 版本。
   - 明确安装器、桌面快捷方式、卸载流程。
   - 验证首次启动和升级后的可用性。
2. 自动升级最终确认
   - 校验 release manifest、版本号、渠道和更新仓库。
   - 验证应用内刷新后能稳定拿到远程清单。
   - 确认升级不影响当前首页状态。
3. 发布产物与签名
   - 统一构建产物命名。
   - 确认发布仓库、清单地址、支持页和官网一致。
   - 需要时补签名与校验流程。

## P1: Windows 原生收口

1. Windows 运行验证
   - 在真实 Windows 环境验证首页。
   - 确认执行路径全部走 Windows 原生进程。
   - 验证网络、文件路径和外链打开行为。
2. 命令抽象回归
   - 确认统一执行抽象在全部关键动作上生效。
   - 验证后续新增能力不会绕开抽象层。

## P1: 线索与投递

1. 联系方式投递链路定版
   - 确认邮箱、存储、归档三路投递方案。
   - 确认回执记录与本地队列格式。
   - 确认断网重试和幂等策略。
2. 后台最小查询页
   - 至少能查看提交记录、状态和重试结果。
   - 不引入重型 CRM 服务。

## P1: 内容与体验收口

1. 模板中心审校
   - 复核 7 个角色、20 个 starter 的命名、输出格式和 QA 清单。
   - 确认默认角色、默认模板和默认供应商是否符合实际使用习惯。
2. 信息中心审校
   - 复核 20 个案例的主题标签、角色标签和推荐 Prompt。
   - 确认来源卡、案例高亮和当前案例之间的映射关系。
3. 设置页最终整理
   - 确认品牌、升级、在线入口、说明卡的口径统一。
   - 如果需要，再做一次视觉层的收口。

## P2: 回归与文档

1. Smoke test
   - 启动页
   - 模板中心切换
   - 信息中心筛选
   - 联系方式队列
   - 设置页刷新
2. 文档补齐
   - 快速开始
   - 常见故障
   - 发布说明
   - 内部维护说明

## 建议推进顺序

1. 先做 Windows 安装包和升级确认。
2. 再做 Windows 原生的真实环境验证。
3. 然后定版联系方式投递链路。
4. 最后做模板中心、信息中心和设置页的人工审校。

## 结论

BondClaw V1 已经不是“从零搭建”，而是进入“收尾、发布、验证”的阶段。后续工作应该优先围绕可交付性，而不是再大幅扩新骨架。
