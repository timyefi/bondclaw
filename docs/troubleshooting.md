# 常见问题

这份文档是给技术同学的排障入口，目标是让第一次接触项目的人能自己定位问题，而不是卡在环境和路径上。

## 常见问题

### 1. 解析或分析跑不起来

- 先确认输入文件类型和路径是否正确。
- 再确认依赖和配置是否可用。
- 如果是批量任务，先用单文件验证。

### 2. 结果不完整

- 先看原始材料是否清晰。
- 再看附注范围是否定位正确。
- 如果正文和附注混在一起，先做标准化，不要直接进分析。

### 3. 结果只是草稿

- 这是预期行为。
- 草稿产物是中间层，不是最终交付。
- 正式报告必须在复核后显式生成。

### 4. 路径和编码出错

- 中文路径要统一用 UTF-8 处理。
- 绝对路径优先。
- Windows 路径要优先按本地路径处理，不要默认转成 POSIX 口径。
- 不要把运行态数据写到核心技能目录。

## 排障顺序

1. 看输入是否正确。
2. 看环境是否满足。
3. 看中间产物是否生成。
4. 看正式产物是否缺失。
