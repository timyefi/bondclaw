# 第5篇 地产去库存预测：房地产市场供需平衡的量化分析

## 摘要

房地产市场的库存水平是影响房价走势和政策预期的重要因素。本项目构建了住宅库存预测框架，基于EDB数据库中的房地产指标，采用双路径预测方法（趋势推演 + 销售-竣工去化），对未来库存水平进行预测。通过期房/现房比例修正，更准确地估算真实库存压力，为房地产相关债券投资提供重要的基本面参考。

**关键词**：房地产库存、去化预测、库销比、线性回归、供需平衡

---

## 一、业务背景与需求分析

### 1.1 研究痛点

房地产库存预测面临的主要挑战：

1. **官方数据局限**：待售面积仅包含现房，不含期房
2. **预测方法单一**：传统预测依赖经验判断，缺乏量化模型
3. **供需动态复杂**：库存变化受销售、竣工、新开工多因素影响

### 1.2 业务价值

住宅库存水平对投资的影响：
- 库存高企 → 房企资金压力大 → 房企债风险上升
- 库销比过高 → 去化周期长 → 房价下行压力
- 政策预期 → 去库存政策 → 地产债机会

## 三、核心算法与实现

### 3.1 累计值到月度值转换

```python
def calculate_monthly_values(df):
    """计算月度值和累计值"""
    df_monthly = pd.DataFrame(index=df.index)

    for col in ['S0049264', 'S0049296', 'S0049585', 'S0029669']:
        df_reset = df[col].copy()

        # 检测年初重置点（当前值小于前一个值时）
        diff_values = df_reset.diff()
        reset_points = diff_values < 0

        # 计算当月值
        monthly_values = diff_values.fillna(df_reset)
        # 在重置点用原值作为当月值
        monthly_values[reset_points] = df_reset[reset_points]

        df_monthly[f'{col}_monthly'] = monthly_values

    # 计算历史累计值
    df_monthly['S0049264_hist_cum'] = df_monthly['S0049264_monthly'].cumsum()
    df_monthly['S0049296_hist_cum'] = df_monthly['S0049296_monthly'].cumsum()

    return df_monthly
```

### 3.2 住宅库存估算

```python
def calculate_inventory_estimate(df, df_monthly):
    """计算住宅库存估算值"""
    # 住宅库存估算值 = S0029674 * (期房历史累计 + 现房历史累计) / 期房历史累计
    inventory_estimate = df['S0029674'] * \
        (df_monthly['S0049296_hist_cum'] + df_monthly['S0049264_hist_cum']) / \
        df_monthly['S0049296_hist_cum']

    return inventory_estimate
```

### 3.3 趋势线性回归预测

```python
def trend_forecast(data, forecast_start_date, forecast_end_date):
    """趋势线性回归预测"""
    # 准备训练数据
    train_data = data[data.index >= pd.to_datetime(forecast_start_date)].dropna()

    # 创建时间变量
    train_data = train_data.copy()
    train_data['time'] = range(len(train_data))

    # 创建预测日期范围
    last_date = data.index.max()
    future_dates = pd.date_range(
        start=last_date + pd.DateOffset(months=1),
        end=forecast_end_date,
        freq='M'
    )

    # 线性回归模型
    model = LinearRegression()
    X_train = train_data[['time']]
    y_train = train_data.iloc[:, 0]

    model.fit(X_train, y_train)

    # 预测
    future_time = range(len(train_data), len(train_data) + len(future_dates))
    future_values = model.predict(np.array(future_time).reshape(-1, 1))

    forecast_df = pd.DataFrame(
        index=future_dates,
        data=future_values,
        columns=[data.columns[0]]
    )

    return forecast_df
```

### 3.4 销售-竣工去化预测

```python
def sales_completion_destocking_forecast(inventory_estimate, rolling_data,
                                          forecast_start_date, forecast_end_date):
    """销售-竣工去化预测"""
    # 获取销售和竣工的趋势预测
    s3_forecast = trend_forecast(rolling_data[['s3']], forecast_start_date, forecast_end_date)
    completion_forecast = trend_forecast(
        rolling_data[['completion_12m']], forecast_start_date, forecast_end_date
    )

    # 计算月度销售和竣工
    monthly_sales = s3_forecast['s3'] / 12
    monthly_completion = completion_forecast['completion_12m'] / 12

    # 计算月度去化值（销售 - 竣工）
    monthly_destocking = monthly_sales - monthly_completion

    # 从最后实际库存开始，逐月计算
    last_inventory = inventory_estimate.dropna().iloc[-1]
    inventory_forecast = []

    for destocking in monthly_destocking:
        if len(inventory_forecast) == 0:
            new_inventory = last_inventory - destocking
        else:
            new_inventory = inventory_forecast[-1] - destocking
        inventory_forecast.append(new_inventory)

    return pd.Series(inventory_forecast, index=monthly_destocking.index)
```

## 四、心得要点

### 心得一：双路径预测方法的互补性设计

**问题背景**：单一预测方法存在偏差风险，难以应对市场环境变化。

**解决方案**：设计双路径预测方法：
- **趋势推演法**：假设历史趋势延续，使用线性回归外推
- **销售-竣工去化法**：基于供需平衡假设，库存变化 = 销售 - 竣工

两种方法的预测结果可以互为验证：
- 当两种方法结果接近时，预测可信度高
- 当结果差异大时，提示市场可能存在结构性变化

**业务价值**：为投资决策提供更全面的情景分析，降低单一预测方法的偏差风险，支持更稳健的投资判断。

### 心得二：期房/现房比例修正库存估算

**问题背景**：统计局公布的"商品房待售面积"仅包含现房，不包含已售未交付的期房，低估了真实库存压力。

**解决方案**：设计库存估算公式：
```
住宅库存估算值 = 待售面积 × (期房累计销售 + 现房累计销售) / 期房累计销售
```

核心逻辑：期房销售代表未来供应压力，通过期房/现房比例推算包含期房的真实库存水平。

**业务价值**：更准确地反映市场真实库存压力。例如，某时期待售面积10亿平米，期房/现房比例为4:1，则真实库存估算约12.5亿平米，比官方数据高25%。

### 心得三：累计值到月度值的智能转换

**问题背景**：统计局公布的房地产数据为累计值，1-2月合并公布，年初数据需要特殊处理。

**解决方案**：设计智能转换算法：
1. 计算月度差分值
2. 检测年初重置点（当前值 < 前值）
3. 在重置点使用原值作为当月值
