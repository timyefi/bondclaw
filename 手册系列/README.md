# 信用研究手册系列

这是一套面向技术人员的信用研究工作流手册系列，用于把“债券版 openclaw”相关认知拆成多册连续交付的文档。设计原则是：每一册都能单独发群阅读，但彼此之间又形成递进关系，避免单册过长、主题过散或内容重复。

## 系列定位

- 第 1 册负责建立总心智模型，只讲信用研究工作流是什么、为什么需要、关键环节如何串联。
- 第 2-6 册分别展开数据、证据、评级、决策、持有期管理、AI 介入边界，但都不落到单个项目实现。
- 第 7 册作为能力模块地图，只讲“可复用能力”的组合方式，为后续项目级手册留接口。

## 建议发布节奏

- 适合每 1-2 天发一册，形成稳定的连续输出节奏。
- 第 1 册可以作为群内总开篇，第 2-4 册作为核心方法论，第 5-7 册作为延伸和能力地图。
- 如果后续要继续扩展到单项目手册，可以在第 7 册之后自然接项目级文档。

## 写作规则

- 不把单个项目的实现细节写进总纲册。
- 不为了凑字数重复同义表达，优先增加定义、边界、流程、校验和常见误区。
- 所有事实性内容都预留来源位，后续再补双来源校验。
- 每一册都要保留独立标题、目录、结论和适用范围，确保可单独阅读。
