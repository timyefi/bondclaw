# BondClaw V1 Windows 原生架构决策

BondClaw V1 的对外交付形态已经明确为 **Windows 原生桌面应用**。

## 决策结论

- 客户安装后不需要额外安装 Linux 发行版或终端环境。
- 客户不需要单独安装 Linux 发行版。
- 客户不需要理解脚本、命令行或终端切换。
- 所有可见流程以 Windows 桌面体验为准。
- 内部如果保留兼容执行路径，也只能作为开发维护选项，不能成为客户前提。

## 保留不变的部分

- BondClaw 首页
- 模板中心
- 信息中心
- 联系方式
- 设置
- 品牌集中配置
- 在线升级与发布清单
- 本地队列、缓存和审校流程

## 迁移原则

1. 页面结构保留。
2. 品牌命名保留。
3. 执行层改成 Windows 原生进程或 Windows 可用方案。
4. 文档、安装器、验证脚本和示例配置统一收口到 Windows 口径。
5. 如果某项能力暂时不能完全原生化，就先标记为内部或后置，不写成客户必需步骤。

## 当前执行标准

- 默认执行环境是 Windows 原生。
- 默认路径处理是 Windows 本地路径。
- 默认打开方式是 Windows 桌面程序。
- 默认验证环境是干净 Windows 机器。

## 风险提示

- 仍需确认所有桥接动作都保持 Windows 原生执行口径。
- 仍需确认第三方工具的调用都走 Windows 可用路径。
- 仍需继续清理文档和示例中的旧口径，避免客户误读。
