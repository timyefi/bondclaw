# BondClaw V1 Windows 原生验证计划

这份计划只围绕 Windows 原生环境。

## 验证范围

- 安装
- 首次启动
- 首页
- 模板中心
- 信息中心
- 联系方式
- 设置
- 升级
- 回退
- 路径与缓存

## 前置条件

- 一台干净的 Windows 机器或虚拟机。
- 一个可用的安装包。
- 一个可访问的发布清单。
- 一份可复核的品牌配置。

## 验证步骤

1. 安装 BondClaw。
2. 启动 BondClaw。
3. 确认能进入 BondClaw 首页。
4. 确认模板中心可浏览。
5. 确认信息中心可浏览。
6. 确认联系方式页能看到队列状态。
7. 确认设置页能看到执行模式和默认配置。
8. 刷新运行态并确认配置没有异常。
9. 执行升级并确认首页状态保留。
10. 回退到上一个版本并确认可再次启动。

## 通过标准

- 安装过程不依赖额外环境。
- 页面可以正常打开。
- 执行结果保持 Windows 原生执行口径。
- 升级和回退都不破坏首页状态。
- 验证记录可以复核。

## 失败处理

- 安装失败，先看安装包与权限。
- 启动失败，先看日志与路径。
- 页面异常，先看运行态和本地缓存。
- 升级异常，先回退到上一个稳定版本。

## 记录建议

- Windows 版本
- BondClaw 版本
- 安装包版本
- 升级前后版本
- 是否成功进入首页
- 是否成功打开各页面
- 是否完成回退
