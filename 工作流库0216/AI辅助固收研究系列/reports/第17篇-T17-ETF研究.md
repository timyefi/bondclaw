# 第17篇 ETF研究框架：债券ETF的分类、模拟与对冲组合分析

## 摘要

本项目构建了一套完整的ETF研究框架，聚焦于债券ETF的专项分析能力。核心创新包括：(1) 规则+机器学习混合分类策略，实现高准确率与高灵活性的ETF自动分类；(2) 基于债券曲线数据的ETF绩效模拟，支持久期策略回测；(3) 3+3股债对冲组合框架，简化资产配置研究流程。2025年国内债券ETF市场规模增长376.52%，达8290亿元，本项目的研究框架具有重要的实践价值。

---

## 1. 项目背景与目标

### 1.1 市场背景

国内债券ETF市场近年来呈现爆发式增长：
- **规模增长**：2025年债券ETF市场规模达8290亿元，同比增长376.52%
- **产品丰富**：首批基准做市信用债ETF、科创债ETF相继成立
- **需求上升**：机构投资者对久期匹配和精细化管理需求持续增加

### 1.2 研究目标

本项目旨在构建一套适合固收研究场景的ETF分析框架，核心目标包括：
- 建立支持债券ETF细粒度分类的分类体系
- 实现基于债券曲线数据的ETF绩效模拟功能
- 提供简化版的股债对冲组合分析工具

### 1.3 技术架构

采用**模块化Notebook架构**，核心组件包括：

| 模块 | 职责 | 特点 |
|------|------|------|
| config.py | 配置集中管理 | dataclass封装、环境变量加载、连接池配置 |
| utils.py | 工具函数库 | DataLoader、PerformanceAnalyzer、ETFClassifier、Visualizer |
| Notebooks | 章节式研究 | 7个专题章节，循序渐进 |

## 2. ETF分类体系

### 2.1 分类维度设计

针对债券ETF的特殊需求，设计了多维度的分类体系：

| 维度 | 类别 |
|------|------|
| 市场方向 | 股票、债券、商品、QDII、货币 |
| 策略类型 | 被动指数、主动管理、Smart Beta |
| 久期分类 | 超短期(<1年)、短期(1-3年)、中期(3-5年)、中长期(5-7年)、长期(>7年) |
| 信用分类 | 利率债、信用债、城投债、可转债 |

### 2.2 混合分类策略

采用**规则优先+机器学习兜底**的混合分类架构：

#### 规则分类器

基于预定义模式进行精确匹配：
- 股票ETF：沪深300、上证50、中证500、创业板、科创板
- 债券ETF：国债、城投、信用债、可转债
- 商品ETF：黄金
- QDII：纳斯达克、标普、恒生

**优势**：分类速度快、准确率高、可解释性强

#### 机器学习分类器

对于规则无法覆盖的边缘情况，使用TF-IDF + 朴素贝叶斯分类：
- 特征提取：ETF名称、全名、基准指数、投资类型
- 文本向量化：TF-IDF提取关键词特征
- 分类模型：朴素贝叶斯进行多分类

**优势**：灵活处理新模式、支持增量学习

### 2.3 反馈机制设计

建立分类错误的反馈收集机制：
- 收集人工修正的分类结果
- 用于机器学习模型的增量训练
- 支持规则库的持续扩展

## 3. 债券ETF专项分析

### 3.1 债券类型覆盖

支持7大类债券曲线数据分析：

| 代码 | 债券类型 |
|------|----------|
| CD | 同业存单 |
| Financial | 金融债 |
| Tier2 | 二级资本债 |
| Municipal | 城投债 |
| Corporate | 企业债 |
| ABS | 资产支持证券 |
| Treasury | 国债 |

### 3.2 久期分类标准

覆盖全期限谱系的11个久期节点：
- 短端：0.5年、1年、2年
- 中端：3年、4年、5年
- 长端：7年、10年、15年、20年、30年

### 3.3 ETF绩效模拟方法

创新性地实现了基于债券曲线数据的ETF绩效模拟：

**模拟原理**：
1. 从债券曲线获取各久期的日度收益率数据
2. 利用"收益率下降=价格上涨"的负相关关系计算日收益
3. 通过日收益累乘构建净值序列
4. 计算年化收益、波动率、最大回撤、夏普比率

**应用场景**：
- 久期策略回测：比较不同久期配置的历史表现
- 信用利差分析：同久期不同信用等级的收益差异
- 曲线陡峭度分析：期限利差变化的影响

## 4. 对冲组合分析

### 4.1 3+3对冲策略框架

设计了适合固收研究场景的简化对冲策略：

**策略逻辑**：
- 股票端：3只股票型ETF
- 债券端：3只债券型ETF
- 权重优化：均值-方差模型

**优化目标**：最小化组合波动率

**约束条件**：
- 权重和 = 1
- 单项权重上限 = 40%

**预设股债比例**：
| 股票权重 | 债券权重 |
|----------|----------|
| 70% | 30% |
| 60% | 40% |
| 50% | 50% |
| 40% | 60% |
| 30% | 70% |

### 4.2 有效前沿计算

基于均值-方差模型计算资产配置的有效前沿：

**方法论**：
- 期望收益：日收益均值 × 252个交易日
- 协方差矩阵：日收益协方差 × 252
- 优化求解：在目标收益约束下最小化波动率
- 输出结果：20个前沿点的风险-收益坐标

### 4.3 绩效指标体系

完整的绩效分析指标：

| 指标 | 计算方法 |
|------|----------|
| 年化收益率 | 累计收益^(252/天数) - 1 |
| 年化波动率 | 日波动率 × sqrt(252) |
| 最大回撤 | 峰值到谷底的最大跌幅 |
| 夏普比率 | (年化收益 - 无风险利率) / 年化波动率 |
| VaR | 95%置信度下的在险价值 |

## 5. 技术实现亮点

### 5.1 数据库连接池设计

```python
# SQLAlchemy连接池配置
pool_size = 5           # 连接池大小
pool_recycle = 1800     # 连接回收时间(秒)
pool_pre_ping = True    # 连接健康检查
```

**价值**：减少连接创建开销，避免连接超时导致的查询失败

### 5.2 懒加载模式

DataLoader的数据库引擎采用@property懒加载：
- 按需创建连接，节省资源
- 支持上下文管理器(with语句)
- 自动资源清理，防止连接泄漏

### 5.3 参数化查询

所有SQL查询采用参数化方式，防止SQL注入攻击：
```python
query = "SELECT * FROM etf_info WHERE category = :category"
params = {"category": "债券"}
```

### 5.4 可视化实现

使用Plotly构建交互式可视化：
- 净值对比图：多ETF净值走势叠加
- 回撤分析图：水下曲线展示
- 相关性热力图：ETF间相关系数矩阵
