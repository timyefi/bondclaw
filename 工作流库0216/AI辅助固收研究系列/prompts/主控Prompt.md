## 执行流程（完全自动化版本）

### Step 1: 读取进度

使用Read工具读取进度追踪文件：
```
G:\我的云端硬盘\workspace\projects\工作库\AI辅助固收研究系列\进度追踪.json
```

获取：
- 下一个status为"pending"的项目
- 批量执行限制（batch_size）
- 已完成数量

### Step 2: 检查批量限制

```
IF batch_size > 0 AND current_batch_completed >= batch_size:
    输出批量完成报告
    结束执行
ELSE:
    继续执行Step 3
```

### Step 3: 启动三子代理流水线

对于每个待处理项目，依次执行三个子代理：

#### 3.1 探索子代理

```
Task工具参数：
- subagent_type: "general-purpose"
- description: "探索{TXX}项目"
- prompt: （读取prompts/子代理Prompt_探索.md，替换项目信息）
```

等待完成后，读取 `temp/{TXX}_探索结果.json`

#### 3.2 分析子代理

```
Task工具参数：
- subagent_type: "general-purpose"
- description: "分析{TXX}项目"
- prompt: （读取prompts/子代理Prompt_分析.md，替换项目信息，包含探索结果）
```

等待完成后，读取 `temp/{TXX}_分析结果.json`

#### 3.3 撰写子代理

```
Task工具参数：
- subagent_type: "general-purpose"
- description: "撰写{TXX}报告"
- prompt: （读取prompts/子代理Prompt_撰写.md，替换项目信息，包含分析结果）
```

等待完成，报告已生成到reports目录

### Step 4: 更新进度

使用Edit工具更新进度追踪文件：
- 将项目status改为"completed"
- 更新statistics.completed计数
- 更新current_batch_completed计数

### Step 5: 输出摘要

输出完成摘要（仅输出，不等待确认）：
```
✅ 第{N}篇报告完成：{TXX}-{项目名称}
   - 独特性：{简要描述}
   - 心得要点：{简要描述}
   - 累计进度：{completed}/66
```

### Step 6: 自动继续

```
IF 还有pending项目 AND 未达批量限制:
    自动返回Step 1（继续处理下一个项目）
ELSE:
    输出总体完成报告
    结束执行
```

### Step 7: 错误处理

如果子代理执行失败：
1. 将项目status改为"error"
2. 记录错误信息到进度文件
3. **继续处理下一个项目**（不中断整体流程）

## 核心心得要点知识库

以下是已知的核心心得要点，在分析项目时应关联和扩展：

| 心得要点 | 核心内容 |
|---------|---------|
| **工程化思维** | AI和技术与人协同工作的本质是工程。工程化能够帮助提高效率、规范工作、提升质量、确保可复现性 |
| **重复冗余设计** | AI本质是概率论，出错和考虑不全面是必然的。通过多次重复，能够极大程度减少错误 |
| **多模型评估** | 设计阶段使用不同的大模型来反复评估方案，不同模型的视角和盲点不同，交叉验证能发现更多问题 |
| **模块化设计** | 复杂系统要拆分为独立模块，每个模块职责单一 |
| **数据驱动验证** | 任何结论都需要数据支撑，通过数据验证假设 |
| **迭代优化** | 先有基础版本，再逐步优化 |
| **可追溯性** | 代码和分析过程要能追溯 |
