# AI辅助固收研究系列报告 第26篇：T26-海外债曲线

> **研究领域**: 海外债/利率债/信用利差
> **研究目的**: 构建中资海外债收益率曲线分析框架，支持美债利率联动分析和信用利差追踪
> **适用场景**: 日常跟踪/专题研究/策略验证
> **更新频率**: 按需/每日

## 1. 研究背景与问题

### 1.1 中资海外债市场的结构性变化

2025年，中资海外债市场经历了深刻的结构性变化。这一变化的标志性事件是美债利率联动效应的显著增强。根据中信建投和中金固收的研究数据，2025年1月至4月间，10年期美债与中资美元债收益率相关系数从去年同期的0.33跃升至0.73——这意味着中资美元债与美债利率的相关性提高了整整一倍以上。

这一变化的背后是多重因素的叠加：美联储加息周期接近尾声但利率维持高位、国内货币政策保持独立性、中美利差倒挂持续、以及投资者对美债基准敏感度的提升。对于中资美元债投资者而言，这意味着过去"闭眼买"的普涨策略已经失效，投研方法论必须从贝塔策略向阿尔法策略转型。

截至2025年8月，中资美元债存量规模约为5925亿美元，到期收益率主要集中于3%-6%区间。从行业分布来看，金融和地产债占据主导地位，城投和产业债呈现多元化补充格局。在城投化债持续推进、地产风险尚未完全出清、金融和央企债相对稳健的分化格局下，精细化数据框架的构建已成为投研转型的基础设施。

### 1.2 核心研究问题

基于上述市场背景，T26项目需要回答以下核心研究问题：

- **问题一：如何量化美债利率联动效应？**
  当美债利率波动时，不同行业、评级、期限的中资海外债敏感度如何？这直接关系到组合的利率风险敞口管理。

- **问题二：如何构建多维度的信用利差追踪框架？**
  传统的人工追踪方式难以应对分行业、分评级、分期限的多维度利差变化监测需求。

- **问题三：如何为多币种债券构建可比的收益率曲线？**
  中资海外债包括美元债、港元债、点心债（离岸人民币债）等多个币种，不同币种之间的定价基准如何统一？

- **问题四：如何支撑从贝塔策略向阿尔法策略的转型？**
  在普涨策略失效的市场环境下，如何通过数据基础设施的升级支持结构性机会的挖掘？

### 1.3 研究价值

这项研究的核心价值在于：**为投研方法论转型提供数据基础设施支撑**。

具体而言：
- **利率风险管理**：通过美债联动分析，帮助投研人员量化持仓对美债利率变动的敏感度
- **信用利差监控**：建立系统化的利差追踪框架，及时发现行业和评级的利差异常变化
- **跨币种定价**：为美元债、港元债、点心债提供可比的收益率曲线基准
- **策略转型支撑**：在市场分化格局下，为从贝塔策略向阿尔法策略的转型提供数据基础

## 2. 研究思路与方法

### 2.1 整体研究框架

本项目的核心定位是**收益率曲线分析框架构建**，而非单纯的数据管道工程。研究框架围绕以下四个维度展开：

```
数据采集层 → 数据处理层 → 数据存储层 → 分析应用层
    │              │              │              │
 Excel数据     ETL模块化      双数据库架构    多维度分析
 导入          清洗转换        MySQL+PostgreSQL
```

这一框架的设计理念是：数据管道解决"数据有无"问题，分析框架解决"投研洞察"问题。T26项目在T44-海外债项目的数据采集能力基础上，进一步构建面向投研的分析框架。

### 2.2 核心方法论

#### 方法一：美债利率联动分析

通过计算中资海外债收益率与美债收益率的相关性和敏感度，量化不同债券对美债利率变动的反应程度。具体包括：

- **相关系数追踪**：动态监测中资海外债与各期限美债的相关性变化
- **敏感度计算**：基于历史数据回归，计算美债利率每变动1bp对应的债券收益率变动
- **分行业敏感度**：区分地产、城投、金融、央企等不同行业的敏感度差异

根据研究，中美利率差距每扩大1个百分点，中资投资级美元债信用利差走阔约43个基点，这一量化关系为利率风险敞口管理提供了直接参考。

#### 方法二：信用利差多维追踪

构建分行业、分评级、分期限的信用利差追踪体系：

- **行业维度**：地产、城投、金融、央企、产业等
- **评级维度**：A+、A、A-、BBB+、BBB、BBB-等
- **期限维度**：1年以内、1-3年、3-5年、5-10年、10年以上

近80%的行业经隐含评级调整后利差出现收窄，这一发现对于选择性地参与信用下沉策略具有指导意义。

#### 方法三：收益率曲线构建

为美元债、港元债、点心债三币种债券构建基准收益率曲线，支持：

- **定价基准**：为新发债券提供定价参考
- **相对价值分析**：比较不同币种债券的收益率差异
- **套利交易支持**：识别跨币种套利机会

### 2.3 与传统研究方式的对比

**传统方式**：
- 手工从Bloomberg或Wind导出数据到Excel
- 人工计算利差、敏感度等指标
- 缺乏系统化的数据存储和历史追溯能力
- 每次分析需要重复数据准备工作

**技术辅助后**：
- ETL自动化数据导入和清洗
- 双数据库架构支持高效查询和历史追溯
- 模块化设计支持分析维度扩展
- 数据与计算逻辑分离，提升复用性

**主要改进**（定性描述）：
- **效率提升**：将原本需要数小时的数据准备工作缩短至分钟级别
- **一致性保障**：统一的数据处理逻辑避免了人工操作的误差
- **可追溯性**：历史数据和分析结果可随时调取复核
- **扩展性**：新增分析维度无需重构整体架构

## 3. 数据来源与处理

### 3.1 数据来源说明

| 数据类型 | 来源 | 更新频率 | 用途 |
|---------|------|---------|------|
| 债券基础信息 | basicinfo_hk.xlsx | 按需 | 债券代码、发行人、票面利率、到期日等 |
| 收益率数据 | Excel导入 | 日/周 | 到期收益率、价格数据 |
| 美债基准 | 外部数据源 | 每日 | 美债收益率曲线 |
| 信用评级 | 债券基础信息 | 月度 | 外部评级和隐含评级 |

### 3.2 数据处理核心逻辑

ETL模块的数据处理包含以下关键步骤：

**类型检测自动化**：
通过列名关键词（'日期'/'date'/'dt'）自动识别日期列，无需人工指定列类型。这一设计大幅降低了数据源格式变化时的适配成本。

**日期转换与清洗**：
统一将各种日期格式转换为标准格式，处理缺失值和异常值。对于数值列，自动识别并处理非数值字符（如百分号、逗号等）。

**数据验证**：
导入后自动生成分布直方图和分类统计，帮助快速发现数据质量问题。

### 3.3 双数据库架构设计

本项目采用MySQL与PostgreSQL双数据库架构，这是针对金融数据特性的专业化设计：

**MySQL（关系型数据库）**：
- 存储债券基础信息（bond.overseas_bond_curve表）
- 处理高并发事务写入
- 支持复杂的关系型查询

**PostgreSQL（时序数据库）**：
- 存储收益率和价格的时序数据
- 优化时间范围查询效率
- 为收益率曲线构建预留技术扩展空间

PostgreSQL的TimescaleDB扩展可实现毫秒级写入速度，支持高效时间范围查询，特别适合收益率曲线和波动率曲面的构建场景。这种OLTP/OLAP分离的架构设计借鉴了金融系统建设的最佳实践。

## 4. 核心分析框架

### 4.1 分析维度一：美债利率联动分析

**业务背景**：
2025年中资美元债市场最显著的变化是美债利率联动效应的增强。相关系数从0.33跃升至0.73，意味着中资美元债的定价逻辑正在发生根本性变化——从过去主要受国内信用基本面驱动，转向同时受美债利率和信用基本面双重驱动。

**分析内容**：
- 计算不同期限中资海外债与对应期限美债的相关系数
- 通过回归分析计算利率敏感度系数（美债利率变动1bp对应的债券收益率变动）
- 分行业、分评级统计敏感度差异

**关键发现**：
- 金融和央企债对美债利率的敏感度较高，因其信用风险相对较低，利率因素主导定价
- 地产债的敏感度受信用事件影响波动较大，需要结合信用风险综合分析
- 城投债在化债背景下敏感度有所下降，反映国内政策因素权重提升

**业务含义**：
对于投资组合管理而言，美债联动分析直接服务于利率风险敞口的量化管理。当预期美债利率上行时，可以主动降低高敏感度债券的持仓比例；反之亦然。

### 4.2 分析维度二：信用利差追踪

**业务背景**：
在市场分化格局下，信用利差的变化往往领先于信用事件的发生。系统化的利差追踪可以帮助投研人员及时发现行业和评级的异常变化。

**分析内容**：
- 按行业维度：追踪地产、城投、金融、央企、产业等行业的利差走势
- 按评级维度：追踪A+至BBB-各评级的利差变化
- 按期限维度：追踪不同期限债券的利差结构

**关键发现**：
- 近80%的行业经隐含评级调整后利差出现收窄，反映市场对信用风险的定价趋于理性
- 地产债利差仍处于历史高位，但波动率有所下降，可能进入风险出清的后期阶段
- 城投债利差在化债政策支持下持续收窄，但区域分化明显

**业务含义**：
利差追踪为信用下沉策略提供了量化依据。在利差收窄趋势明确、信用基本面改善的行业，可以适度参与信用下沉；反之则需要规避。

### 4.3 分析维度三：多币种收益率曲线

**业务背景**：
中资海外债涵盖美元债、港元债、点心债三个主要币种。不同币种的定价基准和投资者结构存在差异，但核心都是中资发行人的信用风险。构建可比的收益率曲线有助于识别跨币种套利机会。

**分析内容**：
- 美元债收益率曲线：作为主要参考基准
- 港元债收益率曲线：考虑港美联系汇率制度的影响
- 点心债收益率曲线：反映离岸人民币资金成本

**关键发现**：
- 三币种债券的信用利差走势高度相关，但绝对水平存在差异
- 点心债的流动性溢价相对较高，在市场波动时期表现更为明显
- 跨币种套利机会主要出现在流动性冲击或汇率波动期间

**业务含义**：
多币种收益率曲线的构建为跨币种投资决策提供了定量参考。当同一发行人的不同币种债券出现显著利差时，可能存在套利空间。

## 5. 技术/AI辅助要点

### 5.1 双数据库架构的技术价值

本项目的双数据库架构设计是针对金融数据特性的专业化选择：

**MySQL的优势场景**：
- 债券基础信息存储：发行人、票面利率、到期日等属性数据
- 高并发事务处理：支持多用户同时查询和更新
- 成熟的生态工具：便于与现有系统对接

**PostgreSQL时序数据库的优势场景**：
- 收益率历史数据存储：每天、每周的数据点累积
- 高效时间范围查询：构建收益率曲线需要频繁查询特定时间区间
- 与TimescaleDB扩展配合：实现毫秒级写入和秒级时间范围查询

这种OLTP/OLAP分离的设计在金融系统建设中已得到广泛应用。通过Canal+Kafka等技术可以实现两个数据库之间的实时数据同步，保证数据一致性。

### 5.2 模块化ETL设计

ETL模块采用职责分离的设计原则：

```
数据采集 → 类型检测 → 数据清洗 → 数据存储 → 可视化验证
    │           │           │           │           │
 Excel导入   自动识别    格式转换    双库写入    分布统计
            日期/数值   缺失值处理
```

这种设计使得每个环节都可以独立测试和优化，当数据源格式变化时，只需调整类型检测和数据清洗模块，无需重构整体流程。

### 5.3 动态列处理

可视化模块采用动态列处理设计，自适应处理前4个数值列。这一设计降低了代码与数据结构的耦合度，当债券基础信息表新增字段时，可视化模块无需修改即可正常工作。

## 6. 心得要点

### 要点一：数据架构的前瞻性设计

**背景**：
项目启动时，数据源相对简单（basicinfo_hk.xlsx），分析需求也较为基础。然而，如果仅仅基于当前需求设计单数据库架构，后续扩展将面临架构重构的高昂成本。

**做法**：
采用MySQL+PostgreSQL双数据库架构。MySQL存储债券基础信息，PostgreSQL时序数据库存储收益率和价格数据。ETL模块实现从Excel批量导入到数据库的自动化管道，为未来接入更多数据源（Wind、iFinD）预留接口。

**洞察**：
技术架构设计应该基于业务扩展方向而非当前数据规模。金融数据系统特别需要考虑时序数据的特殊需求：

1. **时序数据的高效查询**：收益率曲线构建需要频繁查询特定时间区间的数据，传统关系型数据库的时间范围查询效率较低
2. **连续曲线的构建需求**：从离散的收益率数据点构建连续曲线，需要专业的插值算法和高效的数据访问
3. **历史数据的快速回溯**：回测和回顾分析需要快速调取历史数据

PostgreSQL的TimescaleDB扩展专门针对时序数据优化，可实现毫秒级写入和高效时间范围查询。这种"为未来预留空间"的设计理念，与"模块化设计"心得相呼应——通过合理的架构分层，降低未来扩展的成本。

**关联**：
这一心得与"工程化思维"心得紧密相关。工程化思维强调的不仅是解决当前问题，更重要的是以可扩展、可维护的方式解决问题。双数据库架构正是这一思维的具体体现。

### 要点二：市场转型驱动的数据需求

**背景**：
2025年中资美元债市场发生了结构性变化——美债利率联动效应显著增强。相关系数从去年同期的0.33跃升至0.73，这一变化不是渐进式的，而是跳跃式的。对于投研人员而言，这意味着过去的分析框架可能已经失效。

**做法**：
项目设计支持三大分析维度，直接回应市场变化带来的新需求：

1. **美债利率联动分析**：计算中资海外债对美债利率的敏感度，帮助投研人员量化利率风险敞口
2. **信用利差追踪**：按行业、评级、期限维度追踪利差变化，及时发现市场分化信号
3. **收益率曲线构建**：为多币种债券提供定价基准，支持跨币种比较研究

**洞察**：
数据基础设施建设应该服务于投研方法论的变化，而非孤立存在。当市场环境发生结构性变化时，数据框架需要快速响应新的分析需求。

这一心得的核心是：**用数据驱动验证市场变化，用基础设施升级响应验证结果**。

1. **识别变化**：通过量化指标（相关系数从0.33到0.73的变化）识别市场结构性变化
2. **响应变化**：基于变化的方向（美债联动增强）设计数据框架的重点能力
3. **持续迭代**：随着市场环境的进一步变化，持续优化数据框架

这与"数据驱动验证"心得相呼应——不依赖主观判断，而是通过数据量化市场变化，进而驱动决策。

**关联**：
这一心得与"迭代优化"心得紧密相关。市场环境是动态变化的，数据基础设施也需要持续迭代。关键是要建立"市场变化 → 需求识别 → 框架迭代"的闭环机制。

### 要点三：跨境债券分析的差异化定位

**背景**：
在AI辅助固收研究系列中，存在多个与海外债相关的项目。T44-海外债项目专注于数据采集和存储的工程实现，支持Wind/iFinD数据源。T26需要明确自己的定位，避免功能重叠。

**做法**：
T26定位于**收益率曲线分析框架构建**，而非单纯的数据管道。核心价值在于：

1. **分析而非存储**：重点不是把数据存进去，而是把洞察取出来
2. **框架而非数据**：重点不是原始数据本身，而是基于数据的分析方法论
3. **决策支持而非信息汇总**：重点不是告诉投研人员"发生了什么"，而是帮助判断"意味着什么"

**洞察**：
项目定位需要明确"数据管道"与"分析框架"的区别：
