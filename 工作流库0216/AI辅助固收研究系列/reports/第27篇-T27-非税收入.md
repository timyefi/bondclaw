# AI辅助固收研究系列报告 第27篇：T27-非税收入

> **研究领域**: 地方财政质量/城投债信用风险
> **研究目的**: 建立非税收入比例的标准化计算与存储流程，为地方财政质量评估和城投债信用分析提供核心指标数据
> **适用场景**: 城投债区域筛选、地方政府债信用评级、财政健康度监控
> **更新频率**: 月度更新

---

## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

在固定收益投资中，地方政府财政质量是评估城投债和地方政府债信用风险的核心要素。传统的财政分析往往聚焦于"量"——GDP规模、财政收入总量、债务余额等绝对数值，但对财政"质"的关注相对不足。

**非税收入比例**恰恰是衡量财政"质量"的关键指标。非税收入主要包括行政事业性收费、罚没收入、国有资源（资产）有偿使用收入等。与税收收入相比，非税收入具有明显的不可持续性特征：

- **行政收费**：受政策影响大，随"放管服"改革持续压缩
- **罚没收入**：波动性强，不可作为稳定财源
- **资产处置**：本质上是在"卖资产"，是一次性行为而非经常性收入

根据2025年财政数据，全国非税收入约3.97万亿元，同比下降11.3%。这一下降固然有基数效应（2024年中央单位一次性上缴专项收益）的影响，但也反映了财政结构优化的政策导向。

### 1.2 核心研究问题

本项目聚焦于以下研究问题：

- **问题一**：如何建立非税收入比例的标准化计算流程，确保数据处理的一致性和可复现性？
- **问题二**：当原始数据字段不完整时（如部分省份缺少非税收入字段），如何保证计算的连续性？
- **问题三**：如何将处理后的数据结构化存储，便于与其他财政指标联动分析？

### 1.3 研究价值

非税收入比例的研究价值体现在三个层面：

**投资层面**：非税收入占比过高的地区，其财政"造血"能力存疑，偿债能力可持续性较差。实务中，非税占比超过35%的区域往往被列为重点监控对象。

**风险识别层面**：部分区县级政府非税收入占比甚至超过50%，意味着其财政收入的一半以上来自"卖资产"或"收费罚款"，而非源自经济活动的税收。这类区域发行的城投债信用风险显著偏高。

**政策跟踪层面**：追踪各省份非税收入比例的时间序列变化，可以识别财政政策的执行效果。2025年初预算草案中，非税收入目标增速为-14.2%，体现了财政降低对非税收入依赖度的决心。

## 2. 研究思路与方法

### 2.1 整体研究框架

本项目的核心逻辑是将原始财政数据转化为投资决策可用的风险指标，遵循"数据-指标-决策"的转化链条：

```
原始数据(Excel) -> 数据清洗(标准化) -> 指标计算(非税比例) -> 数据存储(MySQL) -> 风险分析
```

这一框架的设计理念是：**数据层与业务层分离**。原始数据可能来自不同渠道、格式各异，但最终输出的指标是标准化的，便于与其他财政指标（GDP、负债率、财政自给率等）联动分析。

### 2.2 核心方法论：非税收入比例的双路径计算

本项目最核心的方法论创新是非税收入比例的**双路径计算逻辑**：

**路径一：直接计算法**

当原始数据包含"非税收入"字段时，直接计算：

```
非税收入比例 = 非税收入 / 一般公共预算收入
```

**路径二：间接推算法**

当原始数据缺少"非税收入"字段，但有"税收收入"字段时，通过恒等式推算：

```
非税收入比例 = 1 - 税收收入 / 一般公共预算收入
```

这一设计的业务逻辑是：一般公共预算收入 = 税收收入 + 非税收入。因此，当缺少非税收入数据时，可以反向推算。

**双路径计算的实战意义**：

在实际工作中，不同省份的财政数据披露标准存在差异。部分省份详细披露税收和非税收入的分项数据，而部分省份只披露税收收入和总收入。如果只采用单一路径计算，会导致部分省份数据缺失，影响区域对比分析的完整性。双路径设计确保了数据处理的鲁棒性。

### 2.3 数据清洗策略

财政数据的特殊性在于其时间序列特征和可能存在的异常值。本项目采用以下清洗策略：

**负值处理**：财政数据中出现负值的情况（如退库、调整）会被替换为空值，避免异常数据影响比例计算的准确性。

**缺失值填充**：采用前向填充（Forward Fill）策略处理月度数据的缺失值。这一策略的业务逻辑是：如果某月数据缺失，使用上一期数据是合理的近似，因为财政数据通常具有连续性和稳定性。

**省份去重**：使用集合（processed_provinces）记录已处理省份，避免同一省份因数据源重复而被重复计算。

### 2.4 与传统研究方式的对比

| 维度 | 传统方式 | 本项目方式 |
|------|---------|-----------|
| 数据处理 | 手工从Excel复制粘贴，逐省份计算 | 自动化读取、清洗、计算 |
| 数据完整性 | 遇到缺失数据只能放弃 | 双路径计算确保覆盖率 |
| 可复现性 | 计算过程难以追溯 | 数据处理流程标准化 |
| 横向对比 | 需要手工汇总各省份数据 | 统一存储格式，便于对比分析 |
| 更新频率 | 季度或年度更新 | 支持月度更新 |

## 3. 数据来源与处理

### 3.1 数据来源说明

| 数据类型 | 来源 | 更新频率 | 用途 |
|---------|------|---------|------|
| 一般公共预算收入 | 省级财政厅/统计局月报 | 月度 | 分母 |
| 税收收入 | 省级财政厅/统计局月报 | 月度 | 间接推算的分子 |
| 非税收入 | 省级财政厅/统计局月报 | 月度 | 直接计算的分子 |

**数据覆盖范围**：当前项目覆盖15个省份，时间跨度约20年。覆盖省份包括但不限于北京、重庆、新疆等具有代表性的区域，能够进行东中西部财政质量对比分析。

### 3.2 数据处理要点

**列名标准化**：不同来源的Excel文件可能使用不同的列名（如"一般公共预算收入"、"公共财政收入"、"财政收入"等），需要统一映射到标准字段名。

**数值类型转换**：Excel中的财政数据可能以文本格式存储，需要转换为数值类型。转换失败的数据标记为缺失值。

**比例计算与存储**：计算完成后，数据存储于MySQL数据库的`non_tax_ratio_df`表中，字段包括日期和各省份非税收入比例（0-1之间的小数）。

## 4. 核心分析框架

### 4.1 分析维度一：区域财政质量对比

**业务意义**：非税收入比例是财政质量的"晴雨表"。比例越低，说明财政收入对税收的依赖度越高，经济"造血"能力越强。

**典型数据**：

| 地区 | 税收占比 | 非税占比 | 财政质量评价 |
|------|----------|----------|-------------|
| 北京市 | 86.5% | 约13.5% | 全国最优 |
| 陕西省 | 77.01% | 约23% | 良好 |
| 重庆市璧山区 | 63.3% | 约36.7% | 一般，需关注 |
| 部分西部区县 | <50% | >50% | 较差，高风险 |

**业务含义**：北京作为首都，产业结构高端（金融、科技、总部经济），税源丰富且稳定，税收占比全国最高。而部分中西部地区的财政高度依赖转移支付、土地出让和非税收入，自主财力较弱。

对于城投债投资者而言，非税占比超过35%的区域应当重点关注其财政可持续性风险。2025年，多地披露的财政数据显示，部分区县级政府非税占比甚至超过50%，这类区域的城投债信用利差理应更宽。

### 4.2 分析维度二：时间序列趋势追踪

**业务意义**：单一时点的非税收入比例只能反映静态情况，时间序列分析能够揭示财政质量的动态变化趋势。

**分析方法**：本项目通过月度数据的前向填充，构建各省份非税收入比例的时间序列。通过趋势图可以识别：

- **改善趋势**：非税占比持续下降，说明财政结构优化
- **恶化趋势**：非税占比持续上升，需警惕财政压力加大
- **异常波动**：某月非税占比突然跳升，可能是资产处置等一次性因素

**典型场景**：2024-2025年，多地政府响应"过紧日子"号召，主动压缩非税收入（如减免行政收费、规范罚没收入），非税占比整体呈下降趋势。但部分财政压力较大的地区，可能通过"盘活存量资产"（实质是出售政府资产）来增加非税收入，这类区域的财政质量改善需要审慎评估。

## 5. 技术/AI辅助要点

本项目的技术亮点主要体现在**数据处理流程的工程化设计**上。

### 5.1 双路径计算的实现逻辑

```python
# 伪代码示意
if 'non_tax_revenue' in df.columns:
    # 路径一：直接计算
    non_tax_ratio = df['non_tax_revenue'] / df['general_budget_revenue']
else:
    # 路径二：间接推算
    non_tax_ratio = 1 - df['tax_revenue'] / df['general_budget_revenue']
```

这一设计的核心思想是**"备份思维"**：当首选方案（直接计算）不可用时，有替代方案（间接推算）确保流程继续。这种容错设计在处理多来源、异构数据时尤为重要。

### 5.2 数据处理流水线

项目采用标准化的数据处理流水线：

```
读取Excel -> 列名映射 -> 类型转换 -> 负值处理 -> 缺失值填充 -> 比例计算 -> 数据库存储
```

每一环节都有明确的输入输出规范，便于调试和维护。当数据源发生变化时，只需调整相应环节的处理逻辑，而不需要重构整个流程。

## 6. 心得要点

### 要点一：指标选取的金融逻辑——非税收入比例作为财政质量代理变量

**背景**：固定收益研究中需要评估地方政府的偿债能力和财政健康状况，但原始财政数据维度众多（GDP、负债率、债务率、税收、非税收入等），如何选择核心指标是关键问题。

**做法**：本项目聚焦于"非税收入比例"这一单一指标，而非试图构建一个综合评分模型。

**洞察**：

非税收入比例之所以有效，是因为它直接反映了财政收入的"质量"而非"数量"。在信用分析中，我们关注的不仅是"这个政府有多少钱"，更是"这些钱从哪里来、可持续性如何"。

非税收入（行政收费、罚没、资产处置等）具有明显的不可持续性特征。一个高度依赖非税收入的地方政府，本质上是在"吃老本"——要么通过收费罚款增加企业负担，要么通过出售政府资产获取一次性收入。这种财政模式难以长期维持，在债务偿还压力下尤其脆弱。

这启示我们：**在构建分析框架时，应选择具有明确经济学含义且与风险正相关的指标。指标的业务含义理解比技术实现更重要。**

**关联**：这一思路与"第一性原理"思维相通——回到问题的本质，找到最能反映核心风险的代理变量，而非堆砌指标。

### 要点二：数据处理的容错设计——双路径计算与缺失值策略

**背景**：原始财政数据来自Excel文件，不同省份的数据字段可能存在差异：部分省份有非税收入字段，部分省份只有税收收入字段，需要处理数据不完整的情况。

**做法**：设计了双路径计算逻辑——优先使用非税收入直接计算，若字段缺失则通过税收收入间接推算。同时，对于负值（可能是数据错误或特殊调整）替换为空值，并用前向填充处理时序数据缺失。

**洞察**：

数据处理的鲁棒性设计是分析项目成功的关键。在实际工作中，"理想的数据"几乎不存在——字段缺失、格式不一致、异常值、重复记录等问题层出不穷。如果代码只能处理"干净"的数据，那么它的实用价值将大打折扣。

本项目的双路径计算体现了一种"备份思维"：当首选方案不可用时，有替代方案确保流程继续。这种设计在处理多来源、异构数据时尤为重要。

更一般地说，好的数据处理代码应该具备"**优雅降级**"的能力——当输入数据质量下降时，输出质量相应下降但不崩溃，并且能够清晰记录哪些数据使用了替代方案。

**关联**：这与软件工程中的"防御性编程"理念相通——假设一切可能出错的地方都会出错，并提前设计应对方案。

### 要点三：数据与业务的双向映射——从Excel到投资决策

**背景**：原始数据是Excel格式的财政月度数据，最终用途是支持固定收益投资决策（城投债、地方政府债风险评估），需要建立数据到业务的映射。

**做法**：项目通过三个层次建立映射：

1. **数据层**：Excel -> DataFrame标准化，处理格式差异
2. **指标层**：计算非税收入比例，将原始数据转化为分析指标
3. **存储层**：MySQL数据库持久化，供后续分析工具调用

**洞察**：

数据分析项目的核心价值不在于技术复杂度，而在于能否建立数据与业务的有效映射。本项目的价值不在于使用了什么高级算法（事实上只用到了基础的除法运算），而在于将原始财政数据转化为投资决策可用的风险指标。

这一"数据->指标->决策"的转化链条是所有分析项目的通用模式。技术只是工具，业务理解才是灵魂。一个没有金融背景的程序员可能写出高效的数据处理代码，但如果不理解"非税收入比例高意味着高风险"这个业务逻辑，就无法判断计算结果是否合理。

这启示我们：**在分析项目中，技术能力是基础，业务理解是上限。**

## 7. 实际应用指南

### 7.1 如何使用这项研究

**场景一：城投债区域筛选**

- 关注指标：非税收入比例
- 筛选标准：非税占比 > 35%的区域应重点关注财政可持续性风险
- 决策参考：对于非税占比持续上升的区域，建议提高信用利差要求或降低配置比例

**场景二：地方政府债信用评级辅助**

- 用途：作为财政实力评估的量化指标之一
- 结合指标：需同时参考GDP规模、负债率、财政自给率等指标
- 注意事项：单一指标不足以判断信用风险，应纳入综合评估框架

**场景三：财政健康度监控**

- 更新频率：月度
- 关注内容：各省份非税收入比例的时间序列变化
- 异常信号：某月非税占比突然跳升 > 5个百分点，需查明原因（是否为资产处置等一次性因素）

### 7.2 如何运行这个工具

1. **准备原始数据**：将财政月报数据整理为Excel格式，需包含省份名称、一般公共预算收入、税收收入或非税收入字段
2. **执行数据处理**：运行Notebook执行数据清洗和计算
3. **读取分析结果**：从MySQL数据库的`non_tax_ratio_df`表读取处理后的非税收入比例数据
4. **结合其他指标**：将非税收入比例与GDP、负债率、财政自给率等指标联动分析

### 7.3 输出物说明

**数据表结构**：`non_tax_ratio_df`

| 字段 | 类型 | 说明 |
|------|------|------|
| date | DATE | 数据日期 |
| 北京 | FLOAT | 北京市非税收入比例 |
| 重庆 | FLOAT | 重庆市非税收入比例 |

| 新疆 | FLOAT | 新疆维吾尔自治区非税收入比例 |

**数据范围**：15个省份，时间跨度约20年

**数值含义**：0-1之间的小数，例如0.35表示非税收入占比35%，可转换为百分比展示

## 8. 延伸思考

### 8.1 可扩展的研究方向

**多指标综合评估**：非税收入比例仅反映财政质量的一个维度。可以结合以下指标构建综合评估框架：

- **财政自给率**：一般公共预算收入 / 一般公共预算支出
- **负债率**：政府债务余额 / GDP
- **债务率**：政府债务余额 / 综合财力
- **土地财政依赖度**：土地出让收入 / 综合财力

**区县级下沉分析**：当前项目仅覆盖省级数据。区县级财政的分化更加显著，部分区县非税占比超过50%。如果能获取区县级数据，可以进行更精细的风险识别。

**预警机制构建**：设定非税收入比例的阈值（如30%、35%、40%），当某区域突破阈值时自动触发预警，并生成风险简报。

### 8.2 与其他项目的关联

本项目（T27-非税收入）可与以下项目联动：

- **T系列财政指标项目**：将非税收入比例与其他财政指标（负债率、财政自给率等）整合，构建地方政府财政健康度评分卡
- **城投债信用分析项目**：作为城投债主体信用评级的输入指标之一
- **区域经济分析项目**：结合GDP、产业结构等指标，分析财政质量与经济发展的关联

### 8.3 待改进之处

诚实地说，本项目目前存在以下不足：

1. **数据覆盖有限**：当前仅包含15个省份，未覆盖全国所有省级行政区。如果需要进行全国范围对比分析，需要补充数据。
