> **研究目的**: 构建住宅库存的双口径预测框架，为房地产信用债投资提供库存周期判断依据
> **适用场景**: 专题研究/信用风险评估/宏观背景分析
> **更新频率**: 月度更新

## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

房地产去库存是当前中国经济金融领域的核心议题之一。2025年被视为新一轮房地产去库存元年，中央经济工作会议将政策定调为"控增量、去库存、优供给"。根据各大券商研究机构的测算，2024年底商品住宅广义库存去化周期约为25个月，这意味着按照当前销售速度，消化现有库存需要两年以上。

对于固收投研人员而言，房地产库存周期直接影响两个核心关切：

**一是房企信用风险评估。** 库存去化周期是判断房企现金流回笼速度的关键指标。当库销比超过一定阈值时，房企面临销售承压、回款放缓、资金链紧张的风险，这将直接传导至债券信用利差。

**二是宏观利率环境判断。** 房地产作为中国经济的支柱产业，其库存周期与经济景气度、货币政策取向高度相关。去库存压力较大时，政策往往需要保持宽松以支持需求释放，这对利率债投资具有重要的信号意义。

据开源证券等研究机构预测，在政策持续发力的背景下，2026年末住宅广义库存去化周期有望降至13.7个月。但这一预测存在较大的不确定性——不同的假设条件将导向截然不同的结论。这就引出了本研究的核心问题。

### 1.2 核心研究问题

本项目聚焦于以下研究问题：

- **问题一：住宅库存未来将如何演变？** 在延续历史趋势和供需平衡两种假设下，库存路径有何差异？
- **问题二：官方"待售面积"指标是否足够？** 如何将其调整为更准确反映实际库存压力的口径？
- **问题三：如何量化预测的不确定性边界？** 当不同预测方法的结果出现分歧时，如何解读其含义？

### 1.3 研究价值

本研究的核心价值在于：

1. **提供双口径预测框架**：不仅给出点预测，更通过多口径对比揭示预测的不确定性边界
2. **建立口径调整方法论**：将官方待售面积转换为包含期房潜在库存的广义口径
3. **支持信用投资决策**：为评估房企信用风险提供量化的库存周期判断依据

## 2. 研究思路与方法

### 2.1 整体研究框架

本研究采用"双口径预测 + 口径调整"的框架设计：

```
                    ┌─────────────────────────────────────┐
                    │         原始数据输入                 │
                    │  (官方公布的待售面积、销售、竣工等)    │
                    └─────────────────┬───────────────────┘
                                      │
                                      ▼
                    ┌─────────────────────────────────────┐
                    │         口径调整模块                 │
                    │  待售面积 → 广义库存估算             │
                    │  累计值 → 月度值转换                 │
                    └─────────────────┬───────────────────┘
                                      │
                    ┌─────────────────┴───────────────────┐
                    │                                     │
                    ▼                                     ▼
    ┌───────────────────────────┐       ┌───────────────────────────┐
    │     口径一：趋势推演       │       │     口径二：销售-竣工     │
    │                           │       │         去化法            │
    │  基于历史数据的线性回归    │       │   基于供需平衡的动态推演   │
    │  假设：过去趋势延续       │       │   假设：库存=销售-竣工    │
    └─────────────┬─────────────┘       └─────────────┬─────────────┘
                  │                                   │
                  └─────────────┬─────────────────────┘
                                │
                                ▼
                    ┌─────────────────────────────────────┐
                    │         结果对比与分析              │
                    │   · 两种口径的库存预测路径          │
                    │   · 库销比指标计算                  │
                    │   · 不确定性边界识别                │
                    └─────────────────────────────────────┘
```

### 2.2 核心方法论

本研究的方法论包含三个关键组件：

#### 2.2.1 住宅库存口径调整公式

官方公布的"待售面积"指标存在一个重要局限：它只统计已竣工未售的现房，不包含已预售但尚未交付的期房。在当前中国房地产市场中期房销售占主导的背景下，这一指标可能显著低估实际库存压力。

为解决这一问题，本项目建立了库存估算公式：

**库存 = 待售面积 × (期房累计销售 + 现房累计销售) / 期房累计销售**

这一公式的核心逻辑是：利用现房与期房销售的比例关系，将官方待售面积放大为包含潜在期房库存的广义口径。当期房销售占比较高时，放大系数相应增大，反映了市场中期房交付压力的累积。

#### 2.2.2 双口径预测方法

**口径一：趋势推演法**

使用历史库存数据进行线性回归，假设过去的趋势将延续至未来。这种方法的优势是简单直观，局限性在于无法捕捉市场拐点。

**口径二：销售-竣工去化法**

基于供需平衡原理，通过销售与竣工的月度差值动态推演库存变化。当销售大于竣工时，库存下降；反之则库存上升。这种方法更贴近库存变动的经济学逻辑。

#### 2.2.3 库销比指标

库销比（库存/过去12个月销售）是判断库存消化能力的关键指标。本项目分别基于两种预测口径计算库销比，为判断库存周期位置提供多维参考。一般而言：

- 库销比 < 12个月：库存处于健康水平
- 库销比 12-18个月：库存压力适中
- 库销比 > 18个月：库存压力较大，需要警惕

### 2.3 与传统研究方式的对比

**传统方式**：券商研究通常采用"剪刀差分析法"——通过计算新开工与销售的累计差值来估算广义库存。这种方法的数据来源是土地成交和新开工面积，但其假设是所有新开工最终都会形成可售库存，可能高估实际库存压力。

**本项目的技术方案**：采用"销售-竣工去化法"，从供需两端直接观察库存变动。核心逻辑是：库存增量 = 竣工增量 - 销售增量。这种方法更直接地反映了实际可售库存的变化。

**主要改进**：

1. **多口径预测**：不是给出单一预测值，而是通过两种假设的对比揭示不确定性边界
2. **口径转换**：将官方待售面积调整为更准确的广义库存口径
3. **动态更新**：框架支持月度更新，可及时跟踪市场变化

## 3. 数据来源与处理

### 3.1 数据来源说明

| 数据类型 | 来源 | 指标名称 | 用途 |
|---------|------|---------|------|
| 住宅待售面积 | 国家统计局 | 商品房待售面积:住宅 | 库存口径调整的基数 |
| 住宅销售面积 | 国家统计局 | 商品房销售面积:住宅（累计值） | 销售-竣工去化法计算、库销比分母 |
| 住宅竣工面积 | 国家统计局 | 房屋竣工面积:住宅（累计值） | 销售-竣工去化法计算 |
| 期房销售面积 | 国家统计局 | 商品房销售面积:住宅:期房（累计值） | 口径调整的放大系数计算 |
| 现房销售面积 | 国家统计局 | 商品房销售面积:住宅:现房（累计值） | 口径调整的放大系数计算 |
| 新开工面积 | 国家统计局 | 房屋新开工面积:住宅（累计值） | 辅助分析参考 |

### 3.2 数据处理要点

#### 3.2.1 累计值转月度值的智能处理

统计局公布的房地产数据多为累计值（年初至当月），需要转换为月度值才能用于滚动计算。但这里存在一个数据发布特性：1月份可能不单独公布，2月直接公布1-2月累计值，导致数据存在"跳跃"特征。

本项目采用智能识别算法处理这一问题：

**核心逻辑**：通过检测相邻累计值的差值符号变化来自动识别年初重置点。当差值小于零时，说明当前值小于前值，必定是年度重置（因为累计值只会增加或持平，不可能下降，除非跨年）。

在重置点的处理：直接使用原值作为当月值（因为是新一年的第一个月）；否则使用差值作为当月值。

这一处理体现了工程化思维：不假设数据完美，而是设计鲁棒的处理逻辑应对数据不规则性。用数据自身的特征（如累计值必然在年初下降）来识别特殊情况，而非依赖外部标记，这种"自识别"机制使代码具有更强的适应性和可移植性。

#### 3.2.2 滚动窗口处理

对于销售、竣工、新开工等指标，采用12个月滚动求和的方式，以消除季节性波动，反映年度化的供给和需求水平。

## 4. 核心分析框架

### 4.1 分析维度一：库存预测的双口径对比

**分析内容**：分别使用趋势推演法和销售-竣工去化法预测未来库存路径，对比两种口径的差异。

**关键发现**：

1. **趋势推演口径**：假设历史趋势延续，给出的是"惯性情景"下的预测。当市场处于稳定期时，这一口径的预测较为准确；但当市场面临拐点时，可能低估或高估库存变化。

2. **销售-竣工去化口径**：基于供需平衡逻辑，给出的是"平衡情景"下的预测。这一口径假设销售和竣工的趋势独立延续，动态计算库存变化。它能更早地捕捉到市场拐点信号。

3. **两种口径的差异含义**：当两种口径的预测结果出现较大分歧时，说明市场存在较大的不确定性或转折风险。差异本身就是有价值的信息——它标记了预测的不确定性边界。

**业务含义**：

对于债券投资者而言，当两种口径差异扩大时，应提高警惕：
- 如果趋势推演预测库存下降，而销售-竣工法预测库存上升，说明市场可能面临需求不足的风险
- 如果趋势推演预测库存上升，而销售-竣工法预测库存下降，说明市场可能存在供给收缩带来的改善机会

### 4.2 分析维度二：库销比的动态跟踪

**分析内容**：基于两种预测口径分别计算库销比（库存/过去12个月销售），跟踪库存消化能力的变化。

**关键发现**：

1. 库销比是判断库存周期位置的关键指标。当库销比处于高位时，即使库存绝对值下降，也可能意味着销售端承压。

2. 两种口径下的库销比可能呈现不同的趋势。需要同时关注两个口径，以全面评估库存压力。

**业务含义**：

库销比是房企信用风险评估的重要参考指标：
- 库销比持续上升：房企销售承压，回款放缓，信用风险上升
- 库销比持续下降：房企销售顺畅，回款改善，信用风险下降

## 5. 技术/AI辅助要点

本项目的技术亮点集中在数据处理方法论上，而非复杂的算法模型。

### 5.1 口径调整公式的建立

口径调整公式（库存 = 待售面积 × (期房累计 + 现房累计) / 期房累计）的建立过程，体现了"数据翻译"的方法论：

当直接数据不可得时，可以通过相关变量之间的逻辑关系进行估算推演。核心思路是找到与目标变量逻辑相关的可获取数据，建立映射关系。

### 5.2 累计值处理的鲁棒设计

通过数据自身特征（累计值在年初必然下降）来识别特殊情况，而非依赖外部标记。这种设计使代码具有更强的适应性：

- 不需要预先知道数据发布的日历规则
- 可以自动适应数据发布节奏的变化
- 便于迁移到其他具有类似累计值特征的数据集

## 6. 心得要点

### 要点一：多口径预测的价值——不确定性边界的识别

**背景**：房地产库存预测面临不确定性，单一预测模型难以覆盖所有情景。本研究设计了两种不同假设的预测口径：趋势推演和销售-竣工去化。

**做法**：趋势推演使用线性回归，假设历史趋势延续；销售-竣工去化基于供需平衡，通过月度销售与竣工差值动态计算库存变化。两者的差异反映了不同假设下的市场情景。

**洞察**：多口径预测不是简单的"多算一次"，而是通过不同假设条件的对比，识别预测的不确定性边界。两种口径的差异本身就是有价值的信息——当差异扩大时，说明市场存在更大的不确定性或转折风险。

这种方法论可推广到其他预测场景：与其追求单一精确预测，不如构建多情景框架，让决策者了解预测的不确定性区间。在固收研究中，无论是利率预测、信用利差预测还是宏观指标预测，都可以借鉴这一思路。

**关联**：与"多模型评估"、"数据驱动验证"等心得要点相关。

### 要点二：口径调整的方法论——数据翻译的艺术

**背景**：官方公布的"待售面积"指标存在口径局限：只统计已竣工未售的现房，不包含已预售但未交付的期房。这导致官方数据可能低估实际库存压力。

**做法**：项目独创建立库存估算公式，核心思路是利用现房与期房销售的比例关系，将待售面积放大为包含潜在期房库存的广义口径。

**洞察**：口径调整的本质是"数据翻译"——将可获得的公开数据转换为决策所需的指标。在金融数据分析中，官方指标往往存在滞后性、口径局限等问题，分析者需要建立从公开数据到决策指标的映射关系。

这个公式的启示是：当直接数据不可得时，可以通过相关变量之间的逻辑关系进行估算推演。关键在于找到逻辑上相关、统计上可获取的代理变量。

**关联**：与"数据驱动验证"心得要点相关。

### 要点三：累计值数据处理的工程化思维——自识别机制的设计

**背景**：统计局公布的房地产数据多为累计值（年初至当月），需要转换为月度值才能用于滚动计算。但1月份可能不单独公布，2月直接公布1-2月累计值，导致数据存在"跳跃"特征。

**洞察**：这个处理体现了工程化思维：不假设数据完美，而是设计鲁棒的处理逻辑应对数据不规则性。核心原则是：用数据自身的特征（如累计值必然在年初下降）来识别特殊情况，而非依赖外部标记。

这种"自识别"机制使代码具有更强的适应性和可移植性。在处理其他具有类似特征的数据时（如财政数据的累计值、进出口数据的累计值等），可以直接复用这一逻辑。

## 7. 实际应用指南

### 7.1 如何使用这项研究

**场景一：房企信用风险评估**

- 关注指标：库销比（两种口径）、库存变动趋势
- 使用方式：当库销比超过18个月且两种口径均显示上升趋势时，提高对该房企债券的风险警惕
- 注意事项：需结合房企个体特征（土储布局、融资能力等）综合判断

**场景二：宏观利率环境判断**

- 关注指标：库存去化周期、销售-竣工差值
- 使用方式：库存压力较大时，政策倾向于保持宽松，对利率债形成支撑
- 注意事项：需结合其他宏观指标（PMI、通胀等）综合判断

**场景三：房地产行业专题研究**

- 关注指标：两种口径预测的差异
- 使用方式：差异扩大时，说明市场存在较大不确定性，需要更密切跟踪

### 7.2 如何运行这个工具

1. 配置数据源连接参数（数据库连接或数据文件路径）
2. 设置预测起止日期（默认预测至2027年12月）
3. 运行main.py获取预测结果
4. 对比两种口径的差异，分析不确定性边界

## 8. 延伸思考

### 8.1 可扩展的研究方向

1. **引入政策变量**：当前模型未纳入限购放松、购房补贴等政策因素对去库存的影响。可考虑将政策指数作为调节变量引入模型。
