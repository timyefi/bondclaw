## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

可转换债券（可转债）是中国资本市场中一类独特的混合型金融产品，兼具债券的防御属性和股票的进攻属性。近年来，可转债市场发展迅速，已成为固定收益投资领域不可忽视的重要板块。

然而，与股票市场丰富的指数体系相比，可转债市场的指数体系相对匮乏。市场上虽然有中证转债指数等少数宽基指数，但缺乏从不同维度（如债性/股性、估值水平、行业分布）刻画可转债市场结构的细分指数体系。这给投研人员带来了以下困扰：

- **缺乏有效的市场跟踪工具**：无法准确把握不同类型可转债的相对表现
- **策略研究缺乏基准**：双低策略、低溢价策略等量化策略难以找到合适的对比基准
- **行业轮动分析困难**：缺乏行业维度的可转债指数，难以进行行业配置研究
- **转债与正股联动分析不足**：缺乏可比的转债与正股指数，难以分析两者的相对表现

### 1.2 核心研究问题

本研究旨在解决以下核心问题：

1. **如何构建可转债的多维度分类体系？** 可转债具有债性和股性双重特征，单一维度的分类无法全面刻画其风险收益特征。需要设计能够同时反映债券属性和股权属性的分类方法论。

2. **如何管理动态变化的样本池？** 可转债市场样本数量有限且动态变化（新发行、提前赎回、转股退出），如何在保证指数代表性的同时确保成分稳定性？

3. **如何选择合理的加权方式？** 传统市值加权在可转债领域存在局限性，需要设计更适合可转债特点的加权方法。

4. **如何构建转债与正股的联动分析框架？** 转债与正股高度相关，如何设计指数体系支持两者的对比分析？

### 1.3 研究价值

本研究的价值体现在以下三个层面：

**业务层面**：
- 为可转债投资提供细分的指数跟踪工具
- 支持可转债量化策略的回测与评估
- 为行业配置和风格轮动研究提供数据基础

**方法论层面**：
- 提出债性+溢价的双重分类方法论
- 设计季度样本池+多级放宽的动态管理机制
- 构建转债正股联动指数体系

**技术层面**：
- 实现连接池+Parquet缓存+向量化计算的全链路性能优化
- 提供可复用的指数计算代码架构

### 2.2 核心方法论

#### 2.2.1 债性分类方法论

债性分类基于纯债溢价率，将可转债分为三类：

| 分类 | 纯债溢价率阈值 | 特征描述 |
|------|---------------|---------|
| 偏债型 | < -10% | 债性保护强，价格接近或低于纯债价值 |
| 平衡型 | -10% ~ 10% | 债性与股性相对均衡 |
| 偏股型 | > 10% | 股性突出，价格显著高于纯债价值 |

**纯债溢价率计算公式**：
```
纯债溢价率 = (转债价格 - 纯债价值) / 纯债价值
```

这一分类的核心逻辑是：纯债溢价率越低，说明可转债价格相对于纯债价值的偏离越小，债性保护越强。当纯债溢价率为负时，可转债价格甚至低于同等条件普通债券的理论价值，提供了较高的安全边际。

#### 2.2.2 溢价四象限分类方法论

溢价分类从价格和转股溢价率两个维度构建四象限矩阵：

| 象限 | 价格条件 | 转股溢价率条件 | 特征描述 |
|------|---------|---------------|---------|
| 双低 | <= 110元 | <= 20% | 兼具安全边际和上涨潜力 |
| 低价高溢 | <= 110元 | > 20% | 价格便宜但估值偏高 |
| 高价低溢 | > 110元 | <= 20% | 价格较高但估值合理 |
| 双高 | > 110元 | > 20% | 价格和估值双高，风险较大 |
| 普通型 | 其他 | 其他 | 不满足以上分类的补充 |

**转股溢价率计算公式**：
```
转股溢价率 = (转债价格 - 转股价值) / 转股价值
转股价值 = 正股价格 × 转股比例
```

四象限分类的核心价值在于：双低转债是可转债投资中的"甜蜜点"，既具备价格安全边际，又保留了跟随正股上涨的空间。这一分类方法直接服务于双低策略等量化投资方法。

#### 2.2.3 季度样本池动态管理

指数样本池采用季度调整周期，每季度末基于以下条件筛选：

1. **存续稳定性**：季度首末日都有未转股余额 > 0
2. **价格合理性**：价格在0-300元正常范围内
3. **流动性筛选**：换手率 < 200%排除异常交易
4. **估值合理性**：溢价率在-80%~300%合理区间

季度内每日仅使用样本池内的债券计算指数，不做动态调整。这种设计平衡了指数的代表性和稳定性。

#### 2.2.4 未转股余额加权方法

采用未转股余额作为权重因子计算加权平均价格：

```
指数价格 = sum(成分券价格 × 未转股余额) / sum(未转股余额)
```

同时设置单只债券权重上限10%，防止单一券种过度影响指数。

选择未转股余额加权而非市值加权的理由：
- 未转股余额反映市场上实际流通的可转债规模
- 避免市值加权中股价波动导致的权重不稳定问题
- 大盘转债（如银行转债）不会过度主导指数走势

### 2.3 与传统研究方式的对比

| 维度 | 传统方式 | 本研究方法 |
|------|---------|-----------|
| 分类维度 | 单一分类（如仅按行业） | 双重分类（债性+溢价）+ 行业分类 |
| 样本管理 | 固定样本或每日调整 | 季度调整+多级放宽机制 |
| 加权方式 | 市值加权或等权 | 未转股余额加权+权重上限 |
| 转债正股关系 | 分离研究 | 联动指数体系 |
| 数据获取 | 手动Excel处理 | 自动化采集+本地缓存 |

### 3.2 数据采集策略

由于同花顺iFinD接口对单次请求的指标数量有限制，本研究采用分步采集策略：

1. 将9个指标分别配置为独立的采集任务
2. 逐个指标调用THS_DS接口
3. 每个指标采集后检查结果，失败则记录日志但不中断整体流程
4. 最后合并所有成功获取的数据

这种设计的优势：
- 单个指标失败不影响其他指标
- 每次请求只获取一个指标，降低接口压力和超时风险
- 每个指标独立的错误处理便于定位问题

### 4.2 样本补充多级放宽策略

在市场极端情况下，严格按照初始条件筛选可能导致样本数量不足。本研究设计了五级递进的样本补充策略：

| 放宽级别 | 调整内容 | 触发条件 |
|---------|---------|---------|
| 第一级 | 溢价率要求放宽±2% | 样本数不足目标 |
| 第二级 | 溢价率要求放宽至±5% | 第一级仍不足 |
| 第三级 | 换手率要求降至0.3% | 第二级仍不足 |
| 第四级 | 换手率要求降至0.1% | 第三级仍不足 |
| 第五级 | 完全放开条件 | 第四级仍不足 |

条件放宽的顺序设计遵循"投资属性优先于交易属性"原则：先放宽价格和溢价率（影响投资价值），再放宽换手率（仅影响交易便利性）。

### 4.3 连续达标天数检查

对于需要连续达标才能入池的场景（如连续N日满足流动性要求），本研究采用向量化检查方法：

```python
def _check_continuous_qualification(self, df, date_col, bond_col, condition_col, required_days):
    """
    通过cumsum和状态变化检测计算连续达标天数
    """
    # 标记不满足条件的点
    df['break_point'] = ~df[condition_col]

    # 累计断点数，用于分组
    df['group'] = df.groupby(bond_col)['break_point'].cumsum()

    # 在每个分组内计算连续天数
    df['continuous_days'] = df.groupby([bond_col, 'group']).cumcount() + 1

    # 筛选连续达标天数满足要求的记录
    qualified = df[df['continuous_days'] >= required_days]
    return qualified
```

这种方法避免了对每只债券逐日循环检查，大幅提升计算效率。

### 4.4 布尔列分类标记

本研究采用独立的布尔列标记分类，而非单一分类列：

```python
# 债性分类
df['偏债型'] = df['纯债溢价率'] < -0.10
df['平衡型'] = (df['纯债溢价率'] >= -0.10) & (df['纯债溢价率'] <= 0.10)
df['偏股型'] = df['纯债溢价率'] > 0.10

# 溢价分类
df['双低'] = (df['收盘价'] <= 110) & (df['转股溢价率'] <= 0.20)
df['低价高溢'] = (df['收盘价'] <= 110) & (df['转股溢价率'] > 0.20)
df['高价低溢'] = (df['收盘价'] > 110) & (df['转股溢价率'] <= 0.20)
df['双高'] = (df['收盘价'] > 110) & (df['转股溢价率'] > 0.20)
```

这种设计的优势：
- 支持一只转债同时属于多个分类
- 便于灵活组合查询
- 代码可读性强

### 5.2 Parquet缓存加速历史数据读取

本研究实现了基于Parquet格式的本地缓存机制：

```python
def get_historical_data(self, start_date, end_date, use_cache=True):
    cache_file = f"{self.cache_dir}/bond_data_{start_date}_{end_date}.parquet"

    if use_cache and os.path.exists(cache_file):
        return pd.read_parquet(cache_file)

    # 从数据库查询
    df = self._query_from_database(start_date, end_date)

    # 保存缓存
    df.to_parquet(cache_file, index=False)

    return df
```

选择Parquet格式的原因：
- 列式存储格式，对宽表（列多）和大数据集的读取性能显著优于CSV
- 保留数据类型信息，不需要重复的类型转换
- 本地文件读取比网络数据库查询快数个数量级

### 5.3 向量化批量指数计算

传统循环逐日计算指数的方法效率低下，本研究采用向量化计算：

```python
def calculate_index_price(self, df):
    # 预计算加权价格列
    df['weighted_close'] = df['收盘价'] * df['未转股余额']

    # 批量聚合计算
    result = df.groupby('交易日期').agg({
        'weighted_close': 'sum',
        '未转股余额': 'sum',
        '债券代码': 'count',
        '换手率': 'mean'
    }).reset_index()

    # 计算加权平均价格
    result['指数价格'] = result['weighted_close'] / result['未转股余额']

    return result
```

向量化计算的优势：
- 利用NumPy的底层优化，避免Python循环开销
- groupby聚合在C层面执行，比Python循环快数十倍
- 预计算加权价格列将复杂的加权平均转换为简单的求和运算

### 5.4 批量数据库写入

使用executemany批量插入替代逐条INSERT：

```python
def _save_index_market_data(self, index_data):
    sql = """
    INSERT INTO index_market_data
    (index_code, trade_date, index_price, sample_count, avg_turnover)
    VALUES (%s, %s, %s, %s, %s)
    ON DUPLICATE KEY UPDATE
    index_price = VALUES(index_price),
    sample_count = VALUES(sample_count),
    avg_turnover = VALUES(avg_turnover)
    """

    values = [
        (row['指数代码'], row['交易日期'], row['指数价格'],
         row['样本数'], row['平均换手率'])

    self.cursor.executemany(sql, values)

```

配合ON DUPLICATE KEY UPDATE实现幂等写入，支持多次运行而不产生重复数据。

## 6. 心得要点

### 6.1 关于混合型资产的分类方法论

可转债作为混合型金融产品，其最大特点是同时具有债性和股性。传统的单一维度分类方法（如仅按行业或仅按评级）无法全面刻画其风险收益特征。

本研究的核心创新在于设计了**两套互补的分类体系**：

1. **债性分类**从"安全边际"角度出发，用纯债溢价率衡量转债相对于纯债的价值偏离
2. **溢价分类**从"投资价值"角度出发，用价格-转股溢价率双维度划分四象限

两套体系互为补充，能够满足不同投资策略的需求：偏债型投资者关注债性分类，量化投资者关注溢价分类。这种多维度分类思想可以推广到其他混合型金融产品（如可交换债、优先股）的分类研究。

### 6.2 关于小样本市场的指数设计

可转债市场是一个典型的小样本市场，全市场可转债数量约500只左右，远少于股票市场的4000+只。小样本市场对指数设计提出了特殊挑战：

- **样本波动大**：新发行、赎回、转股都会改变样本池
- **行业分布不均**：某些行业转债数量有限
- **流动性差异大**：部分转债交易活跃度不足

本研究采用的**季度调整+多级放宽机制**有效解决了这些问题。季度调整与机构投资者调仓节奏匹配，保证了成分稳定性；多级放宽机制确保在极端情况下也能构建有效指数。

这一设计思路可以推广到其他小样本市场（如REITs、新三板）的指数构建。

### 6.3 关于联动指数体系的价值

可转债与正股高度相关，但传统研究往往将两者割裂分析。本研究构建的**转债正股联动指数体系**具有重要的实践价值：

- 可以直接计算转债相对于正股的超额收益
- 可以分析不同类型转债与正股的联动程度差异
- 为转债套利策略提供了基准工具

联动指数的核心是"可比性"：相同的样本池、相同的权重方法、相同的时间点，仅资产不同。这种设计使得差异完全来源于资产属性，而非计算方法的差异。

## 8. 延伸思考

### 8.1 指数体系的扩展方向

当前的指数体系可以从以下方向扩展：

1. **评级维度**：增加按主体评级（AAA/AA+/AA等）分类的指数
2. **期限维度**：增加按剩余期限（短/中/长期）分类的指数
3. **规模维度**：增加按未转股余额（大/中/小盘）分类的指数
4. **策略维度**：构建具体的量化策略指数（如双低增强指数）

### 8.2 实时指数的可能性

当前指数采用T+1计算模式，理论上可以实现盘中实时计算：
- 需要接入实时行情数据源
- 需要处理盘中样本的临时变动
- 需要考虑计算延迟对指数准确性的影响

### 8.3 衍生品指数的借鉴意义
