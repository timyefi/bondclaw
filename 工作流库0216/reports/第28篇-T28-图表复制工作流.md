# 第28篇：图表复制工作流 - 研究报告

## 摘要

本项目是一个图表数据提取与处理工作流，主要用于从图表图像中提取数据点，计算商品指数与金融资产的比率，并将结果写入数据库。该工作流解决了一个投研工作中常见但棘手的问题：如何将外部研究报告中的图表数据转化为可用的结构化数据。通过OpenCV图像处理技术与数据库自动化计算的双重路径设计，本项目为固收研究提供了高效的数据获取与处理能力。

---

## 一、项目背景与业务价值

### 1.1 投研痛点：图表数据的"最后一公里"

在固收研究工作中，研究分析师经常需要参考外部研究报告中的图表数据。这些图表可能来自券商研报、宏观经济分析报告或国际机构发布的数据可视化内容。然而，这些图表通常以图片形式呈现，无法直接获取其背后的原始数据。

传统的解决方案包括：
- **手动记录**：人工读取图表坐标轴数值并逐条记录，耗时且容易出错
- **联系数据源**：尝试联系原始数据提供方，但响应周期长且可能无法获取
- **放弃引用**：因无法获取数据而放弃有价值的分析维度

本项目的"图表复制工作流"正是为解决这一痛点而生，它通过计算机视觉技术自动识别图表中的曲线并提取数据点，将非结构化的图像转化为结构化的时间序列数据。

### 1.2 核心指标：全球实物资产与金融资产比率

本项目处理的核心指标是"全球实物资产比金融资产"比率，其计算公式为：

```
比率 = CRB商品指数 / (0.5 * (美国股票指数 + 盈利修正指数))
```

该指标的经济含义在于衡量实物资产（以CRB商品指数为代表）相对于金融资产（以美国股票指数和盈利修正指数的均值代表）的相对价值。在通胀预期上升时期，实物资产往往表现优于金融资产，该比率上升；而在经济衰退或通缩时期，金融资产可能更具吸引力，该比率下降。

指标的技术规格：
- **基准日期**：2020年11月12日
- **基准值**：0.09427710843373494
- **数据频率**：周度
- **数据来源**：EDB数据库edb.edbdata表

## 二、技术架构与实现方案

### 2.1 双路径数据获取设计

本项目最核心的设计特点是提供了两条数据获取路径：

**路径一：数据库直接计算**

当三个底层指标（CRB商品指数、美国股票指数、盈利修正指数）的数据都已存在于EDB数据库时，系统可以直接从数据库查询数据进行计算。这条路径完全自动化，适合日常数据更新场景。

数据处理流程：
1. 从`edb.edbdata`表查询三个TRADE_CODE的数据（日期>=2020-11-12）
2. 对数据进行日度线性插值，确保日期对齐
3. 计算比值：CRB / (0.5 * (FLOT + EARNREV))
4. 按基准日期进行标准化调整
5. 使用`resample('W').last()`转换为周度数据
6. 插入到`全球实物资产比金融资产`表

**路径二：图像数据提取**

当底层指标数据不完整，或者需要复现外部研报中的特定计算结果时，系统提供了从图表图像中提取数据的能力。这条路径需要人工交互，适合处理无法直接获取源数据的场景。

图像处理流程：
1. 加载图表图像（支持本地文件或URL）
2. 用户交互式框选数据区域（ROI选择）
3. 用户输入坐标轴范围（Y轴数值范围、X轴日期范围）
4. 用户在曲线上选取多个样本点，系统据此计算目标颜色
5. 基于颜色容差创建掩码，提取曲线像素
6. 形态学操作（开运算去除噪点、闭运算连接断点）
7. 轮廓检测并采样数据点
8. 坐标转换（像素坐标转换为日期-数值对）
9. 日期去重并输出结果

### 2.2 图像处理技术细节

图像数据提取是本项目的核心技术难点。以下是关键技术的详细说明：

**颜色识别与动态容差**

系统采用基于样本点的颜色识别策略。用户在目标曲线上选取多个点，系统计算这些点的平均颜色和标准差，然后根据标准差动态确定颜色容差：

```python
avg_sample_color = np.mean(sample_colors, axis=0)
color_std = np.std(sample_colors, axis=0)
tolerance = np.maximum(color_std * 2, [10, 10, 10])  # 最小容差为10
```

这种设计使得系统能够适应不同图表的曲线颜色变化，同时保证足够的识别精度。

**形态学操作链**

为了提高提取质量，系统设计了一套形态学操作链：

1. **开运算（2x2核）**：去除小的噪点，避免将孤立像素误认为曲线数据
2. **闭运算（3x3核）**：连接断开的线段，修复曲线中断问题
3. **水平膨胀（5x1核）**：专门针对水平方向的连接，适应图表中常见的平缓曲线
4. **腐蚀恢复**：恢复线条原始粗细，保持数据精度

**智能采样策略**

系统根据用户指定的频率参数（周度/月度/年度）智能计算目标数据点数：

```python
if frequency == 'year':
    target_points = max(5, min(years_range + 2, len(points)))
elif frequency == 'month':
    target_points = max(10, min(months_range + 2, len(points)))
else:  # week
    weeks_range = int(months_range * 4.33)
    target_points = max(30, min(weeks_range, len(points)))
```

采样时确保首尾点保留，中间区域均匀采样，保证时间序列的完整性。

### 2.3 数据处理与存储

**线性插值对齐**

三个底层指标的更新频率可能不同，存在日期不匹配问题。系统使用Pandas的线性插值功能填充缺失值：

```python
merged_df = merged_df.interpolate(method='linear')
```

**基准标准化**

为确保计算结果与原始图表完全一致，系统以2020-11-12为基准日期进行标准化：

```python
base_value = merged_df.loc[base_date, 'ratio']
adjustment_factor = target_base_value / base_value
merged_df['close'] = merged_df['ratio'] * adjustment_factor
```

**周度重采样**

使用`resample('W').last()`方法将日度数据转换为周度数据，取每周最后一个自然日的值。这种处理方式符合周度经济数据的标准惯例。

## 四、心得要点

### 心得一：双路径设计体现工程化思维

本项目最大的亮点在于提供了"数据库计算"和"图像提取"两条路径。这种设计看似增加了复杂度，实际上极大提升了系统的实用性。

在投研实践中，数据来源的多样性是常态。有时底层指标数据齐全，可以直接计算；有时只有外部研报的图表，需要从图像中提取。如果只提供单一路径，系统将无法覆盖所有实际场景。

双路径设计的另一个好处是"交叉验证"。当同时拥有底层指标数据和外部图表时，可以通过两条路径分别计算结果并进行对比，验证数据处理逻辑的正确性。这种自校验机制在数据质量管控中具有重要价值。

从工程角度看，这种设计体现了一个重要原则：**系统的价值不仅在于处理标准场景，更在于覆盖边缘场景**。投研工作中，边缘场景往往占据大量时间，能够自动化处理这些场景的工具才是真正提升效率的利器。

### 心得二：交互式设计在人机协同中找到平衡

图像数据提取模块采用了半交互式设计，需要用户手动框选数据区域、输入坐标轴范围、选取颜色样本点。这种设计在自动化精度和人工判断之间找到了平衡。

完全自动化的图像识别方案在理论上是可行的，但在实际应用中面临诸多挑战：
- 图表样式多样，坐标轴位置不固定
- 曲线颜色可能与网格线、标注等元素重叠
- 背景干扰（如水印、注释）影响识别精度

本项目选择让用户参与关键决策，由人工判断数据区域和曲线颜色，由机器执行繁琐的像素处理和坐标转换。这种分工既保证了识别准确性，又大幅减少了人工工作量。

特别值得一提的是样本点缓存机制。当用户第一次选取样本点后，系统将其保存到JSON文件中。后续处理相同图表时可以直接加载缓存，避免重复操作。这个看似简单的功能体现了对用户体验的深入思考。

### 心得三：形态学操作链展现图像处理的专业性

在图像数据提取模块中，形态学操作的设计展现了专业的图像处理知识。单一的操作往往无法解决复杂问题，需要设计操作链来逐步优化结果。

开运算去噪、闭运算连接、方向性膨胀、腐蚀恢复，这四步操作形成了一个完整的处理链条。每一步都有明确的目标，前后步骤相互配合，最终实现高质量的数据提取。

这种设计思路在固收量化研究中同样适用。数据处理往往需要多个步骤的组合，单一方法难以覆盖所有情况。理解各种方法的适用场景，设计合理的处理流程，是量化研究的重要能力。

## 五、应用场景与拓展方向

### 5.1 当前应用场景

1. **外部研报数据复现**：将券商或国际机构研报中的图表数据转化为可分析的结构化数据

## 六、风险与注意事项

### 6.1 数据精度风险

图像提取的数据精度受多种因素影响：
- 图像分辨率限制了像素级精度
- 颜色容差设置影响曲线识别完整性
- 坐标轴范围的输入误差会传播到最终结果

建议：在使用图像提取数据前，选取若干已知数据点进行验证。

### 6.2 数据库操作风险

- 基准日期数据必须存在，否则无法进行标准化
- 周度重采样可能导致近期数据缺失（周未结束）
- 重复插入需要预先清理

### 6.3 交互式操作限制

图像提取功能依赖OpenCV的GUI功能，在无图形界面的服务器环境中无法直接使用。如需在服务器端部署，需要预先配置样本点缓存或改用非交互式方案。
