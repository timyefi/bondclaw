# Contributing

感谢你参与 `bondclaw` 的开发。这个仓库希望保持清晰、可交接、可持续迭代的开源协作方式。

## 开始前

1. 先读 [README.md](README.md)。
2. 再读 [docs/README.md](docs/README.md)。
3. 需要理解项目边界时，继续读 [docs/quickstart.md](docs/quickstart.md) 和 [docs/architecture.md](docs/architecture.md)。

## 提交原则

- 一次只做一类改动。
- 文档、样例和实现尽量分开提交。
- 路径、字段名和产物名不要随手改。
- 如果改动会影响运行结果，先补说明，再补实现。

## 推荐工作流

1. 先确认改动范围。
2. 做最小范围修改。
3. 先验证文档链接和示例路径。
4. 再检查脚本、产物和说明是否一致。

## 文档风格

- 标题清晰。
- 先结论后展开。
- 尽量用真实路径和真实产物名。
- 不把草稿写成最终结论。

## 代码风格

- Python 代码保持小函数、直接逻辑。
- 中文路径、日志和 Markdown/JSON 读写统一使用 UTF-8。
- 不要把运行态数据写回能力模块目录。

## 提交信息

建议使用简短祈使句，并带上前缀，例如 `docs:`、`fix:`、`feat:`。

## 许可注意

这个仓库只允许非商业使用。提交内容应当与这一许可边界一致，不要引入和商业分发冲突的表述或依赖。
