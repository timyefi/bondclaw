# AI辅助固收研究系列报告 第1篇：T01-每日数据获取任务

> **研究领域**: 固定收益/利率债/信用债/可转债
> **研究目的**: 构建标准化的债券收益率曲线数据基础设施，解决不同期限债券的可比性问题
> **适用场景**: 日常跟踪/相对价值分析/利差研究/策略回测
> **更新频率**: 每日

---

## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

在固定收益研究中，收益率曲线是最基础也是最重要的分析工具。无论是宏观研究（观察利率期限结构变化）、信用研究（分析信用利差走势）、还是策略研究（寻找相对价值机会），都离不开收益率曲线数据的支持。

然而，实际研究中存在一个棘手的问题：市场上每一只债券的剩余期限各不相同。今天发行的3年期信用债，6个月后剩余期限变成了2.5年；而另一只1年前发行的5年期债券，现在的剩余期限是4年。如果直接比较这两只债券的收益率，得到的是一个"混合"了期限因素和信用因素的结果，无法判断哪只债券真正具有投资价值。

这个问题在编制信用债收益率曲线时尤为突出。国债、国开债等利率品种有活跃的二级市场，可以直接观察到各个关键期限的市场价格。但信用债交易分散、流动性差，很难直接从市场成交中构建连续的收益率曲线。我们需要一种方法，能够将不同期限的信用债"调整"到同一期限维度上进行比较和汇总。

### 1.2 核心研究问题

本项目要解决的核心问题是：

- **问题一：期限标准化** —— 如何将剩余期限各异的债券收益率调整到统一的期限维度，实现跨债券的可比性？
- **问题二：曲线编制** —— 如何基于标准化后的单券数据，编制出反映市场整体情况的品种收益率曲线？
- **问题三：数据处理效率** —— 如何高效地从数据源获取增量数据并完成上述计算，形成每日更新的自动化流程？

### 1.3 研究价值

这项研究的价值体现在三个层面：

**第一，方法论价值**。本项目的期限标准化方法具有通用性，可以应用于利率债、信用债、金融债、ABS等多种品种的相对价值分析，是固收量化研究的基础方法论。

**第二，数据基础设施价值**。通过每日自动化更新，为下游研究提供持续的、标准化的收益率曲线数据，避免了每次研究都需要重新处理原始数据的重复劳动。

**第三，成本控制价值**。自建数据源相比购买Wind、中债等商业数据，可以显著降低数据成本，同时保留根据研究需求定制计算的灵活性。

## 2. 研究思路与方法

### 2.1 整体研究框架

本项目的整体逻辑可以概括为"三个转换"：

```
原始数据 → 期限标准化 → 曲线聚合 → 多维分类
   ↓           ↓           ↓          ↓
不同期限   → 统一期限   → 品种曲线  → 细分指数
的债券     的收益率     利差分析    策略应用
```

**第一层转换**：从同花顺iFinD API获取债券原始数据（行情、估值、基础信息等），这是数据获取层的工作。

**第二层转换**：将每只债券的收益率"虚拟调整"到15个标准期限点。这是本项目的核心，通过Hermite插值算法实现。

**第三层转换**：将同一品种、同一期限的多只债券收益率汇总成品种曲线。采用余额加权平均方法。

对于可转债品种，还有**第四层转换**：按债性、溢价、行业三个维度进行分类，编制多维度的可转债指数。

### 2.2 核心方法论：期限标准化

期限标准化是本项目最具方法论价值的部分。其核心思想是：

**在保持债券相对价值不变的前提下，通过"借用"收益率曲线的期限结构，将不同期限的债券调整到同一期限维度。**

具体操作分三步：

**Step 1：确定"原期限"和"目标期限"**
- 原期限：债券的实际剩余期限（如2.3年）
- 目标期限：标准期限点（如1年、2年、3年等）

**Step 2：计算曲线收益率差异**
- 通过Hermite插值，分别计算原期限和目标期限对应的"曲线收益率"
- 两者之差反映的是期限利差

**Step 3：调整债券收益率**
- 标准化收益率 = 行权估值 + (目标期限曲线收益率 - 原期限曲线收益率)
- 这相当于把债券"平移"到目标期限上

这个方法的巧妙之处在于：它保持了债券与曲线之间的相对位置（即信用利差）不变，只调整了期限维度。这样，无论债券原始期限是多少，调整后都可以与同一目标期限的其他债券直接比较。

### 2.3 与传统研究方式的对比

**传统方式**：研究人员手动从Wind或中债导出债券数据，在Excel中筛选特定期限区间的债券进行平均。这种方式存在几个问题：一是期限区间划分粗糙（如把2-3年都归为一类），实际上2年和3年的利差可能高达20-30bp；二是样本不稳定，每天进入/退出该区间的债券不同，导致曲线跳跃；三是工作重复，每次分析都需要重新处理数据。

**本项目方式**：通过算法将每只债券精确调整到标准期限点，然后用余额加权编制曲线。这样得到的曲线具有三个特点：期限维度精确（可以精确到1年、2年这样的整数点）、样本连续（不会因为债券期限自然下降而出现跳跃）、过程可复现（计算逻辑明确，便于回测验证）。

**主要改进**：

- **精度提升**：从"区间平均"到"点估计"，消除了期限区间内的噪声
- **样本稳定性**：每只债券都被调整到标准期限，不会因为债券到期/新发导致样本剧烈变化
- **可扩展性**：计算逻辑标准化后，可以轻松扩展到更多品种和期限

### 3.1 数据来源说明

| 数据类型 | 来源 | 更新频率 | 用途 |
|---------|------|---------|------|
| 债券基础信息 | 同花顺iFinD API | 每日 | 债券代码、简称、发行人、评级、期限等 |
| 债券行情数据 | 同花顺iFinD API | 每日 | 收盘价、成交量、成交额等 |
| 债券估值数据 | 同花顺iFinD API | 每日 | 行权估值、推荐估值等 |
| 收益率曲线数据 | 同花顺iFinD API | 每日 | 中债各品种曲线的关键期限收益率 |
| 可转债转股数据 | 同花顺iFinD API | 每日 | 转股价、正股价格、转股溢价率等 |

### 3.2 数据处理要点

**增量更新机制**

本项目采用增量更新策略，只拉取数据库中尚未存储的新数据。具体做法是：在每次拉取前，先查询数据库中每只债券的最新日期（通过`SELECT trade_code, max(dt) FROM table GROUP BY trade_code`），然后从该日期的次日开始拉取。

这种设计的优势在于：
- 大幅减少API调用次数，避免触发限流
- 减少网络传输量，提高更新速度
- 减少数据库写入压力，延长硬件寿命

**数据存储分层**

项目使用MySQL存储结构化的债券数据，使用PostgreSQL存储时序分析结果。这种"双库"设计的原因是：

- MySQL擅长处理关系型查询，适合存储债券基础信息和行情数据
- PostgreSQL的时序扩展（如TimescaleDB）更擅长处理时间序列分析

**期限分表存储**

收益率曲线数据按期限分表存储，每个期限一张表（如`hzcurve_credit_1`存储1年期限、`hzcurve_credit_3`存储3年期限）。这种设计将"按期限过滤"的查询条件转换为"选择哪张表"，大幅提升查询效率。

## 4. 核心分析框架

### 4.1 分析维度一：Hermite插值与中债标准

**分析内容**

收益率曲线插值有多种方法：线性插值简单但不够平滑；三次样条插值平滑但可能出现震荡；Hermite插值在平滑性和稳定性之间取得了平衡。更重要的是，中债估值体系采用的是Hermite三次插值，这是国内债券市场的"事实标准"。

**算法原理**

Hermite三次插值的核心思想是：在两个相邻的关键期限点之间，构造一个三次多项式，使得该多项式在端点处既匹配函数值，又匹配导数值（斜率）。

公式为：

```
y(x) = y1*H1(t) + y2*H2(t) + d1*H3(t) + d2*H4(t)
```

其中：
- `t = (x - x1) / (x2 - x1)`，将任意x归一化到[0,1]区间
- `H1 = 1 - 3t^2 + 2t^3`，`H2 = 3t^2 - 2t^3`是基函数
- `H3 = t - 2t^2 + t^3`，`H4 = -t^2 + t^3`是斜率基函数
- `d1`和`d2`是端点处的斜率，通过相邻点计算得到

**关键发现**

- 本项目采用的Hermite插值公式与中债官方完全一致，保证了计算结果与市场定价基准的对齐
- 端点斜率的计算方法对结果影响较大，需要使用中债规定的公式：`d1 = (y2 - y11) / (2*(x2 - x11))`
- 对于1.75年这样的非整数期限，需要在标准关键期限之外额外计算

**业务含义**

采用与中债一致的插值算法，意味着：
- 计算得到的收益率与市场共识一致，便于与其他市场参与者交流
- 相对价值分析的结果更可靠，因为交易对手的定价也是基于中债估值
- 策略回测更准确，因为历史交易价格与中债估值相关

### 4.2 分析维度二：期限标准化算法

**分析内容**

期限标准化是本项目的核心创新点。它要解决的问题是：如何让一只剩余期限为2.3年的债券，与另一只剩余期限为3.7年的债券，在"2年期"这个维度上进行比较？

**算法逻辑**

期限标准化的公式是：

```
标准化收益率 = 行权估值 + (目标期限曲线收益率 - 原期限曲线收益率)
```

这个公式的含义是：

1. **行权估值**：这是债券的"真实"收益率，反映其信用风险
2. **曲线收益率之差**：这是期限带来的利差，需要被"剥离"
3. **调整后的结果**：保持了信用利差不变，只是把期限调整到了目标点

**关键发现**

- 这个方法的核心假设是：债券的信用利差在短期限内是稳定的，期限利差可以通过曲线来估计
- 当债券的剩余期限与目标期限差距较大时，调整的幅度也较大，可能引入一定误差
- 对于流动性差的债券，曲线收益率本身可能不够准确，需要谨慎使用

**业务含义**

期限标准化使得：
- 可以编制连续的信用利差曲线，而不受债券样本期限分布的影响
- 可以进行精确的相对价值分析，比如找出"同期限、同评级"中被低估的债券
- 可以构建跨期限的交易策略，比如做多短期、做空长期的曲线策略

### 4.3 分析维度三：可转债多维分类指数

**分析内容**

可转债具有"债券+期权"的双重属性，单一维度的分类无法满足不同投资策略的需求。本项目建立了三个正交的分类维度：

**维度一：债性分类**

基于平价溢价率，将可转债分为三类：
- 偏债型（< -10%）：平价溢价率为负且较大，转债价格主要受债底支撑
- 平衡型（-10% ~ 10%）：股债属性均衡，对正股和利率都敏感
- 偏股型（> 10%）：平价溢价率为正且较大，转债价格主要跟随正股

**维度二：溢价分类**

基于价格和转股溢价率的组合，分为五类：
- 双低型：价格 ≤ 110元 且 溢价率 ≤ 20%
- 低价高溢型：价格 ≤ 110元 且 溢价率 > 20%
- 高价低溢型：价格 > 110元 且 溢价率 ≤ 20%
- 双高型：价格 > 110元 且 溢价率 > 20%
- 普通型：其他情况

**维度三：行业分类**

基于申万行业分类，编制各行业的可转债指数。

**关键发现**

- "双低策略"（低价格+低溢价）是可转债量化投资的主流策略，本项目的分类体系直接支持这一策略
- 债性分类和溢价分类是正交的，可以交叉使用，比如"偏债型+双低"是一个低风险策略
- 不同行业的可转债指数表现差异显著，行业轮动是有效的策略方向

**业务含义**

多维分类指数使得：
- 投资者可以根据自己的风险偏好，快速定位目标转债类别
- 可以跟踪不同策略指数的历史表现，进行策略有效性验证
- 可以监控市场整体结构的变化，如偏股型转债占比上升可能意味着市场情绪乐观

## 5. 技术/AI辅助要点

本项目的核心技术亮点集中在算法实现层面：

**Hermite插值的精确实现**

中债官方虽然公开了Hermite插值的公式，但实际实现中有很多细节需要处理。比如端点斜率的计算需要用到"前一点"和"后一点"，在曲线端部需要特殊处理；1.75年这样的非标准期限需要额外的插值步骤。本项目完整实现了这些细节，并通过与中债估值数据的对比验证了算法的准确性。

**曲线-债券双层匹配策略**

收益率曲线计算需要将债券匹配到对应的曲线，但曲线和债券的属性维度并不完全一致。本项目采用"先粗后细"的分层匹配策略：首先按信用等级、是否城投等主要属性匹配，再按发行方式、是否永续等细分属性精确匹配。对于匹配失败的债券，使用更宽松的条件兜底。这种设计在保证准确性的同时最大化了覆盖率。

## 6. 心得要点

### 要点一：行业标准的算法选择策略

**背景**：收益率曲线插值有多种方法，线性插值、样条插值、Hermite插值各有优劣。作为研究者，很容易想要"改进"现有方法，使用更复杂的算法。

**做法**：本项目坚持采用中债官方的Hermite三次插值算法，包括插值公式和端点斜率计算方法都与中债保持一致。即使其他方法在某些场景下理论上可能更优，也坚持使用行业标准。

**洞察**：在金融工程中，算法选择不仅要考虑数学性质，更要考虑市场接受度和可比性。

中债估值是国内债券市场的定价基准，采用相同算法可以保证计算结果与市场共识一致。这对于相对价值分析至关重要——如果你的定价与市场基准不一致，你看到的"套利机会"可能只是估值差异。

这个原则可以推广到其他场景：**与市场标准对齐**。在进行任何涉及市场定价的研究时，首先要搞清楚市场的定价基准是什么，然后尽量与这个基准对齐。创新应该体现在应用层面，而不是基础定价层面。

**关联**：与"中债估值体系理解"相关，理解市场定价机制是做好固收研究的前提。

### 要点二：控制变量法在金融数据中的应用

**背景**：不同债券的收益率差异由多个因素构成：期限、信用风险、流动性、税收等。直接比较原始收益率，得到的是一个"混合"的结果。

**做法**：本项目通过期限标准化算法，将"期限因素"从收益率中剥离出来。具体做法是利用收益率曲线来估计期限利差，然后调整债券收益率，使得比较结果只反映信用风险差异。

**洞察**：这是控制变量法在金融数据处理中的典型应用。核心思想是：**在比较两个对象时，控制住某些因素，使得差异只反映我们关心的维度。**

期限标准化的公式 `标准化收益率 = 行权估值 + (目标期限曲线收益率 - 原期限曲线收益率)` 背后的逻辑是：
- 行权估值是债券的"全样本"收益率
- 曲线收益率之差反映了"纯期限因素"
- 相减后，期限因素被抵消，剩余的是信用利差

这个方法论可以推广到其他场景：比较不同评级的债券时，控制期限因素；比较不同行业的债券时，控制评级和期限因素；比较不同地区的城投债时，控制所有财务指标。

**关联**：与"因子正交化"相关，是多因子模型的基础方法论。

### 要点三：增量更新的工程实践

**背景**：债券市场每天产生大量行情数据，如果每次都全量拉取历史数据，既浪费API配额，又增加处理时间。

**做法**：在拉取数据前，先查询数据库中每只债券的最新日期，然后只从该日期的次日开始拉取。利用数据库已存储的历史数据作为"断点记录"，避免重复劳动。

**洞察**：这是状态记录与断点续传原则的应用。**通过记录上次处理的位置，实现增量处理。**

这个设计的关键洞见是：**数据库本身就是最好的状态存储**。不需要额外的状态文件或日志，直接通过SQL查询就能知道每只债券的最新日期。这种方法将数据拉取量从"全量"降为"增量"，大幅减少API调用次数和网络传输量。

这个原则可以推广到其他场景：任何需要定期更新的数据处理任务，都应该考虑增量更新。关键是找到合适的"断点标识"——在债券数据中是"最新日期"，在日志处理中可能是"最后读取的行号"，在网页爬虫中可能是"最后抓取的URL"。

**关联**：与"数据幂等性设计"相关，增量更新要考虑重复执行的正确性。

### 要点四：正交分解的多维度分析方法

**背景**：可转债具有"债券+期权"的双重属性，单一维度的分类无法满足不同投资策略的需求。价值投资者关注债性保护，趋势投资者关注股性弹性，量化投资者关注溢价水平。

**做法**：建立三个正交的分类维度——债性分类（偏债型/平衡型/偏股型）、溢价分类（双低/低价高溢/高价低溢/双高/普通型）、行业分类。每个维度独立编制指数。

**洞察**：这是正交分解方法在金融产品分析中的应用。**将复杂对象分解为多个正交维度，每个维度独立可度量。**

"正交"的含义是：改变一个维度，不会影响其他维度的分类结果。比如一只转债从"偏债型"变成"偏股型"，不会影响它在"溢价分类"中的归属。这种独立性使得：
- 投资者可以单独分析每个维度的影响
- 维度可以自由组合（如"偏债型+双低"）
- 便于构建多因子模型

这个方法论可以推广到其他具有多重属性的金融产品分析：股票的风格因子分解（价值/成长/动量）、基金的持仓特征分析（大盘/小盘、成长/价值）、银行的风险敞口分析（利率/信用/流动性）。

**关联**：与"因子正交化"直接相关，是多因子模型的基础。

### 要点五：规模加权的指数计算逻辑

**背景**：指数计算需要确定成分券的权重。等权重会高估小券的影响，价格权重与收益率概念冲突，市值权重又涉及复杂的调整。

**做法**：采用债券余额作为权重因子，计算加权平均价格。公式为：`指数价格 = sum(close * balance) / sum(balance)`。

**洞察**：这是规模加权原则的应用。**使用市场规模作为权重，使指数代表可投资的平均收益。**

债券余额代表了该债券在市场上的实际规模和流动性。大余额债券对市场的影响更大，应该在指数中占更高权重。这种方法也便于实际投资组合跟踪指数——如果投资组合的权重与指数一致，理论上可以获得与指数相同的收益。

但规模加权也有局限性：它天然偏向大券，可能忽略小券中的投资机会。在实践中，可以根据具体需求选择其他加权方式，如流动性加权（适合交易型策略）、等权重（适合选券型策略）。

**关联**：与"流动性加权变体"相关，是指数编制的通用方法论。
