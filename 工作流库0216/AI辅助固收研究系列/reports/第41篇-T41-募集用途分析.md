# AI辅助固收研究系列报告 第41篇：T41-募集用途分析

> **研究领域**: 城投债/文本分析/募集资金用途
> **研究目的**: 通过文本挖掘技术对比分析转型城投与普通城投的募集用途文本差异，揭示市场化转型对融资行为的影响，为城投债投资决策提供参考
> **适用场景**: 转型城投识别、融资行为分析、募集资金用途监测
> **更新频率**: 月度/按需

---

## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

城投债募集用途文本是了解城投平台融资意图和资金投向的重要窗口。在2025年一揽子化债政策深入推进、城投融资政策延续收紧的背景下，城投平台的募集用途呈现出新的特点。尤其是随着市场化转型进程加速，转型城投与普通城投在募集用途上是否存在系统性差异，成为投研关注的重要问题。

**市场环境变化**

2025年以来，城投债市场呈现以下特征：
- 城投债"总量稳定、净融转负"，募集资金以借新还旧为主
- 60%以上融资平台隐性债务已清零，市场化转型进程加速
- 退平台及市场化转型进程进一步提速
- 资源和政策协调能力分化平台信用

**文本分析的痛点**

传统的人工阅读募集用途存在明显局限：
- 募集用途文本数量庞大，人工阅读难以覆盖全量样本
- 文本描述格式不统一，难以进行标准化比较
- 关键信息分散在长文本中，容易被忽略
- 转型城投与普通城投的差异难以量化

**研究的必要性**

在此背景下，本项目旨在建立一套**系统化、可量化、可复制**的募集用途文本分析框架，通过文本挖掘技术对比转型城投与普通城投的募集用途差异，为投研提供以下支持：
- 识别转型城投的融资特征
- 发现募集用途的异常变化
- 建立资金用途的监测体系

### 1.2 核心研究问题

本项目主要回答以下研究问题：

- **问题一：转型城投的募集用途是否呈现更多元化的资金用途？** 传统城投以借新还旧和偿还有息债务为主，转型城投是否在项目建设、补充流动资金等方面有更多投入？
- **问题二：转型城投与普通城投在关键词使用上有何差异？** 市场化指标、创新发展、业务发展等关键词在转型城投文本中是否更频繁出现？
- **问题三：如何通过文本聚类发现募集用途的潜在模式？** 能否通过无监督学习方法识别出不同类型的募集用途文本？
- **问题四：如何建立系统化的文本分析流程？** 从数据获取到可视化输出，如何构建端到端的分析管道？

### 1.3 研究价值

本项目的核心价值体现在三个层面：

**业务价值**
- 为转型城投识别提供文本维度的参考依据
- 揭示市场化转型对融资行为的影响机制
- 建立募集用途的标准化监测体系

**技术价值**
- 展示如何将自然语言处理技术应用于债券募集用途分析
- 建立完整的文本预处理-特征提取-聚类分析流程
- 实现分析过程的可追溯、可复现

**复用价值**
- 模块化设计便于在其他文本分析场景复用
- 关键词字典可扩展到其他城投相关文本分析
- 可视化组件可快速适配到其他分析项目

### 2.2 核心方法论

#### 2.2.1 转型城投与普通城投的界定

```
普通城投 = 不在上述转型城投名单中的城投发行人
```

#### 2.2.2 文本预处理流程

针对募集用途文本的特点，设计了针对性的预处理流程：

**步骤一：空白字符处理**
```python
# 移除多余空白字符
text = re.sub(r'\s+', '', text)
```

**步骤二：特殊字符过滤**
```python
# 保留中文、英文、数字
text = re.sub(r'[^\u4e00-\u9fa5a-zA-Z0-9]', '', text)
```

**步骤三：数字处理**
```python
# 移除独立数字(保留带单位数字如'亿元')
text = re.sub(r'(?<![亿元万千百十])\d+(?![亿元万千百十])', '', text)
```

**步骤四：文本长度过滤**
```python
# 过滤过短文本(<10字符)
if len(text) < 10:
    text = None
```

**设计特点**：保留了金额数字信息（如"10亿元"），这对于金融文本分析至关重要。

**分析方法**
```python
# 关键词频率统计
def analyze_keywords(text, keyword_dict):
    frequency = {}
    for category, keywords in keyword_dict.items():
        count = sum(text.count(kw) for kw in keywords)
        frequency[category] = count
    return frequency
```

#### 2.2.4 聚类分析方法

采用TF-IDF + K-Means的经典文本聚类组合：

**TF-IDF向量化配置**
```python
TFIDF_CONFIG = {
    'max_features': 1000,  # 最大特征数
    'min_df': 2,           # 最小文档频率
    'max_df': 0.95,        # 最大文档频率(过滤过于常见的词)
    'token_pattern': r'(?u)\b\w+\b'  # 分词模式
}
```

**K-Means聚类配置**
```python
KMEANS_CONFIG = {
    'n_clusters': 5,       # 聚类数
    'random_state': 42,    # 随机种子
    'n_init': 10,          # 初始化次数
    'max_iter': 300,       # 最大迭代次数
    'algorithm': 'elkan'   # 使用elkan算法加速
}
```

**聚类可视化**
- 使用PCA将高维TF-IDF向量降维到2维
- 生成聚类散点图，直观展示文本分布

### 3.3 数据处理要点

#### 3.3.1 文本预处理规则

**空白字符处理**
- 将多个连续空白字符合并为单个空格

- 保留中文、英文、数字

- 保留带单位的金额数字（如"10亿元"、"5000万元"）

**文本长度控制**
- 过滤长度小于10字符的文本
- 这些文本通常是无效或信息量不足的

#### 3.3.2 数据质量保障

**数据完整性检查**
- 核查必填字段的缺失比例

**文本质量检查**
- 检查文本长度分布
- 识别重复文本和模板化文本

**金额数据校验**
- 验证各项用途金额之和与发行规模的关系
- 标记金额异常的样本

## 4. 核心分析框架

### 4.1 分析维度一：关键词频率对比

**分析内容**：统计14个类别的关键词在转型城投与普通城投文本中的出现频率，进行横向对比。

**分析方法**
```
对于每个类别C:
    转型城投频率 = Count(转型城投文本中C类关键词出现次数) / 转型城投文本总数
    普通城投频率 = Count(普通城投文本中C类关键词出现次数) / 普通城投文本总数
    差异 = 转型城投频率 - 普通城投频率
```

**预期发现**
- 债务管理类关键词：两类城投均高频出现，反映化债政策背景
- 市场化指标关键词：转型城投预期更高频率
- 业务发展关键词：转型城投预期更频繁提及
- 创新发展关键词：转型城投预期更高占比

**可视化输出**
- 热力图：展示关键词频率矩阵
- 对比条形图：转型城投vs普通城投的关键词频率对比
- 雷达图：展示14个维度的分布差异

### 4.2 分析维度二：资金用途结构对比

**分析内容**：对比转型城投与普通城投在各项资金用途上的金额分布。

**分析维度**
- 借新还旧占比：借新还旧金额/发行规模
- 偿还有息债务占比：偿还有息债务金额/发行规模
- 补充流动资金占比：补充流动资金金额/发行规模
- 项目建设占比：项目建设金额/发行规模
- 其他占比：其他金额/发行规模

**预期发现**
- 普通城投：借新还旧和偿还有息债务占主导
- 转型城投：补充流动资金和项目建设占比可能更高

**可视化输出**
- 饼图：转型城投与普通城投的资金用途结构对比
- 箱线图：各项用途金额的分布特征对比
- 堆叠柱状图：不同时间段的用途结构变化

### 4.3 分析维度三：文本聚类分析

**分析内容**：通过无监督学习方法发现募集用途文本的潜在模式。

**聚类流程**
```
1. 文本预处理 -> 清洗后的文本
2. TF-IDF向量化 -> 文本向量矩阵
3. K-Means聚类 -> 聚类标签
4. PCA降维 -> 2维坐标
5. 可视化 -> 聚类散点图
```

**聚类解读**
- 分析每个聚类簇的代表性关键词
- 识别聚类簇与城投类型的关联
- 发现潜在的募集用途类型

**聚类数选择**
- 当前设置：5个聚类
- 每个聚类代表一类募集用途模式
- 建议后续增加轮廓系数评估，动态确定最优K值

**可视化输出**
- 聚类散点图：展示文本在2维空间的分布
- 聚类中心词云：展示每个聚类的代表性关键词

### 4.4 分析维度四：时间趋势分析

**分析内容**：追踪转型城投与普通城投的募集用途随时间的变化趋势。

**分析指标**
- 月度发行规模趋势
- 各项用途金额的月度变化
- 关键词频率的时间变化

**预期发现**
- 转型城投发行规模的变化趋势
- 资金用途结构的时间演变
- 关键词使用的变化信号

**可视化输出**
- 月度发行规模趋势图
- 累计金额趋势图
- 时间段对比分析图

## 6. 心得要点

### 要点一：金融文本预处理的特殊性

**背景**：通用的文本预处理方法（如完全移除数字）在金融文本分析中可能造成信息损失。

**问题**：募集用途文本中包含大量金额信息（如"10亿元"、"5000万元"），这些信息对于理解资金投向至关重要。如果直接移除所有数字，将丢失关键的业务信息。

**做法**：设计针对性的数字处理逻辑：
- 保留带单位的金额数字（如"亿元"、"万元"）
- 仅移除独立的纯数字
- 这种设计在文本清洗和信息保留之间取得平衡

**洞察**：金融文本分析需要结合领域知识设计预处理规则：
- **领域理解优先**：理解文本的业务含义，识别哪些信息是关键的
- **规则针对性设计**：不是套用通用NLP流程，而是根据场景定制
- **验证迭代**：通过人工检查预处理结果，验证规则的有效性

**推广价值**：这种方法可以应用到其他金融文本分析场景，如：
- 财报文本分析：保留财务指标数字
- 新闻情感分析：保留涨跌幅信息
- 公告文本分析：保留日期和金额信息
