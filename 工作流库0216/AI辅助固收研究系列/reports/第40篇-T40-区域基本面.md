# AI辅助固收研究系列报告 第40篇：T40-区域基本面

> **研究领域**: 城投债/信用债/区域经济
> **研究目的**: 建立系统化的区域基本面量化评估框架，将评级机构的方法论转化为可执行的代码模型，为城投债投资提供标准化、可复制的区域风险识别工具
> **适用场景**: 区域横向对比、新区域准入评估、持仓区域风险监控、投资组合区域配置
> **更新频率**: 季度/按需

---

## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

区域基本面分析是城投债投研的核心环节。无论是新券发行的定价参考，还是存量持仓的风险监控，都需要对区域财政实力、债务负担、经济发展潜力等进行系统化评估。然而，传统的人工分析方式存在明显局限：

**数据处理的痛点**
- 区域财政数据来源分散、口径不一，人工收集和清洗耗时耗力
- 城投债务数据质量参差不齐，普遍存在缺失、异常问题
- 不同区域数据更新节奏不同，难以保持分析的一致性

**分析框架的痛点**
- 各分析师的评估标准和方法不统一，结论缺乏可比性
- 评级机构的方法论虽然权威，但定性因素较多，难以量化复制
- 缺乏标准化的评分体系，无法实现大规模区域的快速筛选

**效率与深度的矛盾**
- 人工深度分析耗时较长，难以覆盖全国数百个发债区域
- 粗放的筛选方式可能遗漏风险或错失机会
- 研究成果难以沉淀和复用，每次分析几乎从零开始

在这一背景下，本项目旨在建立一套**可量化、可复制、可追溯**的区域基本面评估框架，将评级机构的方法论转化为可执行的代码模型，为投研团队提供标准化的区域风险识别工具。

### 1.2 核心研究问题

本项目主要回答以下研究问题：

- **问题一：如何系统化评估区域综合财力？** 传统方法往往只看一般预算收入，忽略了政府基金收入、转移支付等因素，如何构建更科学的综合财力计算公式？
- **问题二：如何处理城投债务数据的缺失和异常？** 城投债务数据质量普遍较差，如何设计合理的数据修正逻辑，确保后续分析的可靠性？
- **问题三：如何将多维指标整合为可比较的综合评分？** 财政、经济、潜力等多个维度的指标量纲不同，如何设计评分算法实现跨区域、跨维度的可比性？
- **问题四：如何建立与行业接轨的风险评级体系？** 如何设计评级标准，使得本项目的评级结果与主流评级机构的结论具有可比性？

### 1.3 研究价值

本项目的核心价值体现在三个层面：

**业务价值**
- 为区域债投资和城投债分析提供系统化的基本面评估框架
- 支持大规模区域的快速筛选和横向对比
- 为投资决策提供量化依据，减少主观判断的偏差

**技术价值**
- 展示如何将评级机构的方法论转化为可执行的代码模型
- 建立标准化的数据处理和评分计算流程
- 实现分析过程的可追溯、可复现

**复用价值**
- 模块化设计便于在其他项目中复用核心函数
- 配置与代码分离，便于根据业务需求调整参数
- 完整的验证机制，便于结果核查和质量控制

### 2.1 整体研究框架

```
├─────────────────────────────────────────────────────────────────┤

│    数据获取 | 数据清洗 | 债务修正 | 缺失值填充                      │
```


### 2.2 核心方法论

#### 2.2.1 综合财力计算：简化加权模型

综合财力是区域偿债能力的核心指标。行业标准公式包含多个收入来源：

**行业标准公式**
```
综合财力 = 一般预算收入 + 政府基金收入 + 上级补助 + 调入资金 + 其他收入 - 上解支出
```

本项目采用简化加权模型，便于快速计算和数据获取：

**本项目公式**
```
综合财力 = 一般预算收入 * 0.7 + 政府基金收入 * 0.3
```

**权重设计依据**
- 一般预算收入稳定性更高，给予更高权重（70%）
- 政府基金收入（主要是土地出让金）波动较大，权重较低（30%）
- 这一设计在数据可得性和计算准确性之间取得平衡

**财政自给率**
```
财政自给率 = 一般预算收入 / 一般预算支出
```

**局限性说明**：未考虑上级转移支付收入，可能低估部分欠发达地区的实际财力。后续版本可扩展完善。

#### 2.2.2 三维评分体系

本项目采用"财政-经济-潜力"三维评分体系，综合评分采用加权平均：

```
综合评分 = 财政实力评分 * 40% + 经济活力评分 * 30% + 发展潜力评分 * 30%
```

**权重设计理念**
- 财政实力是核心（40%）：决定区域偿债能力的直接因素
- 经济活力是支撑（30%）：决定财政可持续性的中长期因素
- 发展潜力是增量（30%）：决定区域增长空间和投资价值

#### 2.2.3 七级风险评级体系

采用与主流评级机构接轨的七级评级标准，基于综合评分和负债率双维度判断：

| 评级 | 综合评分条件 | 负债率条件 | 评级含义 |
|------|-------------|-----------|---------|
| AAA | >=80分 | <100% | 最优质区域，财政稳健、债务可控 |
| AA+ | >=70分 | <150% | 优质区域，财政良好、债务适中 |
| AA | >=60分 | <200% | 中上区域，财政尚可、债务需关注 |
| AA- | >=50分 | <300% | 中等区域，财政承压、债务偏高 |
| A | >=40分 | 无限制 | 中下区域，财政困难、风险上升 |
| BBB | >=30分 | 无限制 | 弱资质区域，风险较高 |
| BB | <30分 | 无限制 | 高风险区域，需谨慎对待 |

**双维度判断逻辑**
- 高评级（AAA-AA）对负债率有严格要求，体现"优中选优"的理念
- 中低评级只看综合评分，负债率不再是硬约束
- 这种设计与中诚信、联合资信等机构的评级逻辑保持一致

### 2.3 与传统研究方式的对比

**传统方式**
- 各分析师自行收集数据，来源和口径不统一
- 评分标准因人而异，结论缺乏可比性
- 人工Excel计算，效率低且易出错
- 分析过程难以追溯和复现

**技术辅助后**
- 统一的数据源和清洗规则，确保口径一致
- 标准化的评分算法，结论可比较、可复现
- 自动化计算，效率大幅提升
- 代码记录分析过程，便于审计和优化

**主要改进**
- 从"个案分析"升级为"批量评估"，可同时覆盖数百个区域
- 从"主观判断"升级为"量化评分"，减少人为偏差
- 从"手工计算"升级为"自动运行"，节省90%以上的数据处理时间
- 从"黑箱输出"升级为"可追溯过程"，便于结果核查和方法改进

## 3. 数据来源与处理

### 3.1 数据来源说明

| 数据类型 | 来源 | 核心字段 | 用途 |
|---------|------|---------|------|
| 区域财政数据 | bond.fiscal_region_city_county | 一般预算收支、政府基金收支 | 财政实力评分 |
| 债务数据 | bond.fiscal_region_city_county | 城投有息债务、城投债券、地方债务 | 债务率计算 |
| 经济数据 | bond.fiscal_region_city_county | GDP、GDP增速、产业结构 | 经济活力评分 |
| 区域属性 | bond.fiscal_region_city_county | 省、市、区县、地区级别 | 区域标识和层级判断 |

### 3.2 数据处理要点

#### 3.2.1 数据清洗规则

**数值列转换**
- 识别并转换所有数值字段，处理千分位、百分号等格式
- 统一数值类型为float，便于后续计算

**日期处理**
- 将报告日期(DT)字段转换为标准日期格式
- 用于时间序列分析和数据版本管理

**地区级别推断**
- 根据省、市、区县字段的组合自动推断地区级别
- 省级：仅省字段有值
- 市级：省、市字段有值，区县为空
- 区县级：省、市、区县字段均有值

#### 3.2.2 债务数据修正逻辑

城投债务数据质量普遍较差，本项目设计了双层修正机制：

**修正规则一：债券填充**
```
if 城投有息债务余额 < 城投债券余额:
    城投有息债务余额 = 城投债券余额
```
**逻辑依据**：有息债务不应小于债券余额，若出现此情况说明数据有误，用债券余额填充更可靠。

**修正规则二：地方债务估算**
```
if 城投有息债务余额 == 0 or 城投有息债务余额 is None:
    城投有息债务余额 = 地方债务余额 * 2.0
```
**逻辑依据**：城投债务通常约为地方政府显性债务的2倍（经验值），用于估算缺失数据。

**修正记录追踪**
- 代码返回修正记录数，便于数据质量追踪
- 建议在正式分析前查看修正比例，评估数据可靠性

**局限性说明**
- 2.0倍系数是经验值，不同区域可能差异较大
- 未考虑时间序列数据的连续性
- 市本级、省级数据的层级汇总修正未在当前版本实现

#### 3.2.3 缺失值处理

**相邻日期填充**
- 对于有时间序列的数据，使用相邻日期的数值填充缺失值
- 适用于季度数据的平滑处理

**默认值设置**
- 关键指标缺失时，使用合理的默认值或标记为无效
- 避免因缺失值导致后续计算错误

### 3.3 数据质量保障

**数据完整性检查**
- 核查必填字段的缺失比例
- 统计各区域的数据覆盖度

**异常值检测**
- 识别超出合理范围的数值（如负债率>500%）
- 标记并人工复核异常数据

**交叉验证**
- 对比不同来源的同一指标
- 验证计算结果的合理性

## 4. 核心分析框架

### 4.1 分析维度一：财政实力评分

**分析内容**：从财政自给率和负债率两个角度评估区域财政实力。

**评分算法**
```
财政实力评分 = 财政自给率贡献 + 负债率贡献
财政自给率贡献 = 财政自给率 * 50
负债率贡献 = (1 - 负债率/3.0) * 50
```

**设计逻辑**
- 财政自给率和负债率各占50%权重
- 财政自给率80%得40分，体现对财政独立性的重视
- 负债率300%时贡献为0，体现对债务压力的关注
- 评分上限100分，下限0分

**业务含义**
- 高分区域：财政自给能力强、债务负担轻，偿债能力有保障
- 中等区域：财政基本平衡，但债务压力需要关注
- 低分区域：财政依赖外部支持，债务负担较重，风险上升

### 4.2 分析维度二：经济活力评分

**分析内容**：从GDP增速、产业升级、投资活跃度三个角度评估区域经济活力。

**评分算法**
```
GDP增速得分 = min(GDP增速 * 50, 100) * 0.5
产业升级得分 = min(三产占比 / 0.6, 1.0) * 100 * 0.2
投资活跃得分 = min(投资活跃度 * 60, 60) * 0.3
经济活力评分 = min(三项之和, 100)
```

**指标说明**
- GDP增速：反映经济增长动能，6%增速可得约300分（被截断至100分）
- 产业升级度：三产占比/60%，目标值60%，体现经济结构优化
- 投资活跃度：固投/GDP，反映投资拉动能力

**业务含义**
- 高分区域：经济增长强劲，产业结构优化，投资活跃
- 中等区域：经济增速平稳，但结构性问题值得关注
- 低分区域：经济增长乏力，产业转型压力大

### 4.3 分析维度三：发展潜力评分

**分析内容**：从人口红利、城镇化红利、增长惯性三个角度评估区域发展潜力。

**评分算法**
```
发展潜力评分 = 人口红利贡献 + 城镇化红利贡献 + 增长惯性贡献
人口红利贡献 = demographic_bonus * 20
城镇化红利贡献 = urbanization_bonus * 30
增长惯性贡献 = growth_inertia * 30
```

**指标说明**
- 人口红利：人口结构年轻化程度、人口净流入情况
- 城镇化红利：城镇化率提升空间、城市基础设施建设潜力
- 增长惯性：历史经济增速的延续性

**注意事项**
- 人口红利和城镇化红利系数需要外部输入，代码中未提供具体计算方法
- 实际使用时需要补充这些指标的数据来源

**业务含义**
- 高分区域：人口吸引力强，城镇化空间大，增长势头好
- 中等区域：发展潜力一般，需要政策支持
- 低分区域：人口流失，城镇化饱和，增长乏力

### 4.4 综合评分与风险评级

**综合评分计算**
```
综合评分 = 财政实力评分 * 0.4 + 经济活力评分 * 0.3 + 发展潜力评分 * 0.3
```

**风险评级判断**
基于综合评分和负债率双维度，按照评级矩阵确定最终评级。评级结果可用于：
- 投资组合区域配置决策
- 新发债券定价参考
- 存量持仓风险监控

**区域对比分析**
- 生成各维度的排名（评分排名、财政排名、经济排名、潜力排名、综合排名）
- 支持同级别区域的横向对比
- 便于识别各维度的相对优势和劣势

## 6. 心得要点

### 要点一：评级方法论的可量化转化

**背景**：评级机构（如联合资信、中诚信、远东资信）的方法论通常以定性描述为主，如何将其转化为可执行的代码模型？

**做法**：采用"框架借鉴+规则简化"的方法：
1. 梳理评级机构的核心评价维度（财政、经济、债务）
2. 将定性描述转化为定量规则（如负债率<100%对应AAA级）
3. 设计权重体系，保持与行业主流的一致性
4. 用代码实现评分和评级逻辑

**洞察**：评级方法论的量化转化需要把握两个平衡：
- **准确性与简洁性的平衡**：过于复杂的模型难以维护，过于简单的模型准确性不足。本项目采用"三维评分+双维度评级"的简化框架，在保持核心逻辑的同时降低了复杂度。
- **标准化与灵活性的平衡**：固定权重和阈值便于比较和复现，但可能忽视区域异质性。后续可通过配置化设计和动态权重调整来解决这一问题。

这种方法论可以推广到其他评级场景，如产业债评级、银行评级等。核心步骤是：**理解原方法论的底层逻辑 -> 识别可量化的关键指标 -> 设计评分规则和阈值 -> 用代码实现并验证**。

**关联**：与"工程化思维"、"模块化设计"等原则相通——将复杂的定性判断分解为可处理的量化模块。

### 要点二：数据质量修正的必要性

**背景**：城投债务数据质量普遍较差，存在缺失、异常、不一致等问题。直接使用原始数据进行分析，可能导致评分失真甚至计算错误。

**做法**：设计双层修正机制：
1. **逻辑校验修正**：当债务余额小于债券余额时，用债券余额填充——基于"有息债务不应小于债券余额"的逻辑
2. **缺失值估算**：当债务数据缺失时，用地方债务余额的2倍估算——基于"城投债务约为显性债务2倍"的经验

**洞察**：金融数据分析中，数据质量处理往往占据60%以上的工作量。数据修正的核心原则是：
- **有据可依**：修正规则要有明确的业务逻辑支撑，而非随意填充
- **可追溯**：记录修正过程，便于后续审计和优化
- **适度保守**：宁可标注为缺失，也不要过度估算

本项目采用的经验系数（2.0倍）需要结合区域特征动态调整。后续可考虑：
- 建立历史数据回溯机制，用时间序列数据优化估算
- 对接多个数据源交叉验证，提高数据准确性
- 增加数据质量报告，统计修正比例和影响范围

**关联**：与"数据驱动验证"、"迭代优化"等原则相通——数据修正规则需要在实践中不断优化。

### 要点三：双维度评级的设计智慧

**背景**：传统的单维度评级（仅看综合评分）可能遗漏重要风险信号。例如，某区域经济增速很高、综合评分优秀，但负债率已经超过300%，实际上风险很高。

**做法**：采用综合评分和负债率的双维度评级：
- 高评级（AAA-AA）对负债率有严格要求
- 中低评级只看综合评分，负债率不再是硬约束

**洞察**：双维度评级体现了"优中选优"的理念：
- **高评级门槛更高**：不仅要综合评分达标，负债率也要在安全范围内
- **低评级宽容度更高**：综合评分不达标时，不再用负债率"落井下石"
- **风险敏感度递减**：评级越高，对风险的容忍度越低

这种设计思想可以推广到其他风险管理场景：
- **信用审批**：优质客户审批条件更宽松，高风险客户审批条件更严格
- **投资决策**：核心持仓的风控要求更高，卫星持仓可以适当放宽
- **合规管理**：重点领域的合规标准更高，一般领域可以适度灵活

**关联**：与"风险分层管理"、"差异化风控"等原则相通——根据风险等级设置差异化的管控标准。
