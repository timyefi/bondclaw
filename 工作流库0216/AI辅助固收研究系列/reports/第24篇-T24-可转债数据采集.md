## 1. 项目背景与目标

### 1.1 为什么需要可转债数据采集系统？

可转换债券是中国资本市场中一类独特的混合型金融产品，兼具债券的防御属性和股票的进攻属性。随着可转债市场规模不断扩大，投研工作对高质量、高时效性数据的需求日益迫切。

然而，构建一套完整的可转债数据采集系统面临诸多挑战：

- **数据源分散**: 可转债相关数据涉及基础信息、行情数据、估值指标等多个维度，需要从专业数据终端获取
- **指标复杂**: 可转债特有的转股溢价率、纯债溢价率、到期收益率等指标计算复杂
- **样本动态变化**: 可转债市场样本数量有限且持续变化（新发行、提前赎回、转股退出）
- **指数体系缺乏**: 市场上缺乏从不同维度刻画可转债市场的细分指数体系

### 1.2 项目核心目标

本项目旨在构建一套完整的可转债数据采集与指数计算系统，实现以下目标：

1. **自动化数据采集**: 基于同花顺iFinD接口实现可转债数据的自动化采集与更新
2. **高质量数据存储**: 设计合理的数据库表结构，确保数据的完整性和一致性
3. **多维度指数计算**: 构建覆盖行业、债性、溢价等多维度的可转债指数体系
4. **转债正股联动**: 同步计算可转债及对应正股的联动指数，支持对比分析

### 1.3 项目价值定位

| 维度 | 具体价值 |
|------|----------|
| 业务层面 | 为可转债投资提供完整的数据支持和指数跟踪工具 |
| 方法论层面 | 建立季度样本池+多级放宽的动态管理机制 |
| 技术层面 | 提供可复用的数据采集和指数计算代码架构 |

### 3.1 同花顺iFinD接口对接

数据采集模块基于同花顺iFinD数据接口实现，这是国内专业的金融数据终端，提供了丰富的可转债数据。

**连接管理实现**:

```python
class THSBondDataCollector:
    def __init__(self):
        self.login_status = False
        self._trading_days_cache = set()
        self._load_trading_days_cache()
        self.connect()

    def connect(self):
        """连接同花顺数据接口，支持重试机制"""
        retry_count = 0
        while not self.login_status and retry_count < 3:
            login_result = THS_iFinDLogin(THS_CONFIG['username'], THS_CONFIG['password'])
            if login_result == 0:
                self.login_status = True
                break
            retry_count += 1
            time.sleep(5)
```

### 4.2 连接池设计

系统采用单例模式实现数据库连接池，支持主备数据库自动切换：

```python
class DatabaseConnectionPool:
    _instance = None
    _lock = threading.Lock()

    def __new__(cls):
        with cls._lock:
            if cls._instance is None:
                cls._instance = super(DatabaseConnectionPool, cls).__new__(cls)
            return cls._instance

    def _create_connection(self, use_backup: bool = False):
        """创建新的数据库连接，支持主备切换"""
        db_config = self.config['backup' if use_backup else 'primary'].copy()
        for attempt in range(retry_count):
            try:
                return pymysql.connect(**db_config)
            except Exception as e:
                if attempt == retry_count - 1 and not use_backup:
                    return self._create_connection(use_backup=True)
                time.sleep(retry_delay)
```

### 4.3 上下文管理器封装

连接池通过上下文管理器提供简洁的使用接口：

```python
class DatabaseConnection:
    """数据库连接上下文管理器"""
    def __enter__(self) -> pymysql.Connection:
        self.conn = self.pool.get_connection()
        return self.conn

    def __exit__(self, exc_type, exc_val, exc_tb):
        if self.conn:
            self.pool.return_connection(self.conn)

# 使用示例
with DatabaseConnection() as conn:
    cursor = conn.cursor()
    cursor.execute("SELECT * FROM marketinfo_equity1")
```

| 偏股型 | > 10% | 股性突出，弹性较大 |

| 分类 | 价格条件 | 转股溢价率条件 | 策略特征 |
|------|---------|---------------|---------|
| 双低 | <= 110 | <= 20% | 经典双低策略标的 |
| 低价高溢 | <= 110 | > 20% | 低价但估值偏高 |
| 高价低溢 | > 110 | <= 20% | 高价但估值合理 |
| 双高 | > 110 | > 20% | 高价且估值偏高 |
| 普通型 | 不满足以上条件 | - | 其他转债 |

### 5.2 季度样本筛选机制

```python
def _get_quarterly_candidates(self, df: pd.DataFrame, quarter_start: str, quarter_end: str):

    # 获取季度数据
```

### 5.4 加权指数计算

指数采用未转股余额加权方式计算：

```python
def _calculate_index_batch(self, market_data: pd.DataFrame, group_data: pd.DataFrame, group_by: str):
    """批量计算指数"""
    # 合并数据
    merged_data = pd.merge(market_data, group_data, on=['dt', 'trade_code'], how='inner')

    # 计算加权价格
    merged_data['weighted_close'] = merged_data['close'] * merged_data['balance']

    # 按日期和分类聚合
    result = merged_data.groupby(['dt', group_by]).agg({
        'weighted_close': 'sum',
        'balance': 'sum',
        'trade_code': 'count',
        'turnover_rate': 'mean'
    }).reset_index()

    # 计算加权平均价格
    result['close'] = result['weighted_close'] / result['balance']

    return result
```

```python
def _save_index_market_data(self, index_data: pd.DataFrame, index_code: str):
    """批量保存指数行情数据"""
    values = [
        (index_code, row['dt'], float(row['close']), float(row['total_balance']),
         int(row['component_count']), float(row['turnover_rate']))
        for _, row in index_data.iterrows()
    ]

        cursor = conn.cursor()
        cursor.executemany("""
            INSERT INTO bond.marketinfo_ebindex
            (trade_code, dt, close, total_balance, component_count, turnover_rate)
            VALUES (%s, %s, %s, %s, %s, %s)
            ON DUPLICATE KEY UPDATE
            close = VALUES(close), total_balance = VALUES(total_balance)
        """, values)
        conn.commit()
```

```python
def _calculate_moving_averages(self, df: pd.DataFrame):
    """向量化计算移动平均"""
    metrics = ['close', 'ths_conversion_premium_rate_cbond',
              'ths_pure_bond_premium_rate_cbond', 'turnover_rate']

    # 一次性计算所有指标的移动平均

    )
```

## 7. 心得要点总结

### 7.1 季度样本筛选机制的巧妙设计

项目采用季度为单位进行样本筛选，在季度首末日都要求样本有效（未转股余额>0），并按季度平均未转股余额排序选样。这种设计确保了样本的连续性和代表性，同时避免了频繁调仓带来的交易成本和指数失真。季度调仓频率平衡了跟踪精度和维护成本，是指数编制的成熟实践。

**关键实现要点**：
- 季度首末日双重有效性检验
- 平均未转股余额排序选样
- 季度内样本池保持稳定

### 7.2 五步渐进放宽的样本补充策略

当标准条件下样本数量不足时，系统设计了五步渐进放宽策略：从溢价率±2%逐步放宽到完全放开条件。这种渐进式策略既保证了指数的代表性，又确保了在市场极端情况下仍能维持指数计算，体现了工程的鲁棒性设计。

**策略步骤**：
1. 放宽溢价率条件±2%
2. 放宽溢价率条件±5%
3. 降低换手率要求到0.3%
4. 进一步降低换手率到0.1%
5. 完全放开条件

### 7.3 转债正股联动指数的创新价值

项目不仅计算可转债指数，还同时计算对应正股的联动指数。这种设计使得研究者可以直接对比转债与正股的表现差异，分析转债相对正股的超额收益来源，为转债投资策略研究提供了独特的数据基础。正股指数采用转债样本对应的正股构成，确保了两类指数的可比性。

**业务价值**：
- 支持转债-正股套利策略研究
- 分析转债相对正股的超额收益
- 研究转债定价偏离程度

### 7.4 连接池上下文管理器的优雅实现

DatabaseConnection类实现了上下文管理器协议（__enter__/__exit__），使得数据库连接可以像文件操作一样使用with语句自动管理生命周期。这种设计既保证了连接的正确释放，又使代码更加简洁易读。结合单例模式的连接池，实现了高效且安全的数据库访问。

**代码优雅性**：
- 自动连接获取和释放
- 异常安全的资源管理
- 简洁的调用方式

### 7.5 多维度分类指数体系的完整性

项目构建了完整的多维度分类指数体系：按债性分类反映转债的股债属性；按溢价分类反映估值水平；按行业分类反映行业特征。这种多维分类体系为投资者提供了丰富的分析维度，可以针对不同投资偏好选择相应的指数跟踪。

**分类维度**：
- 债性分类：偏债型/平衡型/偏股型
- 溢价分类：双低/低价高溢/高价低溢/双高/普通型
- 行业分类：申万一级行业

### 7.6 ON DUPLICATE KEY UPDATE的幂等写入

所有数据库写入都采用INSERT ... ON DUPLICATE KEY UPDATE语句，这种幂等设计使得数据采集任务可以安全重复执行。无论是补采历史数据还是修正错误数据，都不会产生重复记录或主键冲突。这是金融时间序列数据入库的标准最佳实践，特别适合需要频繁更新的数据场景。

**技术优势**：
- 支持数据补采和修正
- 避免主键冲突
- 提高系统可靠性
