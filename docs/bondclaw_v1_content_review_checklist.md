# BondClaw V1 发布前内容审校清单

本文档用于发版前的内容与体验复核。  
它关注的是“文字是否顺、标签是否对、入口是否自然”，避免功能已经有了，但内容口径还没收干净。

## 审校目标

- 模板中心的 20 个 starter 是否可以直接拿来使用。
- 信息中心的 20 个案例是否能形成完整的研究链路。
- 联系方式、设置、首页的文案是否统一。
- 角色名、主题名、动作名是否足够直白。

## 审校范围

### 1. 首页

- 首屏焦点卡是否清楚。
- 产品状态条是否能让用户一眼知道当前能力。
- 模板中心、信息中心、联系方式、设置四个入口是否容易理解。
- 当前上下文是否真的反映筛选状态。

### 2. 模板中心

- 7 个角色名称是否清楚。
- 20 个 starter 的名称是否符合真实工作场景。
- 默认角色、默认模板、默认供应商、默认技能是否合理。
- 每个模板的输出格式是否适合直接执行。
- QA 清单是否足够具体。
- 角色清单和 prompt 预览是否能帮助用户快速判断。

### 3. 信息中心

- 主题总览是否覆盖宏观、利率、信用、转债、多资产、固收基金经理、固收交易员。
- 来源卡是否能让用户理解每组来源的覆盖范围。
- 案例高亮是否能准确指向对应 Prompt。
- 当前案例的证据卡、Prompt 卡、操作卡是否都能对上。
- “按主题筛选”“进入来源组”“查看对应 Prompt”“查看案例”这些动作是否自然。

### 4. 联系方式

- 页面名是否统一显示为“联系方式”。
- 队列状态是否易懂。
- 投递顺序是否清楚。
- 本地队列、待重试、提交量这几个词是否能被客户理解。
- 表单和回执字段是否与定版规范一致。

### 5. 设置

- 品牌、升级、在线入口、说明卡的口径是否统一。
- 官网、支持页、文档基址、更新说明是否容易找到。
- 版本、渠道、启用开关是否表达清楚。
- 设置页是否足够像一个控制台，而不是信息堆。

## 审校方法

建议按以下顺序人工浏览：

1. 先看首页。
2. 再进模板中心。
3. 再进信息中心。
4. 再看联系方式。
5. 最后看设置。

每一页都检查：

- 标题是否自然。
- 说明是否过长。
- 按钮文案是否够直接。
- 是否存在同义词混乱。
- 是否有工程术语残留。

## 需要重点关注的词

- 模板中心
- 信息中心
- 联系方式
- 设置
- 角色
- 模板
- 案例
- 来源卡
- 证据
- Prompt
- 升级

## 通过标准

- 关键页面都能让非开发人员看懂。
- 用户能顺着首页快速理解四个主入口。
- 没有明显的工程展示态文案残留。
- 所有关键动作词都足够直接。
