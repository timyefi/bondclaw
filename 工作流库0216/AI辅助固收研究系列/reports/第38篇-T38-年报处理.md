## 摘要

企业年报是信用债研究的核心信息来源，其中包含的股东分红、资本公积变动等关键财务信息对投资决策具有重要参考价值。本文详细介绍了一套基于Python的年报自动化处理系统，该系统实现了从企业预警通网站爬取年报PDF、OCR文本提取、到使用通义千问大模型智能提取财务信息的完整流程。本项目的核心价值在于将原本需要研究人员逐份阅读年报、手工摘录信息的工作模式转变为自动化流水线处理，大幅提升了年报信息获取的效率和覆盖面。本文将深入分析系统的Pipeline架构设计、核心模块实现、大模型Prompt工程、以及数据处理过程中的经验总结，为同类金融文本处理项目提供参考。

## 第一章 项目概述

### 1.1 项目背景

在固定收益研究中，企业年报是了解发行人财务状况、经营成果和未来发展规划的重要信息来源。特别是对于城投债、产业债等信用债品种，年报中的股东分红信息、资本公积变动情况等数据，对判断发行人的政府支持力度、财务健康程度具有重要的参考价值。

然而，传统的年报信息获取方式存在以下痛点：

**人工处理效率低下**：一份年报通常有100-300页，人工阅读和摘录关键信息需要30分钟到1小时，对于存续的数千只债券对应的发行人，完全依赖人工处理是不现实的。

**信息提取标准不一**：不同研究人员对同一信息的理解可能存在偏差，导致提取结果的一致性不足，影响后续的横向比较分析。

**时效性难以保证**：年报披露期集中在每年1-4月，大量年报集中发布，人工处理难以在短时间内完成所有目标企业的信息收集。

**关键信息分散**：年报中的股东分红、资本公积等信息通常分散在财务报表附注的不同位置，需要逐页查找，容易遗漏。

### 1.2 核心功能

本项目旨在解决上述问题，构建一套年报自动化处理系统，核心功能包括：

**年报PDF自动爬取**：通过企业预警通API搜索目标公司，获取年报下载链接，自动下载PDF文件到本地，支持断点续传和文件名规范化。

**OCR文本批量提取**：使用PyMuPDF库对PDF文件进行文本提取，将非结构化的PDF文档转换为结构化的文本数据，支持增量处理避免重复操作。

**大模型智能提取**：调用通义千问qwen-long模型，通过精心设计的Prompt指令从OCR文本中提取股东分红和资本公积增加等关键财务信息，输出结构化的JSON数据。

**数据库持久化存储**：将提取结果存储到MySQL数据库，支持后续的数据查询和分析，实现数据的闭环管理。

### 1.3 技术流程概览

系统采用Pipeline模式设计，完整的数据处理流程如下：

| 阶段 | 功能 | 输入 | 输出 | 核心技术 |
|------|------|------|------|----------|
| Stage 1 | 数据源查询 | MySQL数据库 | 公司列表(trade_code, 公司名称) | SQLAlchemy |
| Stage 2 | 公司搜索 | 公司名称 | 企业预警通公司代码 | API POST请求 |
| Stage 3 | 年报URL获取 | 公司代码 + 年份 | 年报PDF下载链接 | 正则匹配过滤 |
| Stage 4 | PDF下载 | 下载URL | 本地PDF文件 | HTTP GET |
| Stage 5 | OCR文本提取 | PDF文件 | 文本文件(.txt) | PyMuPDF |
| Stage 6 | 大模型提取 | 文本文件 | 结构化财务数据(JSON) | qwen-long API |
| Stage 7 | 数据存储 | 结构化数据 | MySQL记录 | SQL INSERT/UPDATE |

这种分阶段设计使得各模块可以独立开发、测试和维护，同时也支持灵活配置和按需执行特定阶段。

## 第二章 系统架构设计

### 2.1 整体架构模式

系统采用"Pipeline模式 + 模块化类封装"的架构设计，将年报处理的复杂流程拆分为独立的功能模块，每个模块由专门的类负责，通过DatabaseManager和配置文件进行模块间的松耦合连接。

**架构特点**：

- **低耦合**：各模块通过DatabaseManager和配置文件连接，可独立测试和复用
- **高内聚**：每个类职责单一，功能聚焦
- **可配置**：所有参数集中在config.py，敏感信息通过环境变量管理
- **可扩展**：支持新增数据源、新增提取目标等扩展需求

### 2.2 核心类设计

DatabaseManager负责统一管理数据库连接，支持双数据库（yq数据库和timinfo数据库）的访问。采用引擎池模式，使用NullPool避免连接泄露问题。

**设计亮点**：
- 使用pool_recycle参数防止连接超时

#### 2.2.2 AnnualReportCrawler - 年报爬取模块

AnnualReportCrawler封装了企业预警通网站的爬取逻辑，实现年报PDF的自动下载。

## 第三章 核心功能详解

### 3.1 年报爬取功能

#### 3.1.1 企业预警通API调用

企业预警通（qyyjt.cn）是一个企业信息查询平台，提供了丰富的企业年报资源。系统通过模拟API请求获取年报下载链接。

**API调用流程**：

1. **公司搜索**：通过公司名称模糊搜索，获取企业预警通公司代码
2. **年报列表获取**：根据公司代码获取该公司的年报列表
3. **年报URL筛选**：通过正则匹配过滤出目标年度的年报PDF链接

**正则过滤规则**：

年报文件名通常包含年份和报告类型，系统通过正则表达式过滤：

- 包含模式：`\d{4}.*年报`、`年度报告`、`annual report`
- 排除模式：`摘要`、`电子书`、`h5`、`仅供查阅`

这种组合过滤策略能够有效筛选出正式的年报PDF文件，排除摘要版和其他无关文件。

#### 3.1.2 文件下载与存储

**断点续传机制**：

系统在下载前会检查目标文件是否已存在，已存在的文件自动跳过，避免重复下载：

```python
def download_pdf(self, url, filepath):
    if os.path.exists(filepath):
        print(f"文件已存在，跳过: {filepath}")
        return True
    # 下载逻辑...
```

**文件名规范化**：

企业年报的原始文件名可能包含特殊字符（如斜杠、冒号等），这些字符在Windows文件系统中是不允许的。sanitize_filename函数会过滤这些危险字符：

```python
def sanitize_filename(filename):
    # 移除Windows文件系统不允许的字符
    invalid_chars = '<>:"/\\|?*'
    for char in invalid_chars:
        filename = filename.replace(char, '_')
    return filename
```

**随机延迟防封禁**：

为避免频繁请求触发API限流，系统在每次API调用后添加随机延迟：

```python

import time

time.sleep(random.uniform(0.5, 1))
```

### 3.2 OCR文本提取功能

#### 3.2.1 PyMuPDF文本提取

PyMuPDF（fitz）是一个高性能的PDF处理库，支持快速提取PDF中的文本内容。

**提取流程**：

1. 打开PDF文件
2. 逐页提取文本内容
3. 合并所有页面的文本
4. 保存为.txt文件

**核心代码结构**：

```python
def extract_text_with_pymupdf(pdf_path):
    doc = fitz.open(pdf_path)
    text_content = []
    for page_num in range(len(doc)):
        page = doc[page_num]
        text = page.get_text()
        text_content.append(f"=== 第{page_num + 1}页 ===\n{text}")
    return "\n".join(text_content)
```

系统通过check_ocr_status函数检查PDF文件是否已经完成OCR处理，已处理的文件自动跳过：

### 3.3 大模型提取功能

#### 3.3.1 通义千问qwen-long模型

通义千问是阿里云推出的大语言模型，其中qwen-long版本专门针对长文档处理进行了优化：

| 特性 | 说明 |
|------|------|
| 最大上下文长度 | 10M tokens |
| 支持文件上传 | 直接上传文档进行处理 |
| 结构化输出 | 支持指定JSON格式输出 |
| 成本效益 | 相比GPT-4更适合批量处理 |

对于年报这类动辄数万字的文档，qwen-long的长上下文能力是关键选择因素。

#### 3.3.2 文件上传与处理

**上传流程**：

1. 调用文件上传API，将文本文件上传到模型服务
2. 获取文件ID
3. 等待文件解析完成（通常需要几秒到几十秒）

**处理等待机制**：

```python
def wait_for_processing(file_id, max_wait=60):
```

#### 3.3.3 Prompt工程设计

Prompt设计是大模型提取质量的关键。本项目针对股东分红和资本公积增加两个提取目标，设计了详细的Prompt指令。

**股东分红提取Prompt要点**：

**包含内容**：
- 现金股利
- 永续债利息
- 可续期公司债利息
- 分配红利、分配现金股利、分配普通股股利、分配优先股股利
- 上缴收入、上缴利润

**排除内容**：
- 股票股利（非现金分红）

**资本公积增加提取Prompt要点**：

**包含内容**：
- 资本公积增加、资本公积增长

**排除内容**：
- 盈余公积、股本、注册资本
- 其他综合收益、其他收益
- 股权认缴、增资扩股、递延收益

**原因分类**：
- 政府现金拨款
- 股权无偿划转
- 资产评估增值
- 项目移交
- 其他

**输出格式要求**：

```
年份|项目|增加原因|金额|原始金额|原始金额单位
```

这种结构化的输出格式便于后续解析和存储。

### 3.4 数据存储功能

#### 3.4.1 金额单位转换

年报中的金额可能以元、万元、亿元等不同单位表示，系统需要统一转换为亿元便于后续分析：

```python
def convert_amount_to_yi(amount, unit):
    unit_factors = {
        '元': 1e-8,
        '万元': 1e-4,
        '亿元': 1,
        '万': 1e-4,
        '亿': 1
    }
    factor = unit_factors.get(unit, 1)
    return float(amount) * factor
```
