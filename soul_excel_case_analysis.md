# Soul Excel 案例分析基线（2026-03）

## 1. 样本范围

本次仅复盘仓库内已有 4 个 Excel 案例：

| 案例 | 类型 | 主要特征 |
|------|------|----------|
| 恒隆地产 | 港股商业地产 | 投资物业、公允价值、债券与银行借款并存 |
| 碧桂园 | 民营房企 | 风险暴露明显，受限资金与抵押重要 |
| 杭海新城控股 | 城投 LGFV | 平台属性、政府补助、对外担保重要 |
| 中海地产 | 央企房企 | 三表摘要、债务结构、同业对比较完整 |

## 2. 直接观察

### 2.1 跨案例重复出现的模块

| 模块 | 出现情况 | 结论 |
|------|----------|------|
| 概览/封面/基本信息 | 4/4（名称不同） | 必须收敛成统一入口页 |
| 有息债务分析 | 4/4（恒隆命名略不同） | 应作为固定骨架 |
| 债券明细 | 3/4 | 应作为高频可选模块 |
| 财务指标/KPI | 3/4 | 应固定为摘要面板，而不是完全自由发挥 |
| 三大报表摘要 | 2/4 | 重要，但不是所有案例都必须完整展开 |
| 行业专题 | 4/4（主题不同） | 应模块化，不适合硬编码进固定结构 |
| 同业对比 | 1/4 | 适合作为按需模块，不宜默认强制 |

### 2.2 当前案例的共性问题

1. 工作簿结构命名不统一，难以直接横向比较。
2. 内部用途内容混入对外成品，例如“技能进化记录”。
3. 所有案例都没有单元格批注和证据索引，外部可追溯性不足。
4. 所有案例都没有公式层，很多指标只是静态数值，不利于复核。
5. 所有案例都没有冻结窗格，交付体验一般。
6. 当前样式整体可读，但离“标准成品模板”仍有差距。

## 3. 从案例反推的结构结论

### 3.1 不适合继续沿用“固定 8 Sheet”思路

原因不是 Sheet 数量多，而是不同案例真正稳定的内容并没有覆盖那 8 个固定页面。硬定死会导致两个问题：

- 一些案例会被迫塞入不适用的页面，结构显得空或假。
- 一些行业特征又只能通过临时加表解决，导致规范形同虚设。

### 3.2 更适合的方式是“骨架 + 模块”

基于案例，当前最稳的 Soul 骨架应是：

1. `00_overview`
2. `01_kpi_dashboard`
3. `02_financial_summary`
4. `03_debt_profile`
5. `04_liquidity_and_covenants`
6. `99_evidence_index`

然后再按案例启用：

- `05_bond_detail`
- `06_rating_view`
- `07_peer_comparison`
- `08_topic_*`
- `90_full_statements`

## 4. 专题模块的初步字典

从现有案例已能看出至少 4 类专题：

| 专题模块 | 来自案例 | 为什么要独立 |
|----------|----------|--------------|
| `topic_investment_property` | 恒隆地产 | 港股商业地产的估值与公允价值变化非常关键 |
| `topic_restricted_assets` | 碧桂园 | 受限资金、抵押物对流动性判断影响很大 |
| `topic_lgfv_features` | 杭海新城控股 | 城投平台不能只用地产企业框架分析 |
| `topic_inventory_quality` | 中海地产 | 房企资产质量判断不能只看债务，还要看存货结构 |

这说明“开放性”应该体现在专题模块，而不是把所有字段都塞进主表。

## 5. 对 Soul 的启示

1. Soul 的第一优先级不是“表越多越全”，而是让外部读者先抓住信用主线。
2. 债务、流动性、KPI 摘要必须高度标准化。
3. 三表、评级、同业对比等内容应视案例和读者目的决定是否展开。
4. 专题模块应该成为以后迭代的主要战场，核心骨架保持稳定。
5. 对外交付工作簿必须剔除内部知识进化痕迹。

## 6. 建议的迭代方法

### 第一轮

- 先用现有 4 个案例确定骨架，不再继续争论“标准版一定要有几张表”。

### 第二轮

- 新增 3-5 个案例后，重点观察：
  - 哪些专题持续重复出现
  - 哪些骨架字段仍然不够用
  - 哪些页面其实很少被用到

### 第三轮

- 再决定 Soul v1.2 是否需要把某些高频专题升级为标准模块。

## 7. 可复用工具

已补充 `analyze_excel_cases.py`，用于持续盘点 `cases/` 下工作簿的：

- Sheet 结构
- 公式/批注/合并单元格/冻结窗格
- 样式复杂度
- 首屏样例行

后续新增案例后，可直接重新运行该脚本，避免每次人工统计。
