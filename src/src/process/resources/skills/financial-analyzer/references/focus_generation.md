# Focus Generation

## 原则

动态重点不是固定专题列表，而是本次案例中最值得优先展开的议题。

## 输入

重点生成至少参考以下信息：

- 章节级风险信号
- 数值证据密度
- 审计意见与持续经营表述
- 债务、流动性、受限资金、减值、公允价值等高影响词
- 动态主题标签

## 选择方法

1. 先按风险信号打分。
2. 再用数值证据密度补充分数。
3. 聚合到主题或规则级别，而不是停留在单条句子。
4. 输出前 5 到 6 个重点。

## 输出要求

每个重点都必须回答三个问题：

- 为什么选它
- 证据来自哪些章节
- 还缺什么知识才能固化

如果重点来自从未沉淀过的新主题，应在 `focus_list_scaffold.json` 和对应章节复核记录中保留证据，后续由 Codex 决定是否直接写入正式知识库。
