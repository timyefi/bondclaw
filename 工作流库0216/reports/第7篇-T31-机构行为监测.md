# 第7篇 机构行为监测：债券市场微观结构的全景透视

## 摘要

机构投资者行为是债券市场最重要的定价力量。本项目构建了14维度的机构行为监测体系，覆盖交易量、交易矩阵、期限偏好、券种偏好、市场份额、利差监测、行为指数、风险定价等核心维度。通过交易对手模拟算法解决了数据限制问题，为债券投资提供了全方位的机构行为视角。标准化配置和模块化设计确保了系统的可扩展性和可维护性。

**关键词**：机构行为、债券市场、交易矩阵、期限偏好、行为指数

---

## 一、业务背景与需求分析

### 1.1 研究痛点

机构行为监测面临的主要挑战：

1. **维度复杂**：机构行为涉及交易量、期限、券种、区域等多个维度
2. **数据限制**：交易明细不包含买卖双方机构信息
3. **标准不一**：不同数据源的期限和券种分类差异大
4. **实时要求**：需要每日更新监测数据

### 1.2 业务价值

机构行为监测的应用场景：
- 跟踪主力机构配置方向
- 识别市场情绪变化
- 发现套利机会
- 预警流动性风险

### 3.1 图表1：各机构类型每日净交易量

```python
def get_chart1_daily_net_trading(self, date: str = None) -> pd.DataFrame:
    """图表1: 各机构类型每日净交易量"""
    sql = f"""
    SELECT
        `机构类型`,
        `净买入交易量（亿元）`
    FROM bond.`现券成交分机构统计表`
    WHERE `交易日期` = '{date}';
    """
    df = self.db_manager.execute_query(sql)

    # 数据清洗
    df['净买入交易量（亿元）'] = clean_numeric_column(df['净买入交易量（亿元）'])

    # 按机构类型分组聚合
    df_result = df.groupby('机构类型')['净买入交易量（亿元）'].sum().reset_index()
    df_result.rename(columns={'机构类型': 'name', '净买入交易量（亿元）': 'value'}, inplace=True)

    return df_result
```

### 3.2 图表2：机构间交易矩阵（含交易对手模拟）

```python
def get_chart2_trading_matrix(self, date: str = None) -> Dict[str, Any]:
    """图表2: 机构间交易矩阵"""
    # 获取机构交易统计（用于概率模拟）
    stats_sql = f"""
    SELECT `机构类型`, `期限`, `债券类型`,
           avg(`买入交易量（亿元）`) AS buy_vol,
           avg(`卖出交易量（亿元）`) AS sell_vol
    FROM bond.`现券成交分机构统计表`
    WHERE `交易日期` <= '{date}'
      AND `交易日期` >= DATE_SUB('{date}', INTERVAL 14 DAY)
    GROUP BY `交易日期`, `机构类型`;
    """
    stats_df = self.db_manager.execute_query(stats_sql)

    # 获取交易明细
    dealtinfo_df = self.db_manager.execute_query(dealtinfo_sql)

    # 模拟交易对手
    dealtinfo_enhanced_df = simulate_trading_parties(dealtinfo_df, stats_df)

    # 生成交易矩阵
    trade_matrix = dealtinfo_enhanced_df.pivot_table(
        index='party_type_s',
        columns='party_type_b',
        values='transaction_amount',
        aggfunc='sum'
    ).fillna(0)

    return format_heatmap_data(trade_matrix)
```

### 3.3 期限分类映射函数

```python
def map_tenor_to_category(term_str: str) -> str:
    """将期限文本转换为分类"""
    if term_str is None:
        return '1-3Y'  # 默认

    if 'Y+' in term_str:
        match = re.match(r'(\d+\.?\d*)Y', term_str)
        if match:
            years = float(match.group(1))

        if years <= 1: return '<=1Y'
        if years <= 3: return '1-3Y'
        if years <= 5: return '3-5Y'
        if years <= 7: return '5-7Y'
        if years <= 10: return '7-10Y'
        if years <= 15: return '10-15Y'
        if years <= 20: return '15-20Y'
        if years <= 30: return '20-30Y'
        return '>30Y'

    elif 'D' in term_str:
        match = re.match(r'(\d+\.?\d*)D', term_str)
        if match:
            days = float(match.group(1))
            years = days / 365
            if years <= 1:
                return '<=1Y'
        return '1-3Y'

    return '1-3Y'  # 默认
```

BOND_ASSET_CLASSES = {

## 四、心得要点

### 心得一：14维度全景式机构行为监测体系

**问题背景**：机构行为分析维度多样，不同研究需求关注点不同，缺乏统一的监测框架。

**解决方案**：构建14维度监测体系，分为6大模块：
- **交易量分析**：净交易量、市场份额
- **交易结构分析**：交易矩阵、配置流向
- **偏好分析**：期限偏好、券种偏好
- **市场监测**：热点追踪、利差监测、活跃券
- **指标监测**：行为指数、风险定价、记分卡、资金面
- **区域分析**：城投债资金来源

**业务价值**：为债券投资提供全方位的机构行为视角，支持从宏观（市场份额）到微观（个券追踪）的多层次分析，满足不同投资策略的监测需求。

### 心得二：交易对手模拟算法突破数据限制

**问题背景**：交易明细（dealtinfo）仅包含债券代码和交易金额，不包含买卖双方的机构信息，无法直接分析机构间交易流向。

**解决方案**：设计交易对手模拟算法：
1. 获取各机构类型的买卖交易量分布（14日平均）
2. 根据分布计算各机构类型作为买方/卖方的概率
3. 使用概率模型模拟每笔交易的对手方
4. 聚合生成机构间交易矩阵

```python
def simulate_trading_parties(dealtinfo_df, stats_df):
    """模拟交易对手"""
    # 计算各机构类型的买卖概率
    total_buy = stats_df['buy_vol'].sum()
    total_sell = stats_df['sell_vol'].sum()

    for idx, row in dealtinfo_df.iterrows():
        # 模拟买方
        buyer_probs = stats_df['buy_vol'] / total_buy
        dealtinfo_df.loc[idx, 'party_type_b'] = np.random.choice(
            stats_df['机构类型'], p=buyer_probs
        )
        # 模拟卖方
        seller_probs = stats_df['sell_vol'] / total_sell
        dealtinfo_df.loc[idx, 'party_type_s'] = np.random.choice(
            stats_df['机构类型'], p=seller_probs
        )

    return dealtinfo_df
```

**业务价值**：在数据限制下实现了机构间交易流向分析，为理解市场微观结构提供参考。虽然模拟结果存在不确定性，但可用于识别大方向的交易格局变化。

### 心得三：期限和券种的智能分类映射

**问题背景**：不同数据源的期限和券种分类标准不统一（如"5Y+" vs "5年"，"中期票据" vs "中票"），导致分析结果不可比。

**解决方案**：设计智能映射函数：
- **期限映射**：支持"Y+"（年）、"D"（天）等多种格式，统一映射到9档标准分类
- **券种映射**：支持细分券种名称，统一映射到3大类资产

**业务价值**：解决了不同数据源分类标准不统一的问题，确保分析结果的一致性和可比性。新增数据源时只需调整映射函数，无需修改核心分析逻辑。
