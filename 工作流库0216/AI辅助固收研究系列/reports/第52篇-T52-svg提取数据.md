## 1. 研究背景与问题

### 1.1 为什么需要这项研究？

在固定收益研究和投资分析工作中，数据是所有研究的基础。然而，实际工作中经常面临一个棘手的问题：许多重要的历史数据和行业数据仅以图表形式存在于研究报告、学术论文或官方统计年鉴中，原始数据表却无从获取。

这种情况在以下场景尤为常见：

**第一，历史研究报告的图表数据。** 许多券商、银行的研究报告发布时附带了宝贵的统计数据图表，但随着时间推移，原始数据文件可能已经丢失或存储在已经无法访问的旧系统中。当我们需要对这些历史数据进行更新或重新分析时，只能面对图表"望洋兴叹"。

**第二，监管机构和行业协会的统计图表。** 央行、银保监会、交易商协会等机构发布的报告中常有重要的市场统计数据，但往往以可视化图表呈现，不提供下载的CSV或Excel文件。研究者需要手工记录数据，效率低下且容易出错。

**第三，学术论文和第三方研究的数据引用。** 学术期刊发表的研究中常有经过精心整理的数据可视化，但这些数据的原始来源可能已经不可追溯。当我们需要验证研究结果或进行延伸分析时，只能从图表中"逆向"提取数据。

**第四，竞品和市场分析。** 在分析竞争对手的产品数据或市场份额时，能够从公开报告中提取图表数据进行量化对比，是形成客观结论的重要基础。

### 1.2 核心研究问题

本项目聚焦于解决以下核心问题：

1. **如何从可视化图表中高精度地还原原始数值数据？** 这涉及建立图表像素坐标与实际数值之间的精确映射关系。

2. **如何利用SVG格式图表的结构特点进行高效数据提取？** SVG作为一种矢量图形格式，其XML结构中包含了丰富的几何信息，这为数据提取提供了独特的便利。

3. **如何设计一套通用的图表数据提取方法论？** 不同类型的图表（柱形图、折线图等）和不同的坐标系统（线性、对数）需要采用不同的处理策略。

4. **如何在精度与效率之间取得平衡？** 数据提取需要保证足够的精度以支持后续分析，同时又不能过于复杂以至于难以实际应用。

### 1.3 研究价值

这项研究的价值体现在以下层面：

**对投研效率的提升。** 传统方式下，研究者需要手工从图表读取数据点，一个包含20个数据点的图表可能需要15-30分钟才能完成数字化。而采用本项目的自动化方法，提取同样数量的数据点只需几秒钟，效率提升数十倍。

**对数据覆盖面的拓展。** 许多重要的历史数据和历史趋势分析因为缺乏原始数据而无法进行。图表数据提取技术的成熟，使得大量"沉睡"在图表中的数据可以被重新激活利用，拓展了研究的数据基础。

**对研究可重复性的增强。** 手工抄录数据容易出现抄录错误，且难以被他人验证。自动化的数据提取过程可以被记录和复现，提高了研究的严谨性。

**对跨源数据整合的促进。** 当数据来源于不同机构的不同报告时，图表数据提取提供了一种统一的数据获取方式，便于进行跨源比较和整合分析。

## 2. 研究思路与方法

### 2.1 整体研究框架

本项目的整体研究框架可以概括为"结构解析-坐标映射-数值转换"三个核心环节：

```
SVG文件 --> XML结构解析 --> 提取图形元素坐标 --> 坐标到数值的映射 --> 输出数据表
```

**环节一：SVG结构解析**

SVG（Scalable Vector Graphics，可缩放矢量图形）是一种基于XML的矢量图形格式。与PNG、JPG等位图格式不同，SVG直接存储图形的几何描述而非像素阵列。这意味着SVG中的每个图形元素（如柱形、线条、文本）都有明确的坐标、尺寸等属性。

本环节使用Python标准库`xml.etree.ElementTree`解析SVG文件的XML结构，提取出表示数据图形的元素。对于柱形图，主要提取`<rect>`元素的x、y、height属性；对于折线图，主要提取`<path>`元素的d属性（路径命令）。

**环节二：坐标映射建立**

SVG坐标系以像素为单位，原点通常在画布左上角，Y轴向下为正。这与我们习惯的数据坐标系（Y轴向上为正）完全不同。因此，需要建立从SVG像素坐标到实际数值的映射关系。

本项目采用"三点标定法"：在图表上识别三个已知数值点（通常选择最小值、中间值、最大值），记录其SVG像素坐标和实际数值，据此构建映射函数。

**环节三：数值转换输出**

基于建立的坐标映射关系，将所有提取的像素坐标转换为实际数值，并生成结构化的数据表。对于时间序列图表，还需根据X轴范围生成对应的时间标签。

### 2.2 核心方法论

本项目的核心方法论是**基于已知点的分段线性插值转换法**。其基本原理如下：

**线性插值的数学基础**

假设我们知道两个已知点：点1的SVG Y坐标为$y_1$，对应实际数值为$actual_1$；点2的SVG Y坐标为$y_2$，对应实际数值为$actual_2$。

对于任意介于$y_1$和$y_2$之间的SVG Y坐标$y$，其对应的实际数值$actual$可以通过线性插值计算：

$$actual = actual_2 - \frac{y_2 - y}{y_2 - y_1} \times (actual_2 - actual_1)$$

这个公式的直观含义是：$y$在$y_1$和$y_2$之间的相对位置，决定了$actual$在$actual_1$和$actual_2$之间的相对位置。

**三点标定的必要性**

为什么需要三个已知点而不是两个？主要原因有两点：

1. **覆盖全数据范围。** 如果只用两个点进行插值，当数据点落在两个标定点范围之外时，需要进行外推，精度会显著下降。使用三个点（最小值、中间值、最大值）可以确保所有数据点都落在已知范围内。

2. **分段插值提高精度。** 对于非线性特征的坐标轴（虽然看起来是线性的，但SVG渲染可能存在微小偏差），分段插值可以更好地捕捉局部特征，提高整体精度。

**Y轴零点自动检测**

对于柱形图，一个重要的技术挑战是如何正确处理正负值。柱形图的正值柱形从零点向上延伸，负值柱形从零点向下延伸，两者在SVG中的表示方式不同。

本项目采用统计方法自动检测Y轴零点位置：统计所有柱形的y坐标值，出现频率最高的y值即为零点位置。这是因为零点是正负柱形的共同起点，出现次数最多。

## 3. 数据来源与处理

### 3.1 数据来源说明

| 数据类型 | 来源 | 更新频率 | 用途 |
|---------|------|---------|------|
| SVG图表文件 | 研究报告网页、浏览器开发者工具 | 按需 | 数据提取的原始输入 |
| 已知点坐标 | 人工从图表识别 | 每次提取时 | 建立坐标映射 |
| 时间范围参数 | 人工根据图表X轴确定 | 每次提取时 | 生成时间标签 |

**SVG文件的获取方式**

获取SVG图表文件通常有以下几种方式：

1. **浏览器开发者工具。** 在网页上右键点击图表，选择"检查"打开开发者工具，在HTML结构中找到对应的SVG元素，复制外层HTML即可。

2. **右键另存。** 部分网站支持直接右键点击SVG图表选择"另存为"。

3. **网络请求拦截。** 对于动态加载的图表，可以通过浏览器的Network面板找到SVG文件的请求地址。

**已知点坐标的获取**

在浏览器开发者工具中，将鼠标悬停在图表上的任意位置，可以看到该位置的像素坐标。选择三个已知数值点（如Y轴的最小刻度、中间刻度、最大刻度），记录其像素Y坐标和对应的实际数值。

### 3.2 数据处理要点

**SVG元素属性的提取**

对于柱形图，关键属性包括：
- `x`：柱形的X坐标（决定柱形在X轴的位置）
- `y`：柱形顶部的Y坐标
- `height`：柱形的高度

对于折线图，关键在于解析`<path>`元素的`d`属性，其中包含SVG路径命令：
- `M`：移动到某点（moveto）
- `L`：画直线到某点（lineto）
- `H`：水平线到某点
- `V`：垂直线到某点
- `C/S/Q/T`：各种贝塞尔曲线命令

本项目使用正则表达式解析这些路径命令，提取出所有控制点的坐标。

**坐标转换的处理流程**

1. 读取SVG文件，解析XML结构
2. 根据图表类型（柱形图/折线图）提取相应元素的坐标
3. 应用三点标定的线性插值函数，将SVG Y坐标转换为实际数值
4. 根据X轴时间范围生成对应的时间标签
5. 组合输出为结构化数据表

## 4. 核心分析框架

### 4.1 分析维度

从业务角度，本项目的分析框架可以划分为以下几个维度：

**维度一：图表类型适应性**

不同类型的图表需要采用不同的提取策略：

- **柱形图**：提取`<rect>`元素，处理正负值差异
- **折线图**：解析`<path>`元素的路径命令
- **面积图**：类似折线图，但需要识别填充区域边界
- **散点图**：提取`<circle>`元素的圆心坐标

本项目目前重点支持柱形图和折线图两种最常见的图表类型。

**维度二：坐标系统处理**

坐标系统的差异直接影响插值方法的选择：

- **线性坐标**：直接应用线性插值
- **对数坐标**：先对实际值取对数，进行线性插值后再反变换

对数坐标的处理公式：
$$log_{actual} = log(actual_2) - \frac{y_2 - y}{y_2 - y_1} \times [log(actual_2) - log(actual_1)]$$
$$actual = 10^{log_{actual}}$$

**维度三：时间序列标签生成**

对于时间序列图表，X轴的映射需要考虑：

- **等间距假设**：当前方法假设时间点等间距分布
- **起止时间参数**：需要人工指定数据的起止年月
- **频率推断**：根据数据点数量和起止时间自动推断是月度还是季度数据

**维度四：精度验证机制**

为确保提取数据的可靠性，需要建立验证机制：

- **已知点回验**：将已知点的SVG坐标代入插值函数，验证输出值与实际值的误差
- **阈值控制**：设置可接受的误差阈值（如0.1），超过阈值则报警
- **异常值检测**：识别明显偏离趋势的数据点进行人工复核

## 5. 心得要点

### 要点一：图表数据数字化的通用方法论

**背景**：在投研工作中，经常会遇到只能获取到研究报告中的图表而无法获取原始数据的情况。传统的解决方案是手工抄录，不仅效率低下，还容易出错。

**做法**：本项目采用的"坐标映射+线性插值"方法具有通用性，可应用于任何可视化图表的数据提取场景。核心步骤包括：(1)识别图表上的已知数值点；(2)建立像素坐标到实际数值的映射关系；(3)应用插值算法转换所有数据点。

**洞察**：图表数据数字化的本质是建立"视觉空间"到"数据空间"的映射关系。关键方法论要点包括：

1. **参考点选择原则**：选择覆盖数据全范围的参考点（最小值、中间值、最大值），确保所有数据点都落在已知范围内，避免外推带来的精度损失。

2. **坐标系统理解**：不同图表可能采用不同的坐标系统（线性/对数），需要正确识别并选择对应的插值方法。对数坐标的处理需要对数值进行对数变换。

3. **特殊元素处理**：正确处理图表中的特殊元素，如柱形图中的正负值区分、折线图中的数据点标记等。这些细节处理直接影响提取准确性。

4. **精度与复杂度权衡**：线性插值在大多数场景下足够精确，计算复杂度O(1)，实现简单。只有在曲线变化剧烈或对精度要求极高的场景下，才需要考虑更复杂的插值方法。

这一方法论可推广至图片格式图表的OCR提取场景，区别仅在于坐标获取方式从直接解析SVG变为图像识别。

**关联**：与WebPlotDigitizer等成熟工具的核心原理相通，但本项目针对SVG格式进行了专门优化，无需图像处理步骤，精度更高。

### 要点二：SVG作为数据载体的独特优势

**背景**：在进行图表数据提取时，数据源的格式直接影响提取的难度和精度。常见的图表格式包括位图（PNG、JPG）和矢量图（SVG、PDF）两大类。

**做法**：本项目选择SVG格式作为主要处理对象，直接解析其XML结构提取坐标数据，而非采用图像处理算法。

**洞察**：SVG格式的图表本质上是一种"结构化数据载体"，具有以下独特优势：

1. **几何信息的完整保留**：SVG中的每个图形元素都有明确的坐标、尺寸属性，不存在位图的像素化问题。这意味着精度可以达到亚像素级——只要SVG坐标有足够的小数位。

2. **无需图像处理算法**：提取数据不需要边缘检测、轮廓识别等复杂的图像处理算法，直接读取XML属性即可。这大大降低了实现复杂度，也避免了图像处理带来的精度损失。

3. **可区分数据系列**：SVG元素的class属性通常包含了数据系列的信息（如不同颜色的柱形代表不同类别），这为多系列数据的分别提取提供了便利。

4. **元数据可利用**：SVG中可能包含文本标签、图例等元素，这些都可以作为辅助信息帮助理解数据含义。

理解SVG结构是高效提取数据的关键。在Web环境中，Highcharts、D3.js、ECharts等主流可视化库都支持输出SVG格式，这意味着本方法有广泛的应用场景。

**关联**：与网页爬虫技术相结合，可以实现研究报告图表数据的批量采集和提取。

### 要点三：插值算法选择的经验法则

**背景**：数据提取的核心是将像素坐标转换为实际数值，这需要选择合适的插值算法。不同的插值方法在精度、效率、实现复杂度上有不同的权衡。

**做法**：本项目采用分段线性插值作为核心算法，同时对对数坐标提供专门支持。

**洞察**：在图表数据提取场景中，插值算法的选择应遵循以下经验法则：

1. **线性插值的适用性**：对于大多数商业图表，Y轴采用线性坐标，线性插值是最直接、最可靠的选择。计算复杂度O(1)，实现简单，不会产生过拟合。

2. **分段处理的必要性**：即使图表整体看起来是线性的，SVG渲染过程中可能存在微小的非线性偏差。使用三个已知点进行分段插值，可以有效消除这种偏差。

3. **对数坐标的识别**：如果图表的Y轴刻度呈现等比递增（如1、10、100、1000），则表明采用了对数坐标。此时需要在对数空间进行插值，然后反变换回原始数值。

4. **避免过度复杂化**：除非有明确的精度需求，否则不建议使用多项式插值或样条插值。这些方法虽然理论上精度更高，但实现复杂度高，且可能产生龙格现象等数值问题。

5. **精度验证是必须的**：无论采用什么插值方法，都应该用已知点进行回验，确保误差在可接受范围内。这是保证数据质量的最后一道防线。

**关联**：这一思路可延伸至时间序列数据的缺失值填补、传感器数据的异常值校正等场景。

### 要点四：正负值处理的自动化思维

**背景**：在处理柱形图时，正负值的正确处理是一个容易出错的环节。正值柱形从零点向上延伸，负值柱形从零点向下延伸，两者在SVG中的表示方式完全不同。

**做法**：本项目采用统计方法自动检测Y轴零点位置，据此区分正值柱形和负值柱形。

**洞察**：正负值处理的自动化体现了数据提取中的一个重要思维模式——从数据的统计特征中提取规则，而非硬编码：

1. **零点是最频繁出现的Y坐标**：在柱形图中，零点是所有柱形的基准点。正值的y属性等于零点位置，负值的y属性也等于零点位置（只是height向下延伸）。统计所有柱形的y值，出现频率最高的就是零点。

2. **区分正负的逻辑**：当y坐标等于零点时，如果height向下延伸（在SVG中height为正，但视觉上向下），则为负值柱形。当y坐标小于零点时，柱形从y坐标向下延伸到零点，为正值柱形。

3. **自动化vs硬编码**：传统方法可能要求用户指定零点位置，但自动检测更灵活，可以适应不同来源的图表。这种"从数据中学习规则"的思路在很多数据处理场景都适用。

4. **边界情况的处理**：当图表只有正值或只有负值时，零点检测仍然有效，只是统计分布会更加偏斜。需要设置合理的阈值来处理这种边界情况。

**关联**：这种基于统计特征的自动化思维可推广至其他图表元素的识别，如自动识别X轴标签、图例位置等。

## 6. 实际应用指南

### 6.1 如何使用这项研究

**使用前提条件**

在使用本方法进行图表数据提取前，需要满足以下条件：

1. **图表格式为SVG**：目标图表必须是SVG格式或可以转换为SVG格式。大多数基于Web的可视化库（Highcharts、ECharts、D3.js等）输出的图表都是SVG格式。

2. **能够识别已知数值点**：需要在图表上识别至少3个已知数值点，通常是Y轴的最小刻度、中间刻度和最大刻度。

3. **了解X轴时间范围**：对于时间序列图表，需要知道数据的起止时间。

**完整操作流程**

**Step 1：获取SVG源码**
- 在浏览器中打开包含目标图表的网页
- 右键点击图表，选择"检查"打开开发者工具
- 在Elements面板中找到对应的SVG元素
- 右键点击SVG元素，选择"Copy" -> "Copy outerHTML"
- 将复制的内容保存为.svg文件

**Step 2：标定已知点**
- 在图表上识别3个已知数值点
